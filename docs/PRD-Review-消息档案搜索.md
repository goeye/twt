## 问题清单

### 🔴 严重问题（阻塞发布）

#### B.1 缺失搜索输入与加载交互细节
**位置**：3.2.1 在线会话搜索 & 3.2.2 档案搜索
**问题**：未说明在线会话的实时过滤机制（如防抖时间、触发最小字符数），也未说明档案页面点击搜索按钮后的加载状态（Loading表现）及如果后端响应慢的交互处理。
**建议**：补充搜索输入的触发机制（如："输入停止 500ms 后触发实时过滤"），以及点击搜索按钮后的加载状态说明。

#### B.2 缺失消息高亮状态的持续行为
**位置**：3.2.1 & 3.2.2 定位规则
**问题**：描述了"消息列表滚动至该选中消息并高亮"，但未定义高亮是永久存在还是定时消失，以及是否有取消高亮的交互机制。
**建议**：明确高亮行为（例如："高亮状态持续 3 秒后渐隐消失"或"高亮状态保持至用户关闭详情抽屉或点击其他消息"）。

#### C.1 缺失特殊类型消息的搜索规则说明
**位置**：3.2.1 搜索规则
**问题**：虽然排除了系统消息和富文本，但遗漏了针对业务上"已撤回/已删除"消息的搜索规则声明。
**建议**：补充说明对已撤回和已删除消息的搜索行为要求（如："包含已撤回、删除的原始消息内容"）。

### 🟡 中等问题（建议修复）

#### A.1 缺失搜索输入框校验规则
**位置**：3.2 交互流程
**问题**：未定义搜索框的最大字符长度限制、是否支持前导/尾随空格的自动修剪（trim）、以及是否允许输入纯特殊符号。
**建议**：补充输入规则："搜索关键词最大长度限制为 100 字符，自动修剪首尾空格"。

#### B.3 缺失弹窗/抽屉的关闭交互路径
**位置**：3.2.1 列表弹窗呈现
**问题**：仅说明了"右上角包含关闭图标"和"点击消息项后关闭"，未说明是否支持点击遮罩层外部空白处关闭、是否支持 Esc 键快捷关闭。
**建议**：补充："支持点击遮罩层外部区域关闭弹窗/抽屉，支持按 Esc 键关闭"。

#### E.1 缺失"无匹配结果"状态的 UI 呈现
**位置**：6. 异常处理
**问题**：未明确说明当输入关键词后没有任何会话/消息匹配时，列表区域和弹窗区域的具体空状态文案和表现。
**建议**：补充空数据状态："搜索无结果时，列表区展示空状态，提示文案为'暂无符合条件的会话/聊天记录'"。

#### G.1 弹窗与抽屉滚动规则模糊
**位置**：3.2.1 & 3.2.2 呈现规则
**问题**：描述了弹窗和抽屉，但未明确当匹配消息列表数据过多超过屏幕高度时，列表容器如何进行内部滚动。
**建议**：明确："弹窗/抽屉内的消息列表区域支持垂直滚动，弹窗头部/底部区域固定"。

### 🟢 轻微问题（优化建议）

#### G.2 异常提示文案不规范
**位置**：6. 异常处理
**问题**："用户点击进入上下文后，发现不到该消息，且无高亮" 表述不专业，未说明系统的正面反馈。
**建议**：修改为："系统弹出 Toast 提示'该消息已被移除，无法定位'，并正常打开会话详情但不执行滚动高亮"。

#### H.1 违规使用 UI 样式描述
**位置**：3.2.2 交互流程 步骤2
**问题**：出现了"蓝色的「N条相关记录」可点击文本"的描述，违反了 PRD 编写规范中禁止使用颜色描述的规定。
**建议**：删除颜色描述，修改为："展示「N条相关记录」的文本按钮"。

---

## 改进建议

1. **补全交互微细节**：搜索实时反馈的时间窗口、无数据的空状态提示、高亮的持久化规则，这些决定了功能体验的最终闭环。
2. **遵守 PRD 规范**：严格剔除所有包含颜色（如"蓝色"）的描述，将 UI 决定权留给设计系统。
3. **闭环异常处理**：遇到找不到消息这种边界情况，应当设计合理的系统提示（如 Toast）安抚用户，而不是仅让用户"发现不到"。

# PRD：网页渠道快捷入口优化

> **版本**：v1.0 · 2026-04-01
> **状态**：已交付

---

## 1. 概述

### 1.1 背景与动机

| 痛点 | 影响 |
|------|------|
| 快捷入口仅支持跳转链接和复制文本，功能单一 | 无法满足自动发送常见问题等场景，访客仍需手动输入 |
| 图标一旦设置无法清除，且为必填项 | 管理员配置灵活度不足 |
| 交互方式字段名称和提示文案不够直观 | 管理员理解成本高 |

本次优化在已有快捷入口功能基础上，新增「发送消息」交互方式（支持自定义内容和关联知识库常见问题），优化图标字段为非必填并支持删除，同时调整字段名称和提示文案使其更加清晰。

### 1.2 目标

| Key Result | 量化标准 |
|-----------|---------|
| KR1：新增发送消息交互方式 | 支持自定义内容和关联常见问题两种消息来源 |
| KR2：关联知识库 | 发送消息支持选择知识库中已录入的常见问题 |
| KR3：优化配置体验 | 图标改为非必填并支持删除，字段文案更直观 |

## 3. 功能设计

### 3.1 核心流程

```mermaid
flowchart TD
  A["选择交互方式：发送消息"] --> B{"消息来源"}
  B -->|"自定义内容"| C["输入消息文本（多行，最多2000字符）"]
  B -->|"关联常见问题"| D["下拉选择知识库问题"]
  C --> E["点击确定"]
  D --> E
  E --> F{"校验通过?"}
  F -->|"是"| G["保存并关闭弹窗"]
  F -->|"否：内容为空"| H["提示输入/选择"]
  F -->|"否：问题已删除"| I["提示重新选择并清空"]
  H --> A
```

### 3.2 子功能详述

#### 3.2.1 图标字段优化

**功能描述**：图标改为非必填，已有图标支持删除。

**需求描述**：
1. 图标字段改为非必填
2. 已有图标时（无论自动填充还是手动上传），显示删除按钮
3. 点击删除按钮清除图标，恢复为空状态，可重新上传

---

#### 3.2.2 字段名称和文案调整

**功能描述**：优化弹窗中的字段名称和提示文案，使其更清晰易懂。

**需求描述**：

| 调整项 | 调整前 | 调整后 |
|--------|--------|--------|
| 字段名称 | 访问内容 | 交互方式 |
| 选项名称 | URL | 打开链接 |
| 选项名称 | 文本内容 | 复制文本 |
| 打开链接输入框提示 | 请输入URL | 请输入链接地址 |
| 复制文本输入框提示 | 请输入文本内容 | 请输入需要复制的文本 |
| 说明提示（tooltip） | 选择 URL 时，请填写完整网页地址，访客点击后会直接跳转；选择文本内容时，访客点击后会直接复制该内容。 | 打开链接：访客点击后跳转到指定网页；复制文本：访客点击后复制指定内容；发送消息：访客点击后自动发送预设消息 |

---

#### 3.2.3 发送消息交互方式

**功能描述**：新增「发送消息」交互方式，访客点击快捷入口后自动发送预设消息。

**用户场景**：管理员希望访客点击快捷入口后自动发送问题，由系统或客服给出回答，减少访客手动输入。

**交互流程**：
1. 在交互方式中选择「发送消息」
2. 系统显示消息来源选择器和对应的输入区域（合并为同一行，左侧为消息来源选择器，中间以竖线分隔，右侧为输入区域）
3. 选择消息来源并填写内容
4. 提交时系统校验内容有效性

**需求描述**：
1. 消息来源选项：
   - **自定义内容**（默认选中）：右侧显示多行文本输入区域，提示文案「请输入消息内容」，最多输入 2000 个字符
   - **关联常见问题**：右侧显示下拉选择器，提示文案「请选择常见问题」，选项来源为知识库中已录入的常见问题（不区分是否训练成功）
2. 切换消息来源时，已输入的内容不保留
3. 校验规则：
   - 自定义内容为空时提交：提示「请输入消息内容」
   - 未选择常见问题时提交：提示「请选择常见问题」

---

#### 3.2.4 常见问题关联与异常处理

**功能描述**：处理已关联的常见问题被删除的异常场景。

**需求描述**：

1. **已选常见问题被删除（弹窗未打开时）**：下次打开编辑弹窗时，系统检测到所选问题已不存在，在下拉选择器下方显示行内警告「所选常见问题已被删除，请重新选择」。点击确定时阻止提交，提示「所选常见问题已被删除，请重新选择」，同时清空选择。
2. **已选常见问题被删除（弹窗已打开时）**：弹窗打开时的常见问题列表为快照数据，被删除的问题仍可见但不影响选择操作。提交时系统做最终校验，若所选问题已不存在，提示「所选常见问题已被删除，请重新选择」并清空选择。

## 4. 数据模型

本次新增字段（在已有快捷入口实体基础上扩展）：

| 字段 | 类型 | 说明 |
|------|------|------|
| 消息来源 | 枚举 | 自定义内容 / 关联常见问题（仅交互方式为发送消息时有效） |
| 关联常见问题标识 | 文本 | 非必填，仅消息来源为关联常见问题时有效 |

已有字段变更：

| 字段 | 变更内容 |
|------|---------|
| 图标 | 由必填改为非必填 |
| 交互方式 | 新增枚举值「发送消息」 |

---

## 7. 异常处理

| 异常场景 | 处理方式 | 用户感知 |
|---------|---------|---------|
| 自定义消息内容为空提交 | 阻止提交 | 提示「请输入消息内容」 |
| 未选择常见问题提交 | 阻止提交 | 提示「请选择常见问题」 |
| 所选常见问题已被删除（弹窗关闭状态） | 打开编辑弹窗时显示行内警告，提交时阻止并清空选择 | 行内警告 +「所选常见问题已被删除，请重新选择」 |
| 所选常见问题已被删除（弹窗打开状态） | 提交时校验阻止并清空选择 | 提示「所选常见问题已被删除，请重新选择」 |
| 自定义消息内容超过 2000 字符 | 输入框限制无法继续输入 | 无额外提示 |

> **文档版本**: v1.1

> **产品**: TWT 海外客服 SaaS
> **模块**: Web Agent 设置中心 & 聊天小部件自定义

---

## 1. 概述

### 1.1 产品背景

TWT 是一款面向海外市场的在线客服 SaaS 平台。客服人员通过 Web Agent 工作台与访客进行实时沟通。为满足企业对客服系统的个性化配置需求，平台提供了「设置中心」和「聊天小部件自定义」两大功能模块，让管理员和客服团队能够灵活配置系统行为、品牌展示和服务流程。

### 1.2 目标用户

| 角色 | 说明 |
|------|------|
| 管理员 | 负责系统全局配置，包括安装部署、团队管理、安全策略等 |
| 客服主管 | 管理客服团队设置、自定义聊天小部件、配置快捷回复 |
| 客服坐席 | 使用个人回复模板，查看安装信息 |

### 1.3 产品目标

- 提供一站式的系统配置入口，覆盖安装、团队、快捷回复、标签、安全、开发等维度
- 支持聊天小部件的全方位自定义，包括外观、内容、表单和通用功能
- 支持多语言内容配置（英文、简体中文、繁体中文）
- 提供实时预览能力，确保配置效果所见即所得

## 3. 功能需求详述

### 3.1 设置中心 — 聊天页面（安装）

#### 3.1.1 方法一：聊天页面链接

**功能描述**: 提供一个可直接分享的聊天页面 URL，访客打开即可对话，无需网站部署。

- 展示系统生成的聊天页面基础链接
- 提供「复制」按钮，一键复制到剪贴板
- 适用渠道：邮件、App、社交媒体等

#### 3.1.2 方法二：HTML 下载部署

**功能描述**: 提供 HTML 部署包下载，用户上传至自有服务器后可通过自有域名访问。

**操作步骤**:
1. **下载 HTML 文件** — 点击「立即下载」获取已配置好的 HTML 部署包
2. **上传至服务器** — 将文件上传至自有服务器（提供教程链接）
3. **使用自有域名分享** — 配置域名指向（提供注意事项说明）

#### 3.1.3 自定义配置（可选）

**语言切换**: 在链接末尾添加 `?lang=语言代码` 参数：
- `zh-cn` — 简体中文
- `zh-tw` — 繁体中文
- `en` — 英文

**客户信息传入**: 支持通过 URL 参数将内部系统的客户信息传入聊天页面。

| 参数名 | 说明 |
|--------|------|
| `sbs` | 客户唯一标识 |
| `sbs_name` | 客户名称 |
| `sbs_mm` | 签名校验值 |
| `ranstr` | 随机字符串 |

**签名生成规则**: `md5(md5(sbs + '_' + AppSecret) + '_' + ranstr)`，md5 值为小写 32 位。

> **安全提示**: `AppSecret` 需在开发设置中生成，请妥善保管，避免泄露。

### 3.2 设置中心 — 客服设置

#### 3.2.1 客服不活跃

**功能描述**: 当客服在工作台上超过指定时间未进行任何操作时，系统自动将其在线状态更改为「离开」。

**配置项**:
- **开关**: 启用/禁用此功能
- **超时时间**: 可配置分钟数（数字输入框，最小值 1）

**业务规则**:
- 触发条件：客服未进行任何操作（包括鼠标点击、键盘输入等）
- 触发动作：自动将状态改为「离开」
- 状态恢复：客服重新操作后应可手动恢复在线状态

#### 3.2.2 会话超时

**功能描述**: 当客服超过指定时间未回复访客消息时，系统自动将该会话移回排队中。

**配置项**:
- **开关**: 启用/禁用此功能
- **超时时间**: 可配置分钟数（数字输入框，最小值 1）

**业务规则**:
- 触发条件：客服在指定时间内未回复访客的消息
- 触发动作：会话自动进入排队中，会话中所有服务客服将被自动释放
- 目的：避免访客因客服长时间未响应而等待过久

### 3.3 设置中心 — 快捷回复

#### 3.3.1 公共回复

**功能描述**: 管理全团队共享的快捷回复模板，所有客服均可使用。

**功能要点**:
- 输入快捷回复文本后按回车或点击「新增」添加
- 已添加的模板以列表形式展示
- 每条模板可单独删除
- 提供「保存」按钮持久化配置

#### 3.3.2 个人回复

**功能描述**: 管理客服个人的快捷回复模板，仅对当前客服可见。

**功能要点**: 与公共回复操作方式相同，但作用域为个人。

## 4. 聊天小部件自定义 (WidgetCustomizePage)

### 4.1 页面结构

页面采用左右分栏布局：
- **左侧**: 配置面板（分 Tab 组织各配置项）
- **右侧**: 实时预览面板（支持 4 种预览模式）

### 4.2 外观 Tab (Appearance)

#### 4.2.1 品牌标识

**功能描述**: 自定义聊天小部件中展示的品牌信息。

**配置项**:
- **品牌 Logo**: 支持上传 PNG/JPG/JPEG 格式图片，点击 Logo 区域触发上传，支持替换
- **品牌名称**: 文本输入框，显示在小部件头部 Logo 旁

**交互细节**:
- Logo 上传区域 hover 时显示替换提示
- Logo 区域显示当前已上传的图片预览
- 品牌名称默认值为 "TWT"

#### 4.2.2 小部件位置

**功能描述**: 设置聊天小部件在访客网页上的悬浮位置。

**配置项**:
- **位置选择**: 左下角 / 右下角（卡片式单选，含位置示意图）
- **距离底部 (px)**: 数字输入框，范围 20-200
- **距离左/右边 (px)**: 数字输入框，范围 20-200（标签文字根据位置选择动态切换）

#### 4.2.3 快捷入口

**功能描述**: 配置访客在聊天界面中可看到的快捷入口按钮，支持点击跳转到指定 URL。

**配置项**:
- **多语言支持**: 英文 / 简体中文 / 繁体中文 Tab 切换
- **快捷入口列表**: 以 Tag 形式展示，每个 Tag 包含图标、标签文本、编辑/删除操作
- **添加按钮**: 点击新增快捷入口

**数据结构**:
```typescript
interface QuickAccessItem {
  id: string;      // 唯一标识
  label: string;   // 显示文本
  url: string;     // 跳转链接
}
```

#### 4.2.4 官方标识

**功能描述**: 控制聊天小部件底部 "Powered by Chat" 标识的展示。

**配置项**:
- **隐藏官方标识**: 开关控件（默认关闭，即默认展示标识）

---

### 4.3 内容 Tab (Content)

#### 4.3.1 自动回复消息

系统提供 3 种自动回复场景，每种都可独立开关并配置多语言内容：

| 场景 | 说明 | 触发时机 |
|------|------|----------|
| **欢迎消息** | 访客进入聊天时的自动欢迎语 | 访客发起新会话时 |
| **结束消息** | 会话结束后的自动提示 | 会话被关闭时 |
| **离线消息** | 无客服在线时的自动提示 | 所有客服离线时 |

**每种消息的配置项**:
- **开关**: 独立启用/禁用
- **多语言编辑**: 英文 / 简体中文 / 繁体中文 Tab 切换
- **文本内容**: 富文本编辑区域（textarea）
- **图片附件**: 支持上传图片，png/jpg/jpeg 格式，单张 < 2MB，最多 10 张

#### 4.3.2 访客评价

**功能描述**: 会话关闭后，系统自动邀请访客对服务进行评价。

**配置项**:
- **开关**: 启用/禁用评价功能
- **评价标题**: 多语言文本配置（英文 / 简体中文 / 繁体中文）

**评价选项**（固定 3 级）:
- 😊 满意
- 😐 一般
- 😞 不满意

**预览效果**: 在聊天预览模式中展示评价卡片 UI，包含标题和三个评价选项。

---

### 4.4 表单 Tab (Form)

#### 4.4.1 会话前表单

**功能描述**: 在访客开始会话之前，展示一个信息收集表单，帮助客服提前了解访客需求。

**配置项**:
- **开关**: 启用/禁用表单
- **多语言支持**: 英文 / 简体中文 / 繁体中文 Tab 切换
- **表单标题**: 文本输入框
- **表单字段**: 动态字段列表

**字段管理**:
- 每个字段包含：拖拽排序手柄、字段名称、占位符文字、必填开关、删除按钮
- 点击「添加字段」新增字段
- 支持拖拽排序（drag-handle UI）

**数据结构**:
```typescript
interface FormField {
  id: string;          // 唯一标识
  type: string;        // 字段类型
  label: string;       // 字段标签
  placeholder: string; // 占位符文字
  required: boolean;   // 是否必填
}
```

### 4.5 通用 Tab (General)

#### 4.5.1 会话功能

##### 发起会话

**功能描述**: 控制访客是否可以主动发起新会话。

**配置项**:
- **开关**: 开启后聊天小部件中显示「新的会话」按钮
- **限制会话数上限**: 二级开关
  - **最大会话数**: 数字输入框，范围 1-99

**业务规则**:
- 启用会话数限制后，如果访客删除未结束的会话，该会话仍占用额度
- 系统提供警告提示：建议关闭「删除会话」功能以避免额度浪费

##### 删除会话

**功能描述**: 控制访客是否可以删除会话。

- **开关**: 开启后访客会话列表显示删除按钮

##### 主动结束会话

**功能描述**: 控制访客是否可以在删除会话时选择同时结束会话。

- **开关**: 开启后访客删除会话时可勾选「同时结束会话」

#### 4.5.2 访客不活跃

**功能描述**: 当访客超过指定时间未回复客服消息时，系统自动关闭会话。

**配置项**:
- **开关**: 启用/禁用此功能
- **超时时间**: 数字输入框，可配置分钟数（最小值 1）

#### 4.5.3 消息与状态

##### 已读回执

- **开关**: 开启后访客可在会话中看到客服是否已读其消息

##### 客服在线状态（聊天）

- **开关**: 开启后访客可在单聊中看到客服的在线状态

#### 4.5.4 隐藏联系我们

**功能描述**: 控制 App 端「我的」页面中「联系我们」入口的显示。

- **开关**: 开启后隐藏该入口

## 5. 实时预览面板

### 5.1 预览模式

自定义页面右侧提供 4 种预览模式，通过下拉选择器切换：

| 模式 | 说明 | 展示内容 |
|------|------|----------|
| **首页 (sessionList)** | 会话列表视图 | 品牌头部、会话列表、新会话按钮、Powered by 标识 |
| **消息 (chat)** | 聊天对话视图 | 聊天头部（头像+标题）、消息气泡、快捷入口、输入框工具栏 |
| **表单 (form)** | 会话前表单视图 | 表单标题、动态字段、提交按钮 |
| **最小化 (minimized)** | 悬浮按钮视图 | 品牌色圆形悬浮按钮，点击可恢复至最小化前的预览模式 |

### 5.2 预览窗口最小化与恢复

#### 5.2.1 功能描述

预览小部件的 Header 顶部右侧提供关闭「×」按钮，点击后将小部件最小化为悬浮圆形图标（FAB）。再次点击 FAB 可恢复到最小化前的预览页面。

#### 5.2.2 交互逻辑

| 操作 | 触发条件 | 行为 |
|------|----------|------|
| **点击关闭按钮** | 预览处于首页/消息/表单任一模式 | 记录当前预览模式，切换为最小化（FAB）视图 |
| **点击 FAB 图标** | 预览处于最小化模式 | 恢复到最小化前的预览模式（首页/消息/表单） |
| **下拉选择器切换** | 通过顶部下拉选择器手动切换预览模式 | 若切换至非最小化模式，同步更新"上次预览模式"记录 |

#### 5.2.3 UI 规格

- **关闭按钮**: 28×28px 圆形热区，居于 Header 右侧，图标为 14×14px 的 `×`
- **默认状态**: 图标颜色 `--agent-color-text-secondary`
- **Hover 状态**: 背景色 `--agent-color-bg-hover`，图标颜色 `--agent-color-text-primary`
- **FAB Hover**: 缩放至 1.08 倍，过渡动画 0.2s
- **覆盖范围**: 首页、消息、表单三种预览模式的 Header 均包含关闭按钮

#### 5.2.4 状态管理

```typescript
// 记录最小化前的预览模式，用于恢复
previewModeBeforeMinimize: PreviewMode  // 默认 "sessionList"
```

- 每次 `previewMode` 变更为非最小化值时，自动更新 `previewModeBeforeMinimize`
- 调用 `minimizeWidget()` 时，先保存当前模式再切换至 `"minimized"`
- 调用 `restoreWidget()` 时，将 `previewMode` 恢复为 `previewModeBeforeMinimize` 的值

### 5.3 预览特性

- **位置同步**: 预览小部件位置根据「小部件位置」配置实时调整
- **品牌同步**: 品牌 Logo 和名称实时反映在预览头部
- **功能联动**:
  - 「发起会话」开关影响预览中「新的会话」按钮的显示
  - 「删除会话」开关影响预览中会话条目删除按钮的显示
  - 「主动结束会话」影响删除确认弹窗中「同时结束会话」选项的显示
  - 「官方标识」开关影响预览底部 "Powered by Chat" 的显示
  - 快捷入口配置实时反映在聊天预览中
  - 自动回复消息内容实时显示在聊天预览气泡中
  - 访客评价开启时聊天预览切换为评价卡片视图

### 5.4 预览中的交互模拟

- **删除会话弹窗**: 点击预览中的会话删除按钮，展示确认弹窗（含「同时结束会话」选项）
- **会话头像和标题**: 预览显示配置的头像/标题信息
- **最小化与恢复**: 点击 Header 关闭按钮最小化窗口，点击 FAB 恢复

## 6. 未保存更改保护

### 6.1 机制描述

自定义页面实现了完整的未保存更改保护机制，防止用户意外丢失配置。

### 6.2 实现方式

- **快照对比**: 页面加载时保存配置快照，后续通过 JSON 序列化对比检测变更
- **导航拦截**: 使用 Vue Router `beforeEach` 守卫拦截路由跳转
- **确认弹窗**: 当检测到未保存更改时，弹出确认对话框，提供「放弃修改」和「继续编辑」选项
- **浏览器关闭保护**: 监听 `beforeunload` 事件，在浏览器关闭/刷新时提示

### 6.3 暴露接口

```typescript
// 供父组件调用的方法
defineExpose({
  requestNavigation(action: () => void): boolean;  // 请求导航，有未保存时显示确认
  hasUnsavedChanges(): boolean;                      // 查询是否有未保存更改
});
```

### 6.4 脏状态通知

通过 `dirty-change` 事件向父组件 (App.vue) 通知脏状态变化，父组件据此决定是否允许设置导航切换。

## 8. 技术实现要点

### 8.1 技术栈

- **框架**: Vue 3 + TypeScript
- **路由**: vue-router 4
- **构建工具**: Vite
- **UI 组件库**: @twt/ui-agent（内部组件库）
- **设计令牌**: @twt/tokens（CSS 自定义属性 `--agent-*`）

### 8.2 组件依赖

| 组件 | 来源 | 用途 |
|------|------|------|
| `AiSettingsNav` | @twt/ui-agent | 设置中心左侧导航 |
| `AgentSwitch` | @twt/ui-agent | 开关控件 |
| `DataTable` | @twt/ui-agent | 参数说明表格 |
| `BaseModal` | @twt/ui-agent | 确认弹窗 |
| `AgentToast` | @twt/ui-agent | 操作提示 |

### 8.3 页面组件关系

```
App.vue (主布局)
├── AiSettingsNav (设置导航)
├── SettingsRoutePage (设置页面路由)
│   ├── 聊天页面 (install)
│   ├── 客服设置 (team)
│   ├── 快捷回复 (quick-reply / personal-reply)
│   └── 占位页面 (其他未实现功能)
└── WidgetCustomizePage (自定义页面，独立路由)
    ├── 配置面板 (左侧)
    │   ├── Appearance Tab
    │   ├── Content Tab
    │   ├── Form Tab
    │   └── General Tab
    └── 预览面板 (右侧)
```

### 8.4 样式规范

- CSS 类名使用 BEM 风格，前缀 `wc-`（Widget Customize）或 `settings-`
- 所有颜色、间距、圆角等通过 `--agent-*` CSS 自定义属性引用
- 禁止在页面文件中使用硬编码颜色值

## 9. 交互规范

### 9.1 手风琴面板

- 配置项以手风琴（Accordion）形式组织
- 同一 Tab 下同一时间只能展开一个手风琴面板
- 点击触发区切换展开/收起
- 部分手风琴头部包含独立的开关控件（点击开关不触发手风琴展开）

### 9.2 保存操作

- 每个 Tab 底部均提供独立的「保存」按钮
- 保存成功后通过 Toast 提示
- 保存操作会更新快照基准线，清除脏状态

### 9.3 图片上传

- 支持格式：PNG、JPG、JPEG
- 单张限制：< 2MB
- 最多上传数量：10 张（自动回复图片）
- 上传后即时预览缩略图
- 支持逐张删除

---

## 10. 非功能需求

### 10.1 性能

- 预览面板应在配置变更后 100ms 内响应更新
- 图片上传支持本地预览（不阻塞 UI）

### 10.2 兼容性

- 支持主流现代浏览器（Chrome、Firefox、Safari、Edge 最新两个大版本）
- 响应式布局：配置面板与预览面板在窄屏下应合理适配

### 10.3 安全

- AppSecret 仅在开发设置中生成，不在前端明文展示
- 图片上传需进行格式和大小校验
- URL 参数签名机制防止访客身份伪造

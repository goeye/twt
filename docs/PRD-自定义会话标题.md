# PRD：自定义会话标题

> **版本**：v1.0 · 2026-03-25
> **状态**：待开发
> **模块编号**：Module 15-A

---

## 1. 概述

### 1.1 背景与动机

| 痛点 | 影响 |
| --- | --- |
| 访客端会话列表和会话详情中无法展示负责客服的信息 | 访客无法直观识别正在服务的客服是谁，降低了服务的个性化感知和信任感 |

在线会话自定义模块已支持外观、内容、会话表单和常规设置。本次在常规标签页中新增**会话标题展示方式**配置能力，让运营团队可以灵活控制访客端的信息展示策略。

### 1.2 目标

| Key Result | 量化标准 |
| --- | --- |
| KR1：会话标题可配置 | 运营可在「AI 生成标题」和「客服信息」之间自由切换，切换后访客端实时生效 |

### 1.3 非目标

*   客服端不受会话标题设置的影响，保持现有展示逻辑


会话标题位于「会话功能」下方、「访客不活跃」上方。

### 3.2 核心流程

```mermaid
flowchart TD
  A[运营进入自定义 → 常规] --> B[展开会话标题折叠面板]
  B --> C[右侧预览面板切换到会话列表预览模式]
  C --> D[选择 AI 生成标题 / 客服信息]
  D --> E[自动保存，提示保存成功]
  E --> F[预览面板实时更新展示效果]

```

### 3.3 功能详述

**功能描述**：设置访客端会话标题的展示方式，支持「AI 生成会话标题」和「客服信息」两种模式。

**用户场景**：运营团队希望根据业务需求选择访客端会话的展示风格——使用 AI 自动生成的语义化标题，或展示负责客服的真实信息以增强信任感。

**前置条件**：

1.  用户具有自定义管理权限


**交互流程**：

1.  运营展开「会话标题」折叠面板

2.  右侧预览面板自动切换到会话列表预览模式

3.  运营在两个选项中点选其一

4.  系统立即保存，提示「保存成功」

5.  预览面板实时更新展示效果


**需求描述（功能规则）**：

1.  **选项说明**：

    *   **AI 生成会话标题**：由 AI 根据会话内容自动生成标题（默认选中）

    *   **客服信息**：展示会话负责人信息

2.  **「客服信息」模式下的访客端行为**：

    *   会话列表：头像替换为负责客服头像，标题根据语言显示：

        *   英文：「Chat with Agent Sarah」

        *   简体中文：「与客服小雨的会话」

        *   繁体中文：「與客服小雨的會話」

    *   会话详情页顶部：头像替换为负责客服头像，标题同上，标题不可修改

    *   无会话负责人时：头像显示项目 Logo，标题显示项目名称，标题不可修改（超出部分省略号截断）

    *   客服头像或昵称发生变更时，访客端同步更新

    *   会话负责人变更时，访客端同步更新

    *   会话从无负责人变为有负责人时，访客端同步更新

3.  **「AI 生成会话标题」模式下的访客端行为**：

    *   会话列表和详情页显示 AI 生成的语义化标题，根据语言自动生成对应语言的标题

    *   示例标题：

        *   英文：「Order Status Inquiry」

        *   简体中文：「订单状态咨询」

        *   繁体中文：「訂單狀態諮詢」

    *   示例最后一条消息：

        *   英文：「Can you help me check the order status?」

        *   简体中文：「能帮我查询一下订单状态吗？」

        *   繁体中文：「能幫我查詢一下訂單狀態嗎？」

4.  **仅影响访客端**：客服端保持现有展示逻辑不变

5.  **保存方式**：选择即保存


**后置条件**：

1.  配置项持久化存储

2.  访客端所有会话（含历史会话）立即按新模式展示

---

## 5. 异常处理

| 异常场景 | 处理方式 | 用户感知 |
| --- | --- | --- |
| 展示「客服信息」模式下会话负责人被移出会话 | 该会话回退到项目头像和项目名称 | 访客看到的头像和标题自动切换 |
| 会话标题模式切换时网络异常 | 保存失败，配置不生效 | 不弹出保存成功提示 |

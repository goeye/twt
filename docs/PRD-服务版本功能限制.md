# PRD：服务版本功能限制

## 1. 概述

### 1.1 背景

产品采用 SaaS 分层定价模式，划分免费版与专业版两个服务等级。通过客服端（web-agent）的功能门控机制，控制不同版本用户可使用的功能范围，同时引导免费版用户升级至专业版。

### 1.2 目标

- 在客服端前端实现功能级别的版本门控，免费版/服务过期后用户触达受限功能时给予明确提示
- 通过升级引导弹窗驱动免费版/服务过用户向专业版转化
- 版本门控与角色权限正交独立，互不干扰

### 1.3 非目标

- 后端版本校验与计费逻辑
- 支付流程与订单管理
- 官网定价页与权益对比表（需其他团队同步更新）

## 2. 用户故事

| 编号 | 角色 | 场景 | 期望 |
|------|------|------|------|
| US-1 | 免费版/服务过期超级管理员 | 点击"新增角色"按钮 | 弹出升级弹窗，显示"取消"和"立即升级"两个按钮 |
| US-2 | 专业版已过期超级管理员 | 进入任意受限功能页面并操作 | 版本视为免费版，操作被拦截并弹出升级弹窗 |
| US-3 | 专业版用户 | 正常使用所有受限功能 | 无任何拦截，功能正常可用 |

## 3. 功能设计

### 3.1 版本体系

系统定义版本等级：

| 版本 | 说明 |
|------|------|
| 免费版（free） | 基础功能可用，18 项专业功能受限 |
| 服务过期（expired） | 基础功能可用，18 项专业功能受限 |
| 专业版（pro） | 全功能可用 |

**过期规则**：专业版到期后，系统自动将生效版本变为过期（视为免费版）。已有数据保留可查看，受限操作被拦截。

### 3.3 升级提示弹窗

当免费版/服务过期用户触发受限操作时，弹出全局升级提示弹窗。

**弹窗结构**：
- 顶部：装饰性插画
- 中部：标题"升级到专业版" + 描述"该功能在当前服务版本不可用，请升级到专业版"
- 底部：显示"取消"+ "立即升级"

**弹窗交互**：
- "立即升级"按钮预留跳转至升级页面的逻辑（当前关闭弹窗）

### 3.4 开关类设置处理规则

部分功能以开关形式存在于设置页面，免费版/服务过期处理规则如下：

| 开关类功能 | 免费版/服务过期行为 |
|-----------|-----------|
| Copilot 智能回复 | 开关默认关闭，尝试开启时弹升级 |
| 文本润色 | 开关默认关闭，尝试开启时弹升级 |
| 边写边译 | 开关默认关闭，尝试开启时弹升级 |
| 聊天翻译 | 开关默认关闭，尝试开启时弹升级 |
| 隐藏官方标识 | 开关默认关闭，尝试开启时弹升级 |
| 隐藏联系我们 | 开关默认关闭，尝试开启时弹升级 |
| Webhooks 事件 | 开关默认关闭，尝试开启时弹升级 |
| AI Agent 总开关 | 默认关闭，尝试开启时弹升级，配置区域只读 |
| 主动营销状态开关 | 免费版/服务过期开关始终显示关闭状态，切换弹升级 |

### 3.5 降级后的联动处理

#### AI Agent 会话处理
- 专业版到期或降级后，已有 AI Agent 正在接待的会话继续正常服务
- 新会话不再分配给 AI Agent 接待

#### Copilot 输入框按钮联动
- 当版本不支持对应 Copilot 功能时，会话输入框中的相关按钮同步隐藏：
  - 文本润色不可用 → 隐藏润色按钮
  - 边写边译或聊天翻译均不可用 → 隐藏翻译按钮
  - 仅当两个翻译功能都不可用时才隐藏翻译按钮，任一可用则显示

### 3.6 交互原则

1. **导航菜单不锁定**：免费版/服务过期用户可正常进入所有页面，不在菜单上做版本限制
3. **操作时拦截**：仅在用户点击受限操作时弹出升级弹窗
4. **列表可查看**：对于知识库、角色管理、主动营销等，免费版/服务过期可查看已有数据列表，增删改操作被拦截

## 4. 权限与版本的关系

### 4.1 正交模型

版本门控（usePlan）与角色权限（usePermission）是两套独立的能力检查体系：

| 维度 | 版本门控 | 角色权限 |
|------|---------|---------|
| 判断依据 | 租户订阅等级（free/pro）+ 是否过期 | 当前用户角色拥有的权限集合 |
| 检查位置 | 页面内操作事件中（guardFeature / canUse） | 路由守卫 + 导航过滤 + 操作前 |
| 拦截表现 | 弹出升级弹窗 | 路由重定向至首页 + 权限变更弹窗 |
| 作用范围 | 租户级别（影响该租户下所有用户） | 用户级别（不同角色权限不同） |
| 相互依赖 | 无 | 无 |

### 4.2 检查顺序

1. **路由守卫（进入页面前）**：仅检查角色权限，不检查版本。无权限则重定向至首页
2. **页面内操作（进入页面后）**：先检查版本（guardFeature），版本不允许则弹升级弹窗；版本通过后由业务逻辑处理
3. 两者不存在冲突场景：权限决定"能否进入页面"，版本决定"页面内能否执行特定操作"

## 5. 数据模型

### 5.1 版本变更刷新逻辑

- **升级到专业版**：需要刷新页面后生效，刷新后才能使用专业版功能
- **过期或降级到免费版**：功能操作立即被后端拦截不可用，但前端开关控件需刷新页面后才会回到关闭状态

## 6. 跨模块联动

### 6.1 官网权益对比表

功能限制清单（3.2 节）的变更需同步更新官网的版本权益对比表，确保前端门控与营销页面宣传一致。官网对比表不在本 PRD 范围内，由其他团队负责，但本模块功能变更时需通知同步。

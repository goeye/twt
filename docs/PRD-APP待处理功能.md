# PRD - APP 待处理功能

## 需求概述

在移动端中实现"待处理"功能，与客服网页端的操作逻辑保持一致。客服可以将会话标记为待处理状态，用于标识需要后续跟进的会话。

## 功能范围

### 包含功能

- 移除会话输入框底部的"标记为待处理"按钮
- 会话详情页面中支持标记为待处理
- 会话详情页面中支持取消待处理

## 交互规则

### 标记为待处理

**触发条件**
- 会话未关闭
- 当前状态不是待处理

**触发入口**
- 移除会话输入框底部的"标记为待处理"按钮，会话标记为待处理无需发送消息
- 会话详情页面的操作菜单中，增加"标记待处理"选项

**操作流程**
1. 客服在会话详情页面点击操作菜单（右上角三点图标）
2. 在菜单中点击"标记待处理"
3. 显示 Toast 提示："已标记为待处理"
4. 会话状态变更为待处理在会话列表中移动到"待处理"标签页。待处理会话无论访客回复还是客服回复都不会更改状态，除非客服手动取消待处理。
5. 标记为"待处理"的会话不会走自动关闭会话的逻辑

**状态变更**
- 会话从当前队列（待回复/已回复）移动到待处理队列

### 取消待处理

**触发条件**
- 会话未关闭
- 当前状态是待处理

**触发入口**
- 会话详情页面的操作菜单中，显示"取消待处理"选项

**操作流程**
1. 客服在会话详情页面点击操作菜单
2. 在菜单中点击"取消待处理"
3. 显示 Toast 提示："已取消待处理"
4. 会话在会话列表中根据当前会话状态移动到对应标签页

**状态变更**
- 会话根据当前会话状态从待处理队列移动到对应队列

### 操作菜单逻辑

**会话详情页面的操作菜单**

当前会话状态为待处理时：
- 修改标题
- 访客信息
- 会话信息
- 取消待处理 ← 新增
- 结束会话

当前会话状态不是待处理时：
- 修改标题
- 访客信息
- 会话信息
- 标记待处理 ← 新增
- 结束会话

**菜单项显示规则**
- 如果会话状态是待处理，显示"取消待处理"，隐藏"标记待处理"
- 如果会话状态不是待处理，显示"标记待处理"，隐藏"取消待处理"

**并发操作**
- 如果多个客服同时操作同一会话，以服务端响应为准
- 客户端需要同步服务端的最新状态

# PRD：成员管理

> **版本**：v2.0 · 2026-03-18
> **状态**：草稿
> **模块编号**：Module 02

---

## 1. 概述

### 1.1 背景与动机

| 痛点 | 影响 |
|------|------|
| 团队需要灵活管理客服人员的加入、离开和岗位变更 | 缺乏系统化管理工具导致运营效率低、坐席资源浪费 |
| 原系统中使用「客服」称呼，但实际团队成员不局限于客服岗位 | 称呼不统一造成用户认知混乱 |

成员管理模块提供团队成员的完整生命周期管理能力，包括邀请加入、坐席名额管理、账号详情编辑等功能。本次迭代同步完成全局更名：将系统中所有「客服」相关称呼统一为「成员」。

### 1.2 目标

| Key Result | 量化标准 |
|-----------|---------|
| KR1：支持邮件邀请新成员加入团队 | 单次最多发送 10 张邀请，每日频率上限为坐席总数 × 2 |
| KR2：坐席名额精确管理 | 邀请中成员占用坐席，取消邀请释放坐席，统计准确 |
| KR3：完成「客服」→「成员」全局更名 | 所有页面中「客服」相关称呼统一替换为「成员」 |

### 1.3 非目标（本期不做）

- 不支持成员自助注册（仅通过邀请方式加入）
- 不支持批量导入成员
- 不涉及权限树配置和权限联动规则（详见「角色管理」PRD）

## 3. 功能设计

### 3.1 信息架构

```mermaid
graph LR
  A[设置] --> B[团队]
  B --> C[成员]
  B --> D[角色]
  B --> E[成员设置]
  C --> F[成员列表]
  F --> G[邀请成员]
  F --> H[账号详情]
  F --> I[取消邀请]
  F --> J[删除成员]
```

**功能入口**：系统导航路径 **设置 → 团队 → 成员**（独立子菜单）。

**导航结构变更**：原「客服」页面含「客服」和「角色」两个 Tab，现改为「团队」分组下三个独立子菜单：成员、角色、成员设置。

### 3.2 核心流程

```mermaid
flowchart TD
  A[进入成员列表] --> B{选择操作}
  B -->|邀请| C[打开邀请弹窗]
  C --> D[选择角色 + 输入邮箱]
  D --> E{校验}
  E -->|坐席已满| F[无法邀请]
  E -->|超出每日上限| G[Toast: 今日邀请次数已达上限]
  E -->|通过| H[发送邀请]
  H --> I[邀请记录出现在列表中]
  B -->|取消邀请| J[确认弹窗]
  J --> K[邀请记录移除，坐席释放]
  B -->|账号详情| L[查看/编辑成员信息]
  B -->|删除| M[确认弹窗]
  M --> N[成员移除]
```

### 3.3 子功能详述

#### 3.3.1 成员列表

**功能描述**：以表格形式展示所有已激活成员和邀请记录。

**用户场景**：管理员查看团队成员状况和邀请状态。

**前置条件**：
1. 用户拥有团队相关权限

**需求描述（功能规则）**：

1. **页面布局**：通栏白底卡片，标题「成员」位于左上角，坐席统计信息在标题下方，「+ 邀请成员」按钮位于右上角
2. **表格列定义**：

| 列名 | 已激活成员 | 邀请记录（邀请中） |
| --- | --- | --- |
| ID | 成员编号（如 10001） | 显示 `-` |
| 姓名 | 头像 + 姓名 | 显示 `-` |
| 昵称 | 成员昵称 | 显示 `-` |
| 加入时间 | 成员加入项目时间 | 显示 `-` |
| 角色 | 角色名称 | 邀请时分配的角色 |
| 邮箱 | 成员邮箱 | 邀请邮箱 |
| 会话限制 | 最大并发会话数 | 显示 `-` |
| 在线状态 | 在线/离开/离线（带颜色圆点） | 显示 `-` |
| 状态 | 启用/停用开关 | 状态文字「邀请中」 |
| 操作 | 更多操作按钮 | 更多操作按钮 |

3. **排序规则**：已激活成员在前（按加入时间倒序），邀请记录在后（按邀请时间倒序）
4. **已激活成员操作菜单**：

| 操作 | 说明 | 禁用条件 |
| --- | --- | --- |
| 发起聊天 | 与该成员发起内部聊天 | 当前用户自身 |
| 账号详情 | 进入账号详情页 | 无 |
| 修改密码 | 发送密码重置邮件 | 当前用户自身 |
| 删除 | 删除该成员 | 当前用户自身 |

5. **邀请记录操作菜单**：仅「取消邀请」一项
6. **分页**：底部分页器

#### 3.3.2 坐席名额管理

**功能描述**：成员列表头部展示坐席统计信息。

**需求描述（功能规则）**：

| 指标 | 计算规则 |
| --- | --- |
| 坐席总数 | 当前服务包购买的坐席数量 |
| 使用中 | 已激活成员数 + 邀请中记录数 |
| 剩余 | 坐席总数 − 使用中 |

**核心规则**：
- 邀请中记录**占用**坐席名额
- 停用成员**不释放**坐席名额
- 页面显示「加购坐席」链接（功能开发中）

#### 3.3.3 邀请成员

**功能描述**：通过邮箱邀请新成员加入团队。

**用户场景**：管理员需要招募新成员加入客服团队。

**前置条件**：
1. 剩余坐席 > 0
2. 用户拥有管理成员权限

**交互流程**：
1. 点击成员列表右上角「+ 邀请成员」按钮
2. 弹出邀请弹窗

**需求描述（功能规则）**：

1. **弹窗内容**：
   - 标题：邀请成员加入团队
   - 描述：`团队成员将通过电子邮件被邀请，最多一次可以发送10张邀请。`
   - 角色选择：下拉选择器，可选项为系统中所有非管理员角色，默认选中「客服」。label「角色」加粗加大
   - 邮箱输入：多行文本框，支持换行/逗号/分号分隔多个邮箱。label「邮箱」加粗加大
   - 邀请按钮：邮箱为空时按钮不可点击

2. **邀请限制**（按以下顺序逐级校验）：

| 校验层级 | 限制规则 | 错误提示 |
| --- | --- | --- |
| 坐席名额 | 剩余坐席 ≤ 0 时不可邀请 | — |
| 每日频率 | 每日邀请总次数不超过坐席总数 × 2 | `今日邀请次数已达 {上限} 上限` |
| 单次邀请 | 单次邀请数超出每日剩余次数 | `今天可邀请 {上限} 次，还剩 {剩余} 次` |

3. **每日频率说明**：按自然天计算（UTC+8 0:00 重置）

**后置条件**：
1. 邀请记录出现在成员列表中，状态「邀请中」
2. 坐席名额使用中 +N，剩余 -N
3. Toast 提示邀请发送成功
4. 关闭邀请弹窗

#### 3.3.4 邀请记录状态与生命周期

**功能描述**：管理邀请的状态流转。

**需求描述（功能规则）**：

```
邀请发送 → 邀请中（链接永久有效）
              ├── 被邀请人点击链接激活 → 已激活（转为正式成员）
              └── 管理员取消邀请 → 已取消（链接立即失效）
```

| 状态 | 说明 | 占用坐席 | 可执行操作 |
| --- | --- | --- | --- |
| 邀请中 | 邀请已发送，等待被邀请人激活 | 是 | 取消邀请 |

**核心规则**：
- 邀请链接**无有效期限制**，不会自动过期
- 取消邀请后，对应邀请链接**立即失效**
- 如需重新邀请同一人，需先取消旧邀请记录，再重新发送

#### 3.3.5 取消邀请

**功能描述**：取消尚未激活的邀请记录。

**用户场景**：管理员需要释放被未激活邀请占用的坐席。

**交互流程**：
1. 从邀请记录操作菜单点击「取消邀请」
2. 弹出确认弹窗

**需求描述（功能规则）**：

1. **确认弹窗**：
   - 标题：取消邀请
   - 描述：`取消后本次邀请将失效，是否取消？`
   - 按钮：「取消」「确定」

**后置条件**：
1. 邀请记录从列表移除
2. 对应邀请链接立即失效
3. 释放坐席名额（使用中 -1，剩余 +1）
4. 对应角色关联成员数 -1
5. Toast 提示「邀请已取消」

#### 3.3.6 删除成员

**功能描述**：删除已激活的成员。

**用户场景**：管理员需要移除离职或不再需要的成员。

**前置条件**：
1. 不可删除当前登录用户自身

**交互流程**：
1. 从已激活成员操作菜单点击「删除」
2. 弹出确认弹窗

**需求描述（功能规则）**：

1. **确认弹窗**：
   - 标题：删除成员
   - 描述：`删除后不可恢复，确认删除？`
   - 按钮：「取消」「确认删除」

**后置条件**：
1. 成员从列表移除
2. Toast 提示「删除成功」

#### 3.3.7 账号详情

**功能描述**：查看和编辑成员的账号信息。

**用户场景**：管理员修改成员角色或查看成员信息；成员查看自己的账号详情。

**前置条件**：
1. 从成员列表操作菜单点击「账号详情」

**需求描述（功能规则）**：

1. **角色字段**：
   - 管理员查看他人：角色下拉框可编辑，选择后立即触发保存
   - 成员查看自己：角色字段为只读
2. **保存方式**：所有可编辑字段采用**失焦自动保存**机制（焦点离开字段时自动保存），无底部保存按钮
3. **保存成功**：Toast 提示「保存成功」

#### 3.3.8 启用/停用成员

**功能描述**：通过开关控制成员账号的启用/停用状态。

**需求描述（功能规则）**：
1. 成员列表「状态」列显示开关控件
2. 点击开关切换账号状态
3. 停用成员**不释放**坐席名额
4. Toast 提示 `{昵称} 账号已启用/已停用`

## 7. 业务规则汇总表

| # | 规则 | 触发条件 | 预期行为 |
| --- | --- | --- | --- |
| 1 | 邀请中记录占用坐席 | 邀请发送后 | 使用中名额包含邀请中记录 |
| 2 | 邀请链接无有效期 | 邀请发送后 | 链接在被激活或被取消前始终有效 |
| 3 | 取消邀请链接立即失效 | 取消邀请后 | 邀请链接立即失效，坐席释放 |
| 4 | 每日邀请频率上限 | 邀请发送时 | 每日邀请总次数 ≤ 坐席总数 × 2 |
| 5 | 单次邀请上限 | 邀请弹窗填写邮箱 | 最多一次发送 10 张邀请 |
| 6 | 成员不可修改自身角色 | 成员查看个人资料 | 角色字段只读 |
| 7 | 账号详情失焦保存 | 编辑账号详情字段 | 字段失焦后自动保存，Toast「保存成功」 |
| 8 | 角色选择即时保存 | 管理员修改他人角色 | 选择后立即触发保存，权限即时生效 |
| 9 | 停用成员不释放坐席 | 停用成员账号 | 使用中名额不变 |
| 10 | 不可删除自身 | 当前用户操作菜单 | 发起聊天、修改密码、删除按钮置灰不可点击 |
| 11 | 角色关联成员数实时统计 | 角色列表展示 | 含邀请中记录在内的使用该角色的成员数 |

## 8. 跨模块联动

| 联动模块 | 联动方式 | 说明 |
|----------|----------|------|
| 角色管理 | 成员的角色字段引用角色列表 | 邀请弹窗和账号详情中的角色选项来自角色列表（排除管理员角色） |
| 角色管理 | 角色的关联成员数统计 | 包含使用该角色的已激活成员和邀请中记录 |
| 角色管理 | 角色有关联成员不可删除 | 若角色仍有关联成员（含邀请中），角色管理侧无法删除该角色 |
| 首页 | 成员概览 | 首页显示成员在线/离开/离线统计及邀请成员入口 |
| 会话 | 添加成员 | 会话头部操作中「添加成员」功能引用成员列表数据 |

## 9. 名词解释

| 名词 | 说明 |
| --- | --- |
| 成员 | 系统中的团队用户，原称「客服」，本次统一更名 |
| 坐席 | 服务包中包含的成员名额，邀请中和已激活成员均占用坐席 |

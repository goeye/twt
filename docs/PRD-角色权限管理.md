# PRD：客服角色权限管理

> 版本：1.1
> 最后更新：2026-03-13
> 状态：已实现

---

## 1. 概述

### 1.1 功能定位

角色权限管理是客服系统的基础能力模块，用于定义和管理客服人员在系统中的操作权限边界。通过角色体系，管理员可以为不同职能的客服分配差异化的功能权限，实现精细化的权限控制。

### 1.2 功能目标

- 支持系统预设角色与自定义角色并存
- 支持三级权限树的灵活配置（一级菜单 → 二级菜单 → 功能权限）
- 确保角色与客服成员的关联约束，防止误删在用角色
- 提供角色的完整生命周期管理（新增、查看、编辑、删除）
- 支持客服邀请的待激活状态管理（重新邀请、删除未激活客服）
- 未激活客服占用坐席名额，确保名额管理准确

### 1.3 功能入口

系统导航路径：**设置 → 客服 → 角色 Tab**

客服管理页面包含「客服」和「角色」两个 Tab，切换到「角色」Tab 即进入角色列表页面。

## 2. 角色体系

### 2.1 系统角色

系统预设两个内置角色，**不可编辑、不可删除**：

| 角色名称 | 权限范围 | 创建人 | 说明 |
|---------|---------|-------|------|
| 管理员 | 全部权限（含所有菜单权限 + 会话操作权限） | 系统 | 拥有系统最高权限 |
| 客服 | 部分权限（查看档案、管理访客、管理客户、管理营销、查看客服、管理快捷回复、管理标签） | 系统 | 基础客服角色 |

系统角色特征：
- 操作菜单中仅显示「查看」，不显示「编辑」和「删除」
- 角色名称、权限配置均为只读状态

### 2.2 自定义角色

管理员可创建自定义角色，支持完整的增删改查操作。自定义角色的权限可从权限树中自由选择组合。

### 2.3 角色属性定义

| 属性 | 说明 | 约束 | 备注 |
|-----|------|------|------|
| 角色名称 | 角色的显示名称 | 必填，最大 50 字符 | - |
| 权限集 | 该角色拥有的权限项集合 | 可为空（允许创建无权限的角色） | - |
| 关联客服数 | 当前使用该角色的客服人数 | 系统自动统计，只读 | 无数据展示0 |
| 创建人 | 创建该角色的管理员名称 | 系统自动记录 | 系统内置角色创建人为「系统」 |
| 创建时间 | 角色创建的时间 | 系统自动记录，格式：YYYY-MM-DD HH:mm | 系统预设角色创建时间为项目创建时间 |

## 3. 权限体系

### 3.1 权限层级结构

权限采用三级树形结构：

```
一级菜单（Group）
  └── 二级菜单（Item）
        └── 功能权限（Feature）
```

- **一级菜单**：对应系统主导航的功能模块
- **二级菜单**：对应一级菜单下的子功能页面
- **功能权限**：对应二级菜单内的具体操作能力

> 并非所有二级菜单都有功能权限层级，部分二级菜单本身即为最细粒度。

### 3.4 管理员角色的特殊权限逻辑

管理员角色的权限在查看时自动填充为全选状态，且不可修改。

客服角色的默认权限集为：

## 4. 角色管理操作

### 4.1 角色列表

角色列表以表格形式展示，包含以下列：

| 列名 | 说明 |
|-----|------|
| 角色名称 | 创建角色时输入的角色名称，超出显示区域用「...」截断 |
| 关联客服数 | 显示当前使用该角色的客服人数|
| 创建人 | 系统角色显示「系统」，自定义角色显示创建客服昵称 |
| 创建时间 | 系统角色创建时间为项目创建时间，格式：YYYY-MM-DD HH:mm |
| 操作 | 更多操作按钮（···），点击展开下拉菜单 |

**排序规则**：系统角色排在前面（管理员、客服），自定义角色按创建时间倒序排列。

**操作菜单**：
- 系统角色：仅显示「查看」
- 自定义角色：显示「查看」「编辑」「删除」

### 4.2 新增角色

**入口**：角色 Tab 页面右上角的「+ 新增角色」按钮。

**流程**：
1. 点击「新增角色」按钮
2. 进入角色创建页面
3. 输入角色名称，配置权限
4. 点击「保存」完成创建

**页面标题**：新增角色

**校验规则**：
- 角色名称为必填项，留空时保存按钮置灰不可点击
- 角色名称最大长度 50 字符
- 权限配置可为空（允许创建无权限的角色）

**默认状态**：
- 角色名称为空
- 所有权限均未勾选（零权限起始）

**保存成功**：返回角色列表，提示「新增成功」

**页面底部**：显示「保存」按钮

### 4.3 查看角色

**入口**：
- 操作菜单中点击「查看」

**流程**：
1. 进入角色详情页面
2. 展示角色信息和权限配置（只读）
3. 点击返回箭头回到角色列表

**页面标题**：角色详情

**展示内容**：
- 角色名称
- 权限配置（只读，checkbox 不可点击）

**页面底部**：不显示操作按钮。

### 4.4 编辑角色

**入口**：操作菜单中点击「编辑」（仅自定义角色可见）。

**流程**：
1. 进入角色编辑页面，自动加载当前权限数据
2. 修改角色名称和/或权限配置
3. 点击「保存」完成编辑

**页面标题**：编辑角色

**可修改项**：
- 角色名称
- 权限配置

**校验规则**：与新增角色一致（名称必填，最大 50 字符）。

**保存成功**：返回角色列表，提示「编辑成功」。编辑角色后已关联客服角色权限立即生效，若客服访问无权限接口/路径，系统应返回403页面。

**页面底部**：显示「保存」按钮

### 4.5 删除角色

**入口**：操作菜单中点击「删除」（仅自定义角色可见）。

**前置校验**：检查该角色的关联客服数。

**场景一：角色有关联客服（memberCount > 0）**
- 弹出确认弹窗
- 弹窗标题：删除角色
- 弹窗描述：`删除后不可恢复、确认删除？`
- 弹窗操作：「确认删除」、「取消」按钮
- 点击确认删除按钮后toast提示：当前角色有 x 个关联客服，暂无法删除

**场景二：角色无关联客服（memberCount = 0）**
- 弹出确认弹窗
- 弹窗标题：删除角色
- 弹窗描述：`删除后不可恢复、确认删除？`
- 「删除」按钮可点击
- 点击「删除」执行删除操作

**删除成功**：角色从列表移除，提示 `删除成功`

## 5. 权限选择交互规则

### 5.1 三级联动选择

权限树的三个层级之间存在联动关系：

| 层级 | 对应概念 | 选择行为 |
|-----|---------|---------|
| Group（一级菜单） | 如「档案」「团队」「设置」 | 选中/取消选中该组下所有 Item 和 Feature |
| Item（二级菜单） | 如「查看档案」「客服」「管理安装」 | 选中/取消选中该 Item 及其下所有 Feature |
| Feature（功能权限） | 如「查看客服」「管理客服」 | 仅影响自身 + 自动选中父 Item |

### 5.2 半选（indeterminate）状态的触发条件

**Group 层级半选**：
- 该 Group 下有部分（非全部）子项被选中时，Group 的 checkbox 显示半选状态

**Item 层级半选**：
- 该 Item 拥有 Feature 子项，且部分（非全部）Feature 被选中时，Item 的 checkbox 显示半选状态

### 5.3 选中 Feature 自动选中父 Item 的逻辑

当用户勾选一个 Feature 权限时：
1. 该 Feature 被选中
2. 系统自动检查其父 Item 是否已选中
3. 若父 Item 未选中，则自动将父 Item 选中

> 这确保了选中「查看客服」时，父级「客服」也会自动被选中。

### 5.4 反选父级自动清除子级的逻辑

**取消选中 Group**：
- 清除该 Group 下所有 Item 和 Feature 的选中状态

**取消选中 Item**：
- 清除该 Item 本身及其下所有 Feature 的选中状态

**取消选中 Feature**：
- 仅清除该 Feature 自身
- 不会影响父 Item 的选中状态（父 Item 可能仍被其他 Feature 或自身选中）

## 6. 角色与客服关联

### 6.1 客服邀请时分配角色

在邀请新客服加入团队时，邀请弹窗中包含「角色」下拉选择器，可选项包括系统中所有角色（系统角色 + 自定义角色）。默认选中「客服」角色。

### 6.2 客服编辑时修改角色

在客服账号详情页面，**管理员**可以修改客服的所属角色。角色变更后，该客服的权限即时更新。若客服访问无权限接口/路径，系统应返回403页面。

> **注意**：客服在「个人资料」中查看自己的账号详情时，角色字段为**只读**，不显示修改入口。仅管理员通过客服管理页面进入他人账号详情时才可修改角色。详见 6.8 节。

### 6.3 角色的关联客服数统计

角色列表中的「关联客服数」列实时显示使用该角色的客服人数。当客服被分配到某角色或从某角色移除时，该计数自动更新。

> **注意**：关联客服数包含「待激活」状态的客服。即已邀请但尚未激活的客服也计入关联客服数。

### 6.4 关联约束：有成员的角色不可删除

若角色当前仍有关联客服（关联客服数 > 0，含待激活客服），则该角色不可删除。

### 6.5 客服列表中的待激活状态

客服被邀请后、完成激活前，处于「待激活」状态。客服列表中的展示规则：

| 属性 | 说明 |
|-----|------|
| 状态标签 | 客服姓名右侧显示「待激活」标签 |
| 在线状态 | 不显示在线/离线状态指示器 |
| 操作菜单 | 显示「重新邀请」「删除」（不显示「发起聊天」「账号详情」「修改密码」） |

**待激活客服的限制**：
- 不可登录系统
- 不可接待会话
- 无法分配会话
- 在「报表」与「档案 -> 会话记录」列表内无法筛选
- 但仍占用坐席名额（见 6.6 节）

### 6.6 坐席名额与未激活客服

**核心规则**：未激活（待激活）的客服**占用**坐席名额。

示例：项目拥有 10 个坐席名额，已邀请 5 个客服但均未激活，则：
- 已使用名额：5
- 剩余名额：5

邀请新客服时，系统校验已使用名额（含待激活）是否已达上限。若已满，提示错误。

### 6.7 重新邀请与删除未激活客服

#### 6.7.1 重新邀请

**入口**：客服列表中，待激活客服的操作菜单点击「重新邀请」。

**功能说明**：对已邀请但未激活的客服重新发送邀请邮件/链接。

**发送限制**：
- 同一客服的重新邀请间隔不少于 **60 秒**
- 在冷却期内，「重新邀请」按钮置灰，鼠标移入后tooltip提示（「60s 后可重新发送」）
- 重新邀请不改变该客服的角色分配和坐席占用

**操作成功**：Toast 提示「邀请已发送」

#### 6.7.2 删除未激活客服

**入口**：客服列表中，待激活客服的操作菜单点击「删除」。

**确认弹窗**：
- 弹窗标题：删除客服
- 弹窗描述：`删除后不可恢复，确认删除？`
- 操作按钮：「确认删除」「取消」

**删除成功**：
- 客服从列表移除
- 释放占用的坐席名额
- 对应角色的关联客服数 -1
- Toast 提示「删除成功」

### 6.8 客服个人资料中的角色展示

客服通过「个人资料」入口查看自己的账号详情时：
- 角色字段以**只读文本**展示，不显示下拉选择器或编辑按钮
- 客服无法自行修改自己的角色
- 仅拥有「管理客服」权限的角色通过客服管理页面进入他人账号详情时，才可修改角色

### 6.9 账号详情页面交互变更

客服账号详情页面的保存方式变更：

**原方案**：页面底部显示「保存」按钮，点击后统一提交。

**新方案**：
- **移除底部保存按钮**
- 所有可编辑字段（昵称、角色等）采用**失焦自动保存**（blur-to-save）机制
- 用户修改某个字段后，焦点离开该字段时自动触发保存请求
- 保存成功后 Toast 提示「保存成功」
- 保存失败后 Toast 提示具体错误信息，字段值回退到修改前的状态
- 只读字段（如邮箱、创建时间）不受影响

## 9. 名词解释

| 名词 | 说明 |
|-----|------|
| 系统角色 | 系统预设的内置角色（管理员、客服），不可编辑和删除 |
| 自定义角色 | 管理员手动创建的角色，支持完整的增删改查操作 |
| 权限树 | 以树形结构组织的权限配置界面，包含一级菜单、二级菜单、功能权限三个层级 |
| 一级菜单（Group） | 权限树的最顶层分组，对应系统主导航模块（如档案、访客、团队等） |
| 二级菜单（Item） | 一级菜单下的子功能页面或操作域 |
| 功能权限（Feature） | 二级菜单内的具体操作能力，是权限的最细粒度 |
| 半选状态（indeterminate） | checkbox 的中间态，表示其子级部分选中 |
| 关联客服数 | 当前使用该角色的在职客服人数 |
| 会话操作权限 | 不在权限树中展示的权限项，包括查看/转接/邀请/关闭会话 |
| 坐席 | 客服系统中的一个客服工位，每个客服占用一个坐席 |
| 待激活 | 客服已被邀请但尚未完成激活的状态，占用坐席名额但不可登录系统 |
| 失焦保存 | 编辑字段后焦点离开时自动触发保存请求的交互方式，替代传统的保存按钮 |

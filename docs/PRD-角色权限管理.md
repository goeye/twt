# PRD：客服角色权限管理

## 1. 概述

### 1.1 功能定位

角色权限管理是客服系统的基础能力模块，用于定义和管理客服人员在系统中的操作权限边界。通过角色体系，可以为不同职能的客服分配差异化的功能权限，实现精细化的权限控制。

### 1.2 功能目标

- 支持系统预设角色与自定义角色并存
- 支持三级权限树的灵活配置（一级菜单 → 二级菜单 → 功能权限）
- 确保角色与客服成员的关联约束，防止误删在用角色
- 提供角色的完整生命周期管理（新增、查看、编辑、删除）
- 支持客服邀请的待激活状态管理（删除未激活客服后链接失效）
- 未激活客服占用坐席名额，确保名额管理准确
- 多层级邀请限制（每日频率上限、坐席总量上限）

### 1.3 功能入口

系统导航路径：**设置 → 客服 → 角色 Tab**

客服管理页面包含「客服」和「角色」两个 Tab，切换到「角色」Tab 即进入角色列表页面。

### 1.4 影响范围

| 页面/模块 | 改动类型 |
|-----------|---------|
| 设置 → 客服 Tab（客服列表） | 坐席统计、邀请入口、待激活状态展示、操作菜单 |
| 设置 → 角色 Tab（角色列表） | 角色增删改查、关联客服数统计 |
| 邀请弹窗 | 邀请每日限制校验、角色分配 |
| 客服账号详情页 | 失焦保存、角色修改 |
| 角色新增/编辑/查看页 | 权限树配置 |
| 删除确认弹窗 | 待激活客服删除、角色删除 |

## 2. 核心概念

### 2.1 角色体系

#### 2.1.1 系统角色

系统预设两个内置角色，均**不可删除**：

| 角色名称 | 权限范围 | 可编辑 | 创建人 | 说明 |
|---------|---------|-------|-------|------|
| 管理员 | 全部权限（含所有菜单权限 + 会话操作权限） | 否 | 系统 | 拥有系统最高权限，不可编辑 |
| 客服 | 部分权限（首页、会话、查看档案、管理访客、管理客户、管理营销、查看客服、管理公共回复、管理个人回复、管理标签） | 是 | 系统 | 基础客服角色，拥有管理客服权限的角色可编辑其权限配置 |

系统角色特征：
- **管理员**：操作菜单中仅显示「查看」，不显示「编辑」和「删除」；角色名称、权限配置均为只读状态
- **客服**：操作菜单中显示「查看」和「编辑」，不显示「删除」；管理员可修改其权限配置，但角色名称不可修改

#### 2.1.2 自定义角色

管理员可创建自定义角色，支持完整的增删改查操作。自定义角色的权限可从权限树中自由选择组合。

### 2.2 角色属性定义

| 属性 | 说明 | 约束 | 备注 |
|-----|------|------|------|
| 角色名称 | 角色的显示名称 | 必填，最大 50 字符 | 不能重复，重复toast提示：重复角色 |
| 权限集 | 该角色拥有的权限项集合 | 可为空（允许创建无权限的角色） | - |
| 关联客服数 | 当前使用该角色的客服人数 | 系统自动统计，只读 | 无数据展示0 |
| 创建人 | 创建该角色的管理员名称 | 系统自动记录 | 系统内置角色创建人为「系统」 |
| 创建时间 | 角色创建的时间 | 系统自动记录，格式：YYYY-MM-DD HH:mm | 系统预设角色创建时间为项目创建时间 |

### 2.3 客服成员状态与生命周期

客服成员包含已激活客服和邀请记录两种形态。

#### 2.3.1 邀请记录状态与生命周期

```
邀请发送 → 待激活（链接永久有效）
              ├── 被邀请人点击链接激活 → 已激活（转为正式客服）
              └── 管理员删除邀请记录 → 已删除（链接立即失效）
```

邀请链接**没有有效期**，在被邀请人激活或管理员删除之前始终有效。

| 状态 | 说明 | 占用坐席 | 可执行操作 |
|-----|------|---------|-----------|
| 待激活 | 邀请已发送，等待被邀请人激活 | 是 | 删除 |

**核心规则**：
- 待激活邀请记录**占用**坐席名额
- 邀请链接**无有效期限制**，不会自动过期
- 删除邀请记录后，对应邀请链接**立即失效**
- 如需重新邀请同一人，需先删除旧邀请记录，再通过邀请弹窗重新发送

## 3. 权限体系

### 3.1 权限层级结构

权限采用三级树形结构：

```
一级菜单
  └── 二级菜单
        └── 功能权限
```

- **一级菜单**：对应系统主导航的功能模块
- **二级菜单**：对应一级菜单下的子功能页面
- **功能权限**：对应二级菜单内的具体操作能力

> 并非所有二级菜单都有功能权限层级，部分二级菜单本身即为最细粒度。

#### 锁定权限

权限树中存在三种锁定机制：

- **锁定一级菜单**（首页、会话）：所有角色默认包含，权限配置树中以勾选 + 置灰状态展示，无法取消
- **锁定二级菜单**（如查看访客、查看客户等）：在权限配置树中以半选（indeterminate）+ 置灰状态展示，不可独立勾选/取消。当所属一级菜单下有任意可配置权限被勾选时，该锁定权限自动包含
- **锁定功能权限**（如查看公共回复、查看个人回复）：在权限配置树中以勾选 + 置灰状态展示，不可独立勾选/取消。当所属二级菜单下有任意可配置权限被勾选时，该锁定功能权限自动包含

### 3.2 完整权限清单

#### 一级菜单：首页

### 3.3 会话操作权限

以下权限不在权限配置树中展示，但属于角色权限数据的一部分：

| 权限名称 | 说明 |
|---------|------|
| 查看会话 | 查看会话列表和消息内容 |
| 转移会话 | 将会话转给其他客服 |
| 添加客服 | 邀请其他客服加入会话 |
| 标记未待处理/移出待处理 | 会话标记为待处理 |
| 关闭会话 | 结束当前会话 |

会话操作权限随角色一起存储和分发，但不通过权限树界面进行勾选配置。

### 3.4 管理员角色的特殊权限逻辑

管理员角色拥有系统中所有权限，包括：
- 权限树中的全部 20 个二级菜单权限和 7 个功能权限
- 全部 4 项会话操作权限

管理员角色的权限在查看时自动填充为全选状态，且不可修改。

### 3.5 系统默认客服角色的权限

客服角色的默认权限集为：
- 菜单权限：查看档案、管理访客、管理客户、管理营销、查看客服（团队 → 客服 → 查看客服）、管理公共回复（快捷回复 → 公共回复）、管理个人回复（快捷回复 → 个人回复）、管理标签

## 4. 角色管理操作

### 4.1 角色列表

角色列表以表格形式展示，包含以下列：

| 列名 | 说明 |
|-----|------|
| 角色名称 | 创建角色时输入的角色名称，超出显示区域用「...」截断，不重复 |
| 关联客服数 | 显示当前使用该角色的客服人数 |
| 创建人 | 系统角色显示「系统」，自定义角色显示创建客服昵称 |
| 创建时间 | 系统角色创建时间为项目创建时间，格式：YYYY-MM-DD HH:mm |
| 操作 | 更多操作按钮（···），点击展开下拉菜单 |

**排序规则**：系统角色排在前面（管理员、客服），自定义角色按创建时间倒序排列。

**操作菜单**：
- 管理员角色：仅显示「查看」
- 客服角色：显示「查看」「编辑」
- 自定义角色：显示「查看」「编辑」「删除」

### 4.2 新增角色

**入口**：角色 Tab 页面右上角的「+ 新增角色」按钮。

**页面标题**：新增角色

**校验规则**：
- 角色名称为必填项，留空时保存按钮置灰不可点击
- 角色名称最大长度 50 字符
- 权限配置可为空（允许创建无权限的角色）

**默认状态**：角色名称为空，所有权限均未勾选（零权限起始）

**保存成功**：返回角色列表，提示「新增成功」

### 4.3 查看角色

**入口**：操作菜单中点击「查看」

**页面标题**：角色详情

**展示内容**：角色名称 + 权限配置（只读，勾选框不可点击）

**页面底部**：不显示操作按钮

### 4.4 编辑角色

**入口**：操作菜单中点击「编辑」（仅可编辑角色可见）。

**页面标题**：编辑角色

**校验规则**：与新增角色一致（名称必填，最大 50 字符）。

**保存成功**：返回角色列表，提示「编辑成功」。编辑角色后已关联客服的权限立即生效，若客服访问无权限的页面，系统应返回 403 页面。

**页面底部**：显示「取消」和「保存」按钮，右对齐

### 4.5 删除角色

**入口**：操作菜单中点击「删除」（仅自定义角色可见）。

**场景一：角色有关联客服**
- 弹窗标题：删除角色
- 弹窗描述：`删除后不可恢复、确认删除？`
- 点击确认删除后提示：`当前角色有 x 个关联客服，暂无法删除`

**场景二：角色无关联客服**
- 弹窗标题：删除角色
- 弹窗描述：`删除后不可恢复、确认删除？`
- 点击「删除」执行删除，提示「删除成功」

## 5. 权限选择交互规则

### 5.1 三级联动选择

| 层级 | 选择行为 |
|-----|---------|
| 一级菜单 | 选中/取消选中该组下所有二级菜单和功能权限 |
| 二级菜单 | 选中/取消选中该二级菜单及其下所有功能权限 |
| 功能权限 | 仅影响自身 + 自动选中所属二级菜单 |

### 5.2 半选状态

- **一级菜单半选**：该一级菜单下有部分（非全部）子项被选中时，显示半选状态
- **二级菜单半选**：该二级菜单拥有功能权限子项，且部分（非全部）被选中时，显示半选状态

### 5.3 反选规则

- **取消一级菜单**：清除该一级菜单下所有二级菜单和功能权限
- **取消二级菜单**：清除该二级菜单及其下所有功能权限
- **取消功能权限**：仅清除该功能权限自身，不影响父级

## 6. 客服管理与邀请

### 6.1 客服列表

客服列表以表格形式展示所有已激活客服和邀请记录：

| 列名 | 已激活客服 | 邀请记录（待激活） | 备注 |
|-----|-----------|----------------| - |
| ID | 客服编号（如 10001） | 显示 `-` | - |
| 姓名 | 头像 + 姓名 | 显示 `-` | - |
| 昵称 | 客服昵称 | 显示 `-` | - |
| 加入时间 | 客服加入项目时间 | 显示 `-` | 原创建时间改名 |
| 角色 | 角色名称 | 邀请时分配的角色 | 新增字段 |
| 邮箱 | 客服邮箱 | 邀请邮箱 | - |
| 会话限制 | 最大并发会话数 | 显示 `-` | - |
| 在线状态 | 在线/离开/离线（带颜色圆点） | 显示 `-` | - |
| 状态 | 启用/停用开关 | 状态标签「待激活」| - |
| 操作 | 更多操作按钮 | 更多操作按钮 | - |

**排序规则**：已激活客服在前，邀请记录在后。已激活客服按照加入时间倒序，未激活客服按邀请时间倒序排列。

### 6.2 坐席名额管理

客服列表页头部显示坐席统计信息：

| 指标 | 计算规则 | 样例 |
|-----|---------|------|
| 坐席总数 | 当前服务包购买的坐席数量 | 10 |
| 使用中 | 已激活客服数 + 待激活邀请数 | 7 |
| 剩余 | 坐席总数 − 使用中 | 3 |

**核心规则**：
- 待激活邀请**占用**坐席名额

### 6.3 邀请客服

#### 6.3.1 入口与弹窗

**弹窗内容**：
1. **角色选择**：新增角色下拉选择器，可选项为系统中所有非管理员角色（客服、高级客服、主管等自定义角色），默认选中「客服」

#### 6.3.2 邀请限制

邀请发送前按以下顺序逐级校验：

| 校验层级 | 限制规则 | 错误提示 |
|---------|---------|---------|
| **坐席名额** | 剩余坐席 ≤ 0 时不可邀请 | - |
| **每日频率** | 每日邀请总次数不超过 **坐席总数 × 2**（如 10 坐席每日上限 20 次） | `今日邀请次数已达 {上限} 上限` |
| **单次邀请** | 单次邀请超出每日评率（邀请弹窗中邀请） | 当超过每日剩余次数时：`今天可邀请 {上限} 次，还剩 {剩余} 次` |

**每日频率说明**：每日限制按自然天计算（UTC+8 0:00 重置）

### 6.4 客服列表操作菜单

#### 6.4.1 待激活邀请的操作菜单

| 操作 | 说明 | 禁用条件 |
|-----|------|---------|
| 删除 | 删除邀请记录，链接立即失效 | 无 |

### 6.5 删除客服/邀请

#### 6.5.1 删除邀请记录（待激活）

**确认弹窗**：
- 弹窗标题：删除客服
- 弹窗描述：`删除后不可恢复，确认删除？`
- 操作按钮：「确认删除」「取消」

**删除成功**：邀请记录从列表移除，对应邀请链接**立即失效**，释放坐席名额，对应角色关联客服数 -1，提示「删除成功，邀请链接已失效」

### 6.6 角色与客服关联

#### 6.6.1 客服编辑时修改角色

在客服账号详情页面，**管理员**可以修改客服的所属角色。角色变更后，该客服的权限即时更新。若客服访问无权限的页面，系统应返回 403 页面。

> 客服查看自己的账号详情时，角色字段为**只读**。

#### 6.6.2 角色的关联客服数统计

角色列表中的「关联客服数」实时显示使用该角色的客服人数（含待激活邀请记录）。

#### 6.6.3 关联约束：有成员的角色不可删除

若角色当前仍有关联客服（含待激活），则不可删除。

### 6.7 账号详情页面

客服账号详情页面新增以下信息：

| 区域 | 字段 | 必填 | 可编辑条件 |
|-----|------|------|-----------|
| | 角色 | — | 管理员可编辑他人角色；查看自己时只读 |

**保存方式**：
- 所有可编辑字段采用**失焦自动保存**机制（焦点离开字段时自动保存）
- 角色下拉框选择后立即触发保存
- 保存成功后提示「保存成功」

### 6.8 客服个人资料中的只读限制

客服通过「个人资料」入口查看自己的账号详情时：
- 角色字段**只读**

## 9. 名词解释

| 名词 | 说明 |
|-----|------|
| 系统角色 | 系统预设的内置角色（管理员、客服），不可删除 |
| 自定义角色 | 管理员手动创建的角色，支持完整的增删改查操作 |
| 权限树 | 以树形结构组织的权限配置界面，包含一级菜单、二级菜单、功能权限三个层级 |
| 半选状态 | 勾选框的中间态，表示其子级部分选中 |
| 关联客服数 | 当前使用该角色的客服人数（含待激活） |
| 待激活 | 客服已被邀请但尚未完成激活，占用坐席名额但不可登录系统 |
| 失焦保存 | 编辑字段后焦点离开时自动触发保存，替代传统的保存按钮 |
| 每日邀请上限 | 每日可发送邀请的最大次数，为坐席总数 × 2 |

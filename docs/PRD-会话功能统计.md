# PRD：会话功能统计

## 1. 概述

### 1.1 功能定位

会话功能统计是客服系统的数据分析模块，用于统计客服人员在会话页面中各项核心操作的使用频次，帮助运营团队了解功能使用情况，优化产品策略。功能分为两个部分：

1. **数据采集端（客服工作台）**：在会话页面的关键操作位置植入埋点，记录每次操作的点击行为
2. **数据展示端（运营后台）**：在运营后台「会话功能统计」页面中，以排行表格和趋势图的形式可视化展示统计数据

### 1.2 功能目标

- 覆盖客服会话页面中 12 项核心操作的使用统计
- 通过功能排行帮助运营判断哪些功能使用频率高、哪些功能可能需要优化推广
- 通过趋势图发现功能使用的变化趋势
- 支持按时间范围筛选，灵活查看不同周期的数据

### 1.3 功能入口

| 入口 | 路径 |
|------|------|
| 数据采集（自动） | 客服工作台 → 会话页面 → 用户执行操作(点击按钮)时自动记录 |
| 数据查看 | 运营后台 → 左侧菜单「数据看板」→「会话功能统计」 |

### 1.4 影响范围

| 页面/模块 | 改动类型 |
|-----------|---------|
| 客服工作台 → 会话页面 | 在 12 个操作位置添加埋点采集 |
| 运营后台 → 会话功能统计页面 | 新增页面，展示排行表格和趋势图 |
| 运营后台 → 侧边菜单 | 「数据看板」菜单下新增「会话功能统计」子项 |

## 2. 核心概念

| 序号 | 功能名称 | 事件标识 | 操作说明 | 备注 |
|------|---------|---------|---------|---------|

| 6 | 远程协助 | remote_assist | 点击消息输入框工具栏的远程协助功能按钮 | 包含会话/聊天 |

| 12 | 移出待处理 | unmark_pending | 将会话从待处理状态移出 | 仅会话 |

### 2.3 埋点采集机制

- **采集时机**：用户执行操作的瞬间同步记录，无延迟

## 3. 数据展示——会话功能统计页面

### 3.1 页面标题

「会话功能统计」

### 3.2 页面布局

页面由上下两个白色卡片区块组成：

1. **功能使用排行**（上方区块）
2. **功能趋势**（下方区块）

### 3.3 日期筛选

位于功能使用排行区块标题行右侧，包含：

| 控件 | 说明 |
|------|------|
| 日期范围选择器 | 双日历面板，格式 YYYY-MM-DD |
| 查询按钮 | 点击后按选定范围刷新数据 |

日期选择器规则：

| 规则 | 说明 |
|------|------|
| 默认值 | 近 7 天（当日往前推 6 天至当日） |
| 快捷按钮 | 面板底部显示 5 个快捷选项：昨天、近7天、近30天、本月、上月 |
| 最大可选范围 | 1 年。超过 1 年前的日期置灰不可选；未来日期不可选 |
| 「昨天」 | 起止日期均为昨天 |
| 「近7天」 | 当日往前推 6 天至当日 |
| 「近30天」 | 当日往前推 29 天至当日 |
| 「本月」 | 当月 1 日至当日 |
| 「上月」 | 上月 1 日至上月最后一天 |

### 3.4 功能使用排行表格

表格列定义：

| 列名 | 说明 |
|------|------|
| 排名 | 数字排名|
| 功能名称 | 功能的中文名称 |
| 事件标识 | 英文下划线格式的事件唯一标识 |
| 使用次数 | 所选时间范围内的累计点击次数，支持排序 |
| 占比 | 该功能使用次数占总使用次数的百分比，附带进度条可视化 |

表格规则：
- 不分页，一次性展示全部 12 条数据
- 「使用次数」列支持点击列头排序（升序/降序切换）

### 3.5 功能趋势折线图

展示使用量排名前 5 的功能在近 7 天的趋势变化。

| 配置项 | 说明 |
|--------|------|
| 标题 | 「功能趋势（近 7 天 · Top 5）」 |
| 图例位置 | 标题行右侧，水平排列 |
| 展示维度 | 固定为排行前 5 的功能 |
| X 轴 | 日期，格式 YYYY-MM-DD |
| Y 轴 | 使用次数，带刻度标签和虚线网格 |
| 数据点 | 各折线节点显示圆形数据点 |

当前 Top 5 功能及对应颜色：

| 功能 | 折线颜色 |
|------|---------|
| 快捷回复 | 蓝色 |
| 帮我润色 | 粉红色 |
| 丰富内容 | 橙色 |
| 精简内容 | 紫色 |
| Copilot推荐回复 | 绿色 |

## 4. 操作流程

### 4.1 数据采集流程

1. 客服人员在工作台会话页面中进行日常操作
2. 当执行统计范围内的操作时（如点击快捷回复按钮），系统自动记录一次点击事件

## 7. 业务规则汇总表

| # | 规则 | 触发条件 | 预期行为 |
|---|------|---------|---------|
| 1 | 仅统计点击类操作 | 用户点击功能按钮 | 记录一次计数 |
| 2 | 埋点不影响原操作 | 执行任何被统计的操作 | 先记录埋点，再执行原操作逻辑，两者互不影响 |
| 3 | 默认时间范围 | 进入页面 | 日期选择器默认选中近 7 天 |
| 4 | 最大可选范围 | 选择日期 | 不可选超过 1 年前的日期，不可选未来日期 |
| 5 | 排行固定展示全部 | 进入页面 | 表格不分页，显示全部 12 条功能项 |
| 6 | 趋势图固定 Top 5 | 进入页面 | 趋势图仅展示使用量前 5 的功能折线 |
| 7 | 排名不随排序变化 | 点击列头排序 | 排名列数字始终按原始使用次数降序排列 |

## 8. 名词解释

| 名词 | 说明 |
|------|------|
| 埋点 | 在用户操作触发点植入数据采集代码，记录用户行为的技术手段 |
| 事件标识 | 每个被统计功能的唯一英文标识符，用于数据上报和存储 |
| 功能排行 | 按使用次数从高到低排列的功能列表 |
| 趋势图 | 以时间为 X 轴、使用次数为 Y 轴的折线图，展示功能使用量的变化趋势 |
| 快捷按钮 | 日期选择器面板底部的预设时间范围按钮，点击可快速选定常用时间段 |

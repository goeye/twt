## 1. 产品概述

### 1.1 产品定位

TWT AI Agent 是集成在海外客服 SaaS 平台中的智能客服机器人。它作为人工客服的补充，自动接待访客、回答常见问题，在无法处理时无缝转接人工，并通过 Copilot 辅助人工客服提升效率。

### 1.2 核心价值

| 价值 | 说明 |
|------|------|
| **降本增效** | AI 自动处理重复性咨询，减少人工工作量 |
| **7×24 全天候** | 离线模式下 AI 自动接管，不漏接任何访客 |
| **无缝人机协作** | 人工可随时接管 AI 会话，AI 可辅助人工回复 |
| **智能语言适配** | 自动检测访客语言，支持 14 种语言 |
| **情绪感知** | 检测访客情绪，高负面情绪自动提升转人工优先级 |
| **主动触达** | 通过 Proactive Campaign 主动营销访客 |

### 1.3 目标用户

- **客服管理员**：配置 AI Agent、管理知识库、设置营销活动
- **一线客服**：使用 Copilot 辅助、接管 AI 会话、查看 AI 会话摘要
- **终端访客**：与 AI Agent 对话、接收主动营销消息

## 3. AI Agent 核心功能

### 3.1 基本设置

| 配置项 | 说明 |
|--------|------|
| **启用开关** | 全局开启/关闭 AI Agent |
| **回复模式** | `始终开启`：所有会话先由 AI 接待；`仅离线`：仅当无人工在线时 AI 接管 |
| **AI 标签显示** | 消息中是否标注 AI 身份 |
| **Bot 形象** | 自定义头像（支持裁切上传）、昵称（必填≤64字符）、业务简介（≤200字符） |
| **回复语气** | 友好 / 专业 / 幽默 / 简洁，四选一 |
| **默认语言** | 14种语言可选，作为语言检测的最终兜底 |

### 3.2 会话策略

| 配置项 | 说明 |
|--------|------|
| **访客不活跃超时** | 1–1440 分钟，超时后自动关闭会话 |
| **回答模式** | `严格模式`：仅基于知识库内容回答；`创意模式`：允许 AI 自主推理 |
| **转人工话术** | 访客请求转人工时展示的消息（必填≤2000字符） |
| **离线消息** | 所有客服离线时的自动回复（必填≤2000字符） |
| **无法处理消息** | 遇到图片/文件等无法处理内容时的回复（必填≤2000字符） |

### 3.3 AI 会话接入决策流程

```
访客发起对话
  → AI Agent 是否启用?
    → 否 → 进入人工队列
    → 是 → 回复模式?
      → 始终开启 → AI 接管会话
      → 仅离线 → 有客服在线?
        → 是 → 进入人工队列
        → 否 → AI 接管会话
```

### 3.4 AI 消息处理管线

AI 接管会话后，每条访客消息按以下管线处理：

```
访客消息进入
  │
  │  ┌─────────────────────────────────────────────────┐
  │  │  系统层预处理（代码逻辑，LLM 之前）                │
  │  │                                                   │
  │  │  Step 0: 消息类型路由                              │
  │  │    ├─ type=text   → 进入 LLM 管线                 │
  │  │    └─ type=image/voice/file/video                 │
  │  │         ├─ 无上下文（首条消息）→ 回复预设话术        │
  │  │         └─ 有上下文 → 转占位描述 → 进入 LLM 管线    │
  │  └─────────────────────────────────────────────────┘
  │
  ▼
  Step 1: 语言检测（Prompt #1）
  │  └─ 无法检测 → 兜底：历史语言 → 浏览器 → IP → 默认语言
  │
  ▼
  Step 2: 意图分类 + 情绪检测（Prompt #2）
  │  ├─ 转人工意图      → 回复转人工话术 → 生成会话摘要 → 人工队列
  │  ├─ 无法处理内容    → 回复预设话术
  │  ├─ 闲聊寒暄        → Step 4: 回复生成（无需检索）
  │  └─ 业务咨询        → Step 3: 查询改写 → 知识库检索 → Step 4
  │
  ▼
  Step 3: 查询改写（Prompt #3，仅业务咨询触发）
  │  └─ 指代消解 + 多轮合并 + 关键词提取 → 结构化查询 → 知识库检索
  │
  ▼
  Step 4: 回复生成（Prompt #4）
  │  └─ 含内置质检约束 + 多模态消息处理逻辑
  │
  ▼
  发送给访客
  │
  └─ Step 5: 多轮上下文压缩（每 N 轮触发，Prompt #6）
```

### 3.5 语言检测与兜底策略

AI Agent 支持自动检测访客语言，用检测结果决定回复语言。

#### 正常流程

访客发送可识别语言的文本 → LLM 检测语言 → 使用检测结果。

#### 兜底策略（无法检测时）

当访客发送纯数字（`666`）、纯 emoji（`😂👍`）、代码片段等无法判断语言的内容时，按以下优先级兜底：

| 优先级 | 来源 | 说明 |
|--------|------|------|
| 1 | 历史语言 | 该访客历史会话中已识别的语言 |
| 2 | 浏览器语言 | 访客浏览器 `Accept-Language` 首选语言 |
| 3 | IP 地区 | 访客 IP 地理位置推断的语言 |
| 4 | 默认语言 | 管理员配置的 `defaultLanguage` |

语言是**会话级别持续追踪**的——后续访客发了可识别语言的文本时，更新该会话的语言标记。

### 3.6 意图分类 + 情绪检测

系统采用 **关键词快速预判 + LLM 深度分类** 双层机制。

#### 第一层：关键词快速预判（系统层代码）

| 类型 | 关键词 | 命中后行为 |
|------|--------|-----------|
| 转人工意图 | `人工` `转人工` `人工客服` `human` `agent` `person` | 直接回复转人工话术，不走 LLM |
| 无法处理内容 | `图片` `文件` `附件` `image` `photo` `picture` `file` `attachment` `pdf` `doc` | 回复预设话术（仅处理访客用文字描述要发图片/文件的场景） |

#### 第二层：LLM 意图分类（关键词未命中时触发）

| 意图类别 | 说明 | 后续路由 |
|---------|------|---------|
| `transfer_to_human` | 访客希望转接人工（含隐晦表达如"找你们负责人"） | 转人工话术 → 会话摘要 → 人工队列 |
| `unsupported_content` | 访客用文字描述要发送 AI 无法处理的内容 | 回复预设话术 |
| `chit_chat` | 闲聊、寒暄、打招呼、告别 | 回复生成（无需知识库检索） |
| `business_inquiry` | 业务咨询（默认类别） | 查询改写 → 知识库检索 → 回复生成 |

#### 情绪检测

| 情绪等级 | 说明 | 系统行为 |
|---------|------|---------|
| `positive` | 积极、满意、感谢 | 正常流程 |
| `neutral` | 平静、正常咨询 | 正常流程 |
| `negative` | 不满、抱怨、焦急 | 正常流程，标记会话 |
| `angry` | 愤怒、强烈不满、威胁投诉 | **自动提升转人工优先级** |

### 3.7 非文本消息处理

当访客发送图片、语音、文件、视频等非文本消息时，由系统层按消息类型路由（而非关键词检测）。

#### 处理策略

```
收到非文本消息
  │
  ├─ 该会话是否有之前的文本上下文？
  │
  ├─ 无上下文（非文本消息是会话第一条 or 之前无有效文本对话）
  │     → 直接回复预设话术（unsupportedQuestionMessage）
  │     → 将消息转为占位描述注入上下文
  │
  └─ 有上下文（之前有过正常文本对话）
        → 将非文本消息转为占位描述，注入上下文
        → 走 LLM 回复生成管线
        → LLM 会：延续之前话题 + 自然告知无法查看 + 引导访客用文字描述
```

#### 占位描述格式

| 消息类型 | 占位描述文本 |
|---------|-------------|
| image | `[访客发送了一张图片，AI 无法查看图片内容]` |
| voice | `[访客发送了一条语音消息，AI 无法收听语音内容]` |
| file | `[访客发送了一个文件：{fileName}，AI 无法读取文件内容]` |
| video | `[访客发送了一段视频，AI 无法查看视频内容]` |

#### 有上下文时的回复示例

```
访客：我的订单 #12345 一直没收到货
访客：已经等了 7 天了
访客：[发送了一张物流截图]

AI 回复（LLM 生成）：
  关于您的订单 #12345 的物流问题，我理解您的着急。
  不过我暂时无法查看您发送的图片，麻烦您用文字描述
  一下图片中显示的物流状态，我来帮您进一步核实。
```

### 3.8 AI 会话生命周期

```
1. AI 接管 → 发送自我介绍
2. 等待访客消息
3. 收到消息 → 系统层预处理（消息类型路由）
   → 进入 AI 消息处理管线（语言检测 → 意图分类 → 查询改写 → 回复生成）
   → 发送 AI 回复 → 回到等待
4. 触发转人工 → 生成会话摘要 → 人工队列
5. 客服主动接管 → 生成会话摘要 → 人工会话
6. 访客不活跃超时 → 自动关闭会话
7. 对话超过 N 轮 → 触发上下文压缩
```

### 3.9 多轮上下文压缩

长对话会累积大量消息，超出 LLM token 限制。系统在对话轮数超过阈值（建议 N=10）时自动触发压缩：

- **保留**：访客的所有问题和诉求、具体实体（订单号/产品名/金额）、AI 给出的关键答复、未解决的问题、情绪变化
- **丢弃**：重复的寒暄、AI 自我介绍、重复表达的相同问题、系统消息

压缩后的摘要替代原始消息历史，作为后续 LLM 调用的上下文输入。

### 3.10 会话摘要

在以下场景自动生成摘要：

| 触发场景 | 用途 |
|---------|------|
| 访客请求转人工 | 摘要展示给即将接手的人工客服 |
| 客服主动接管 AI 会话 | 摘要展示给接管的客服 |
| 会话结束/关闭 | 摘要存入会话记录，供后续查阅 |

摘要内容包含：访客核心问题、AI 已做的处理、待人工处理的事项、访客情绪状态、转接原因。

### 3.11 人工接管流程

1. 访客发送消息，AI 自动回复
2. 人工客服在"AI Agent"队列中看到此会话
3. 客服点击"接管会话"
4. 系统自动生成会话摘要，展示给客服
5. 系统：会话从 AI 队列移至人工队列，插入系统消息"客服已接管"
6. 会话出现在"我的会话"，客服发送人工回复

## 4. Copilot 辅助功能

Copilot 是面向人工客服的 AI 辅助工具集，每个功能独立开关控制，分别对应独立的 Prompt。

| 功能 | 说明 | 对应 Prompt |
|------|------|------------|
| **自动推荐回复** | 根据对话上下文 + 知识库，生成 2-3 条风格各异的候选回复 | Prompt #8 |
| **聊天翻译** | 将访客消息翻译为客服使用的语言，保留客服领域术语 | Prompt #10（chat-translation 模式） |
| **并排翻译** | 客服输入回复时，实时翻译为访客语言供确认 | Prompt #10（side-translation 模式） |
| **文本润色** | 纠正语法拼写、优化表达、保留原意、不添加额外信息 | Prompt #9 |

### 辅助流程

```
收到访客消息
  → 聊天翻译已开启? → 是 → Prompt #10 翻译为客服语言 → 显示翻译结果
                     → 否 → 显示原文
  → 自动推荐已开启? → 是 → Prompt #8 生成 2-3 条候选回复 → 展示推荐
                     → 否 → 客服手动输入
  → 客服编辑/采纳回复
  → 文本润色已开启? → 是 → Prompt #9 润色优化 → 客服确认
  → 并排翻译已开启? → 是 → Prompt #10 翻译为访客语言 → 显示预览
  → 发送回复
```

## 5. 主动营销 (Proactive Campaign)

### 5.1 功能概述

管理员可创建主动营销活动（Campaign），在访客满足指定触发条件时，自动向其展示营销卡片，引导访客执行特定操作。

### 5.2 Campaign 配置项

| 配置 | 说明 |
|------|------|
| **活动名称** | 内部管理标识 |
| **启用状态** | 开启/关闭 |
| **头图** | 卡片顶部展示图片 |
| **标题** | ≤50字符 |
| **描述** | 必填，≤200字符 |
| **按钮** | 最多 5 个，每个按钮有标签(≤20字符)、样式(实心/半透明/描边)、动作类型 |

### 5.3 按钮动作类型

| 动作类型 | 说明 |
|---------|------|
| **打开链接** (link) | 跳转到指定 URL |
| **打开聊天** (open-chat) | 直接打开聊天窗口开始对话 |
| **触发意图** (trigger-intent) | 触发预设的对话流/意图 |

### 5.4 触发规则

| 规则 | 选项 | 说明 |
|------|------|------|
| **受众** | 所有访客 / 新访客 / 回访访客 | 决定展示给哪些访客 |
| **频率** | 仅一次 / 多次 | 同一访客是否重复展示 |
| **触发时机** | 页面加载时 / 滚动时 / 即将离开时 | 何时触发展示 |
| **延迟时间** | 1–600 秒 | 满足条件后延迟多久展示 |

### 5.5 Campaign 运营流程

```
管理员创建 Campaign
  → 使用模板? → 是 → 选择模板预填内容
               → 否 → 空白创建
  → 编辑 Campaign (基本信息 + 按钮 + 触发规则)
  → 实时预览效果
  → 保存并上线

访客满足触发条件
  → 展示 Campaign 卡片
  → 访客操作?
    → 点击按钮 → 按钮类型?
      → link → 跳转外部链接
      → open-chat → 打开聊天窗口 → 进入 AI Agent 会话接入流程
      → trigger-intent → 进入对话流
    → 忽略/关闭 → 结束
```

### 5.6 Campaign 触发决策流程

```
访客进入页面
  → 有已启用的 Campaign? → 否 → 等待访客主动操作
  → 匹配受众? (all/new/returning) → 不匹配 → 等待
  → 频率检查 (once: 是否已展示过?) → 已展示且仅一次 → 等待
  → 可展示 → 等待触发时机 (on-entry/on-scroll/on-exit)
  → 延迟 N 秒
  → 展示 Campaign
```

## 7. 全链路核心流程

```
访客进入网站
  → 有活跃 Campaign? → 是 → 展示 Campaign 卡片
                      → 否 → 访客主动发起对话

  访客点击 Campaign?
    → 打开聊天 → 访客发起对话
    → 跳转链接 → 跳转外部页面
    → 触发意图 → 访客发起对话
    → 忽略 → 访客继续浏览

  访客发起对话
    → AI Agent 启用? → 否 → 直接进入人工队列
    → 是 → 回复模式?
      → 始终开启 → AI Agent 接管
      → 仅离线 → 有客服在线?
        → 是 → 人工队列
        → 否 → AI Agent 接管

  AI Agent 接管
    → 发送自我介绍
    → 等待访客消息
    → 收到消息 → 系统层预处理
      → 消息类型?
        → 非文本 → 有上下文?
          → 无 → 回复预设话术（unsupportedQuestionMessage）→ 继续等待
          → 有 → 转占位描述 → 进入 LLM 管线
        → 文本 → 关键词快速预判
          → 命中转人工 → 回复转人工话术 → 会话摘要 → 人工队列
          → 命中无法处理 → 回复预设话术 → 继续等待
          → 未命中 → 进入 LLM 管线

    LLM 消息处理管线
      → Step 1: 语言检测 → 确定回复语言
      → Step 2: 意图分类 + 情绪检测
        → transfer_to_human → 回复转人工话术 → 会话摘要 → 人工队列
        → unsupported_content → 回复预设话术 → 继续等待
        → chit_chat → Step 4: 回复生成（无需检索）
        → business_inquiry → Step 3: 查询改写 → 知识库检索 → Step 4
      → Step 3: 查询改写（指代消解 + 多轮合并 + 关键词提取）
      → Step 4: 回复生成（含内置质检 + 多模态处理）
      → 发送 AI 回复 → 继续等待
      → 对话超过 N 轮 → 上下文压缩

    情绪升级处理
      → 情绪=angry → 自动提升转人工优先级 → 标记会话

  人工队列 → 客服处理会话
    → 查看 AI 生成的会话摘要（快速了解前情）
    → 使用 Copilot?
      → 推荐回复（Prompt #8）→ 采纳/修改推荐
      → 聊天翻译（Prompt #10）→ 查看翻译后的访客消息
      → 并排翻译（Prompt #10）→ 预览回复的目标语言版本
      → 文本润色（Prompt #9）→ 优化回复文本
      → 直接回复 → 手动编写
    → 发送回复给访客

  客服主动接管 AI 会话?
    → 是 → 生成会话摘要 → 系统消息: 客服已接管 → 客服处理会话

  访客不活跃超时? → 自动关闭会话
```

## 8. AI 消息处理管线详细说明

> 各 Prompt 的完整模板见 `docs/ai-agent-prompt.md`。

### 8.1 Prompt 清单

| # | Prompt | 触发时机 | 优先级 | 输入 | 输出 |
|---|--------|---------|--------|------|------|
| 1 | 语言检测 | 每条访客文本消息 | P0 | 访客消息文本 | 语言代码 + 置信度 + fallback 标记 |
| 2 | 意图分类 + 情绪检测 | 每条访客文本消息 | P0 | 访客消息 + 上下文 | 意图类别 + 情绪等级 + 置信度 |
| 3 | 查询改写 | 意图=business_inquiry | P0 | 访客消息 + 上下文 + 语言 | 结构化查询 + 关键词列表 |
| 4 | 回复生成 | 需要 AI 回复时 | P0 | 上下文 + 知识库结果 + 情绪 + 配置 | 最终回复文本 |
| 5 | 回复质检 | 回复发出前 | P2(v2) | 生成的回复 + 知识库内容 | 检查结果 + 修改建议 |
| 6 | 多轮上下文压缩 | 对话超过 N 轮 | P1 | 完整对话历史 | 压缩摘要 + 关键实体 |
| 7 | 会话摘要 | 转人工 / 会话结束 | P1 | 完整对话记录 | 结构化摘要 |
| 8 | Copilot 推荐回复 | 人工会话收到消息 | P1 | 上下文 + 知识库 + 访客消息 | 2-3 条候选回复 |
| 9 | Copilot 文本润色 | 客服点击润色 | P1 | 客服草稿 + 上下文 | 润色后文本 + 修改说明 |
| 10 | Copilot 翻译 | 客服点击翻译 | P1 | 源文本 + 目标语言 | 翻译结果 |

### 8.2 管线调用时序

```
          系统层                   LLM 管线
          ──────                   ────────
收到消息 ──→ 消息类型路由
              │
              ├─ 非文本 + 无上下文 → 回复预设话术（不调 LLM）
              │
              ├─ 非文本 + 有上下文 → 转占位描述 ──→ Prompt#1 → Prompt#2 → Prompt#4
              │
              ├─ 文本 + 关键词命中 → 直接路由（不调 LLM）
              │
              └─ 文本 + 关键词未命中 ──→ Prompt#1 → Prompt#2
                                          │
                                          ├─ transfer_to_human → 预设话术 + Prompt#7
                                          ├─ unsupported_content → 预设话术
                                          ├─ chit_chat → Prompt#4
                                          └─ business_inquiry → Prompt#3 → 检索 → Prompt#4
```

## 11. 业务规则汇总

| 规则 | 描述 |
|------|------|
| AI 会话队列隔离 | AI 会话独立于人工队列，标识为 `ai-agent-queue` |
| 接管不可逆 | 人工接管后会话永久转为人工模式，不可退回 AI |
| 双层意图检测 | 关键词快速预判（系统层）+ LLM 深度分类（Prompt #2），互为补充 |
| 转人工关键词 | 中文：人工/转人工/人工客服；英文：human/agent/person |
| 无法处理关键词 | 图片/文件/附件/image/photo/picture/file/attachment/pdf/doc（仅处理文字描述场景） |
| 非文本消息路由 | 按消息 type 在系统层路由，不依赖关键词检测 |
| 有上下文的非文本消息 | 转占位描述后走 LLM 管线，生成延续话题 + 引导描述的智能回复 |
| 无上下文的非文本消息 | 直接回复预设话术 `unsupportedQuestionMessage` |
| 语言持续追踪 | 会话级别持续追踪语言，无法检测时走兜底策略链 |
| 情绪升级机制 | angry 情绪自动提升转人工优先级 |
| 转人工自动摘要 | 转人工时自动生成会话摘要，展示给接手客服 |
| 上下文压缩 | 对话超过 N 轮（建议 10）自动压缩，控制 token 消耗 |
| 表单脏检测 | 离开配置页前检查是否有未保存修改，提示确认 |
| 头像裁切 | 固定裁切为 240×240，支持拖拽调整位置 |
| Campaign 按钮上限 | 单个 Campaign 最多 5 个按钮，支持拖拽排序 |
| 存储方式 | 当前使用 localStorage 持久化（`twt:ai-agent-settings`） |

# 人工客服主动邀约需求文档

## 一、 需求背景与目标

*   **核心目标**：在现有访客小组件基础上增加「人工客服主动邀约」能力。当客服主动向访客发送消息时，通过气泡通知强化展示，打破"等客上门"的被动局面，提升客服主动触达效率和会话转化率。

*   **业务定位**：强化客服的主动服务能力。通过高优先级的真人邀约气泡机制，确保客服的主动沟通能够及时、醒目地传达给访客，提升访客响应率。

## 二、 人工客服主动邀约功能

### 2.1 触发时机

客服在 Web 端主动向访客**发送会话消息**时（访客/客户列表、群发消息），且访客未展开聊天面板。

**注意**：单聊/群聊暂不触发提醒。

### 2.2 展示形态

在访客端小组件图标上方，弹出带有**消息文本**的通知小气泡。气泡右上角常驻 `[X]` 关闭按钮。

**消息类型限制**：
- 只展示文本类型的消息
- 系统消息、图片、视频、文件等非文本消息不触发气泡展示

**文本展示规则**：
- 消息文本最多展示 2 行
- 超出 2 行的文本自动截断，末尾显示省略号(...)

### 2.3 交互规则

1.  **叠加与顶替**：最多纵向堆叠展示 **3 条**最新消息。收到第 4 条时，最旧的一条自动消失，整体上移。

2.  **消失与兜底机制**：气泡弹出后，不会主动消失。只能访客手动点 X 关闭气泡，客服原生图标上显示【未读会话数量红点】兜底。

3.  **点击动作**：访客点击任一消息气泡，立即展开完整的聊天主界面。同时关闭通知气泡

4.  **会话结束不影响**：访客点击气泡进入会话后，即使会话被客服关闭或结束，已展示的其他气泡仍然保持展示状态，不会因会话结束而自动消失。

### 2.4 多客服主动邀约处理

**场景描述**：访客端当前已经接收并展示了客服 A 的主动邀约气泡，此时客服 B 也向该访客发送了主动邀约消息。

**气泡展示逻辑（先入为主）**：

1.  前端增加气泡展示状态判断。

2.  当已有气泡处于展示状态时，对于新到达的其他客服邀约消息，前端在**气泡展示层面直接拦截并忽略**，不更新气泡内容。只在客服图标显示未读红点。

3.  直到当前气泡的生命周期结束（如访客点击查看、手动关闭），才允许展示下一条新到达的邀约气泡。

### 2.5 异常场景处理

1.  **网络断开场景**：访客网络断开期间，客服发送的邀约消息暂存在服务端。访客重新连接后，按时间顺序依次展示气泡（遵循最多 3 条堆叠规则）。

2.  **页面刷新/重新加载**：访客刷新页面后，未读/未关闭的邀约消息重新展示气泡（遵循最多 3 条堆叠规则）。已手动关闭的气泡不再重复展示。

3.  **隐藏小组件**：访客端小组件被隐藏后消息气泡不再展示

4.  **消息撤回场景**：客服撤回已发送的邀约消息后，对应的气泡立即从访客端消失。如果该消息是当前展示的 3 条气泡之一，其他气泡自动上移填补空位。

## 三、 防冲突与全局免打扰

为保障访客体验，防止过度打扰，系统需在前端判定弹窗资格。

### 3.1 与主动营销功能的冲突处理

**背景说明**：主动营销功能已上线，支持通过预设规则自动弹出营销气泡。人工客服主动邀约需要与已上线的营销功能进行冲突处理。

#### 3.1.1 人工消息绝对优先权

"真实客服交互"的优先级永远大于"系统自动营销"。

1.  **强制替换**：若当前屏幕正展示着「自动化营销气泡」，此时突然收到客服的「人工主动邀约消息」，前端需**立刻销毁营销气泡**，替换弹出人工消息气泡。

2.  **全局免疫**：只要客服主动发送过消息，在当前页面生命周期内，该访客对所有的自动化营销规则免疫，不再触发任何营销弹窗。

## 四、 小部件配置项

### 4.1 功能开关

在小部件自定义页面「常规」tab 的「排队提醒」配置项下方，新增「主动会话提醒」配置项。

**配置项说明：**
- 名称：主动会话提醒
- 描述：客服主动创建会话时，在聊天图标上方展示消息气泡
- 默认状态：关闭
- 开关标签：开启主动会话提醒

### 4.2 预览联动

展开「主动会话提醒」配置项时，右侧预览区自动切换为最小化状态（显示聊天图标）。

开启开关后，预览区在聊天图标上方展示模拟消息气泡，效果如下：
- 关闭按钮显示在气泡上方右侧
- 气泡最多展示 2 行文字，超出省略
- 示例文案：「您好，有什么可以帮您？」

## 五、 人工邀约与主动营销

### 核心逻辑节点解析

1.  **人工消息的绝对优先级（强制替换）**： 无论营销气泡展示得多好，只要"真实的客服"发话了，系统会**立刻销毁营销气泡**，给人工消息让路。并且在此后当前页面的访问周期内，该访客不再受任何系统自动营销打扰。

2.  **多客服邀约防冲突（先入为主）**： 处理了多个客服同时抢夺一个访客注意力的问题。一旦客服 A 的气泡已经展示，客服 B 的消息只能以"小红点"形式低调提示，必须等访客处理完当前气泡（点击或关闭），才能进入下一轮展示。

# PRD-AI Agent 机器人

## 一、文档概述

### 1.1 文档目的

本文档针对 TWT 客服系统中的 **AI Agent 机器人** 功能进行完整的产品需求描述。涵盖 AI Agent 自身的配置与运行规则，以及其与首页概览、会话模块、档案模块、报表模块、角色模块之间的关联关系及业务影响。同时包含访客端（Web）的 AI Agent 交互需求。

### 1.2 适用对象

产品经理、研发工程师、测试工程师、项目管理人员。

### 1.3 术语定义

| 术语 | 定义 |
| --- | --- |
| AI Agent | 系统内置的智能机器人客服，基于知识库自动回复访客咨询 |
| 人工客服 | 在系统中以真人身份登录并提供服务的客服人员 |
| 接管 | 人工客服从 AI Agent 手中取得某会话的控制权，后续由人工负责回复 |
| 转人工 | AI Agent 主动发起或访客请求，将会话流转给人工客服 |
| 会话 | 访客与客服（含 AI Agent 和人工客服）之间的一次完整对话过程 |
| 知识库 | 供 AI Agent 参考和回复的文档集合，包含文档知识和常见问题 |

## 二、AI Agent 功能定义

### 2.1 功能总述

AI Agent 是一个可配置的智能机器人客服，具备以下核心能力：

1.  **自动接入**：在访客发送第一条消息后，根据配置的规则自动接入并回复访客。
    
2.  **智能回复**：基于关联的知识库内容，按照设定的语气、语言和回复模式，自动生成回复。
    
3.  **兜底处理**：对于无法回答的问题，发送预设的兜底回复。
    
4.  **转接人工**：在出发关键词时，将会话转接给人工客服。
    
5.  **访客跟进**：在访客不活跃时，自动发送跟进消息。
    
6.  **自动关闭**：在访客持续不活跃超过设定时长后，自动关闭会话。
    

### 2.2 AI Agent 开关状态

#### 2.2.1 默认状态

*   AI Agent 功能**默认关闭**。
    
*   新项目创建后，AI Agent 处于关闭状态，需手动开启。
    

#### 2.2.2 开启条件

*   点击"开启 AI Agent"按钮即可直接开启。
    

#### 2.2.3 开启后的业务影响

*   AI Agent 开始按照配置的"回复时机"和"受众范围"自动接入新会话。
    

#### 2.2.4 暂停后的业务影响

*   AI Agent 停止接入新会话。已在 AI Agent 队列中的会话不受影响，继续保持在队列中由 AI Agent 回复直到被人工接管或自动关闭。
    
*   首页"AI Agent 概览"、会话"AI Agent"分组、报表"AI Agent"页面保留历史数据展示，不再有新数据增长。

## 三、AI Agent 配置管理

### 3.1 配置入口

在系统左侧主导航中，点击"AI Agent"进入 AI Agent 配置中心。

### 3.2 AI Agent 核心配置

AI Agent 配置页面包含两个 Tab：**部署**和**设置**。

#### 3.2.1 部署配置（生命周期流程）

部署配置按照会话生命周期分为四个阶段，每个阶段包含若干可展开的配置卡片。

**卡片交互规则：**

*   每张配置卡片支持展开/收起操作。
    
*   **收起状态**：卡片标题后方显示当前设置的摘要预览（如"严格模式"、"已关联 5 篇知识库文档"）。
    
*   **展开状态**：隐藏摘要预览，标题下方显示该配置项的功能描述文字。

**阶段一：当访客发送第一条消息（Entry）**

| 配置项 | 说明 | 选项 | 默认值 |
| --- | --- | --- | --- |
| 受众群体 | AI Agent 的回复对象 | 全部访客 / 仅访客 / 仅客户 | 全部访客 |
| 回复时机 | AI Agent 何时介入回复 | 始终回复 / 仅客服不在线时 | 始终回复 |
| AI Agent 标签 | 在消息气泡中显示标签 | 显示 / 不显示 | 显示 |

业务规则：

*   "全部"：AI Agent 对所有访客和客户的首条消息均自动接入回复。
    
*   "仅访客"：仅对未关联客户资料的访客自动接入。
    
*   "仅客户"：仅对已关联客户资料的访客自动接入。
    
*   "始终由 AI Agent 回复"：无论人工客服是否在线，AI Agent 均自动接入。
    
*   "仅客服不在线时"：仅当所有人工客服均不在线时，AI Agent 才自动接入。
    

**阶段二：当 AI Agent 回复访客（Answering）**

| 配置项 | 说明 | 选项 | 默认值 |
| --- | --- | --- | --- |
| 回复模式 | AI Agent 的回复策略 | 严格模式 — 仅使用知识库内容回复 / 发散模式 — 允许适度推理 | 严格模式 |
| 关联知识库 | AI Agent 参考的知识库内容 | 管理知识库 | — |

业务规则：

*   **严格模式**：AI Agent 仅基于知识库中已有的内容进行回复。若知识库中无匹配内容，AI Agent 根据提示词定义内容进行回复。适用于对回复准确性要求较高的场景。
    
*   **发散模式**：AI Agent 在知识库内容的基础上，可结合上下文和通用知识进行适度扩展回复。适用于需要更灵活互动的场景。
    
*   **关联知识库横幅提醒**：当选择严格模式回复方式是，若关联的知识库文档数量为 0 时，在"关联知识库"设置卡片内展示横幅提醒引导用户丰富知识库内容。当知识库文档数量大于 0 个或选择发送模式时不显示横幅提醒。横幅提示文案：「当前未添加知识库，为获得更精准的回答，请添加相关内容」。操作按钮：「去添加」。点击「去添加」按钮跳转到文档知识页面。
    

**阶段三：当 AI Agent 无法解答时（Fallback）**

| 配置项 | 说明 | 默认值 |
| --- | --- | --- |
| 兜底回复 | AI Agent 无法回答时发送的消息 | 抱歉，我暂时无法处理这类内容，请用文字描述或换个问题。 |
| 转接人工 | 当访客要求转人工时，是否允许转接到人工客服 | **开启** |
| 转接提示语 | 转接人工时向访客展示的提示 | "正在为你转接人工客服，请稍候" |
| 客服离线提示 | 转接时所有人工客服不在线的提示 | "当前客服不在线。你可以先留下问题或联系方式，我们会尽快与你联系。" |
| 不转接人工提示 | 转接人工关闭时，AI Agent 无法解决时的提示 | "当前客服不在线。你可以先留下问题或联系方式，我们会尽快与你联系。" |

业务规则：

*   兜底回复不能为空。若管理员清空该字段，输入框失焦后 Toast 提示"输入内容不能为空"，并回复是输入框内容。
    
*   **转接人工开关开启时**：当 AI Agent 判断需要人工介入时，自动发送转接提示语并将会话转入人工客服队列。
    
*   **转接人工开关关闭时**：AI Agent 不会主动发起转接，当遇到访客要求转接人工时，向访客发送"不转接人工提示"文案。
    
*   转接时，若有在线的人工客服，会话进入"在线会话"的相应队列；若所有人工客服均不在线，向访客发送客服离线提示，会话进入排队状态等待人工上线。

*   不允许转接人工客服时，显示「不转接提示语输入框」。允许转接人工客服时，显示「转接提示语」和「客服离线提示」输入框。
    

**阶段四：访客不活跃（Idle）**

| 配置项 | 说明 | 默认值 |
| --- | --- | --- |
| 主动跟进 | 当访客 5 分钟未回复时，AI Agent 自动发送一条跟进消息 | **开启** |
| 跟进消息内容 | 跟进消息的文案 | 你好，请问还有什么可以帮你的？如果没有其他问题，会话将在稍后自动关闭。 |
| 关闭时间 | 访客持续不活跃后自动关闭会话的等待时长 | 0时 10分 0秒 |

业务规则：

*   跟进消息开关开启后，当访客最后一条消息发送超过 5 分钟无新消息时，AI Agent 自动发送跟进消息。
    
*   跟进消息仅发送一次，不重复发送。
    
*   自动关闭时间从访客最后一次发送消息或最后一次交互开始计时。
    
*   自动关闭后的会话归入"已结束"状态，记录到档案中。
    

#### 3.2.2 设置配置（基本信息 + 回复风格）

**基本信息**

| 配置项 | 说明 | 约束 |
| --- | --- | --- |
| 头像 | AI Agent 的展示头像 | 支持 JPG/JPEG/PNG，不超过 10MB；上传头像需要进行裁剪，头像不在访客端展示，仅在客服端会话列表中显示 |
| 昵称 | AI Agent 的名称标识 | 必填，最长 64 字符 |
| 业务简介 | 描述业务和服务范围 | 最长 2000 字符；AI Agent 会据此生成更贴合场景的回答 |

**回复风格**

| 配置项 | 说明 | 选项 | 默认值 |
| --- | --- | --- | --- |
| 回复语气 | AI Agent 的表达风格 | 友好亲切 / 专业严谨 / 幽默活泼 / 简洁高效 | 友好亲切 |
| 默认语言 | AI Agent 会检测访客语言并使用该语言进行回复，当无法判断访客语言时，将使用默认进行回复 | 英语、西班牙语、法语、德语、葡萄牙语、俄语、简体中文、繁体中文、日语、韩语、越南语、泰语、印尼语、马来西亚语 | 英语 |

### 3.3 配置保存机制

*   所有配置项采用**自动保存**模式：任何配置变更（输入框失焦、下拉切换、开关切换等）立即持久化。
    
*   保存成功后展示 Toast 提示"保存成功"。
    
*   开启/暂停 AI Agent 操作立即生效，并展示对应 Toast 提示（"AI Agent 已开启"/"AI Agent 已暂停"）。
    

### 3.4 配置输入校验规则

所有配置中的必填文本输入框遵循统一校验规则：

| 场景 | 行为 |
| --- | --- |
| 输入框内容为空时失焦 | Toast 提示"输入内容不能为空"，并恢复内容 |

### 3.5 多语言预填充规则

AI Agent 相关配置内容需根据项目创建时所选择的默认语言，预填充对应语言的文案：

| 配置项 | 说明 |
| --- | --- |
| 兜底回复 | 根据项目语言预填充对应翻译 |
| 转接提示语 | 根据项目语言预填充对应翻译 |
| 客服离线提示 | 根据项目语言预填充对应翻译 |
| 不转接人工提示 | 根据项目语言预填充对应翻译 |
| 跟进消息 | 根据项目语言预填充对应翻译 |
| AI Agent 默认语言 | 与创建项目时语言一致 |

示例：若项目语言为英语，则兜底回复预填为 "Sorry, I can't handle this type of content right now. Please describe it in text or try a different question."；若项目语言为简体中文，则预填为"抱歉，我暂时无法处理这类内容，请用文字描述或换个问题"。

当管理员更新任何可翻译文案（兜底回复、转接提示语、客服离线提示、不转接人工提示、跟进消息）后，系统将自动同步翻译为其他已支持的语言版本。输入框下方显示提示“更新文案后，系统将自动同步翻译为其他已支持的语言”。

## 五、会话模块 — AI Agent 会话队列（新增）

### 5.1 AI Agent 队列定义

会话模块左侧导航新增"AI Agent"分组，包含一个队列项"AI Agent"。"AI Agent"队列展示所有当前由 AI Agent 处理的活跃会话。

### 5.2 AI Agent 导航计数

*   AI Agent 导航项上显示**计数红点**，标识当前队列中有多少条 AI Agent 会话（如 "AI Agent (3)"）。
    
*   AI Agent 会话列表中**不显示**未读红点
    

### 5.3 会话列表能力

AI Agent 队列中的会话列表须支持与在线会话一致的以下能力：

| 能力 | 说明 |
| --- | --- |
| 关键词搜索 | 支持按客户标识、访客姓名、访客备注名搜索 |
| 列表排序 | 默认按最新消息时间倒序排列 |
| 消息预览 | 显示最新一条消息摘要 |
| 时间显示 | 显示最新消息时间 |
| 分类筛选 | 搜索栏下方显示"全部 / 访客 / 客户"分类标签，支持按访客类型筛选会话列表。各标签后跟随实时计数，数据为空显示 0。默认选中"全部"，切换队列时自动重置。当系统未接入客户标识时，仅显示"全部"标签 |

### 5.4 AI Agent 会话详情

进入 AI Agent 队列中的某条会话后：

*   **消息区域**：展示完整的访客-AI Agent 对话记录。消息角色包括访客、AI Agent、system（系统消息）。-
    
*   **消息顶部**：AI Agent 会话没有任何操作按钮（没有结束会话、转移会话、邀请客服、标记为待处理）。
    
*   **操作栏**：消息区域底部显示"分配会话"和"接管会话"两个按钮。人工客服不能在 AI Agent 会话中直接发送消息，必须先接管。
    
*   **右侧面板**：与在线会话一致，展示访客信息和会话信息。
    

### 5.5 人工接管 AI Agent 会话

#### 5.5.1 接管入口

| 入口位置 | 触发方式 | 适用角色 |
| --- | --- | --- |
| 会话模块 — AI Agent 队列 — 会话详情底部 | 点击"接管会话"按钮 | 所有人工客服 |
| 档案模块 — 会话记录列表 — 操作菜单 | 点击"接管会话"（当会话归属为 AI Agent 时显示） | 管理员：打开分配弹窗选择接管客服；普通客服：确认后直接分配给自己 |
| 档案模块 — 会话详情抽屉 — 操作按钮 | 点击"接管会话"按钮 | 同上 |

#### 5.5.2 接管确认流程

1.  人工客服点击"接管会话"。
    
2.  系统弹出确认对话框："确定接管该会话吗？"
    
3.  人工客服点击"确认"完成接管，或点击"取消"关闭对话框。
    

#### 5.5.3 接管后会话状态判定规则

接管会话成功后，会话从"AI Agent"队列移入"在线会话"分组。具体进入哪个队列，取决于接管时 AI Agent 是否已回复过访客的最近一条消息。

接管时若 AI Agent 正在回复则不展示 AI Agent 回复内容

| 判定条件 | 目标队列 | 说明 |
| --- | --- | --- |
| 访客最新消息之后 AI Agent **已回复** | 已回复 | AI Agent 已经给出了回复 |
| 访客最新消息之后 AI Agent **未回复** | 待回复 | AI Agent 尚未回复访客的最新消息 |

#### 5.5.4 接管后的系统行为与系统消息规范

接管/转移/分配会话时，系统在聊天记录中插入一条系统消息。不同场景的消息格式如下：

| 场景 | 系统消息格式 | 示例 |
| --- | --- | --- |
| AI Agent 主动转接人工客服 | AI Agent已将会话转移给{接收客服名} | AI Agent已将会话转移给客服主管 |
| 人工客服主动接管 AI Agent 会话 | AI Agent已将会话转移给{接管客服名} | AI Agent已将会话转移给客服主管 |
| 管理员在档案中分配 AI Agent 会话给指定客服 | {管理员昵称}已将会话转移给{指定客服名} | 客服主管已将会话转移给王珂 |

接管完成后的其他系统行为：

1.  会话的负责人从 AI Agent 变更为接管的人工客服。
    
2.  接管完成后展示 Toast 提示"接管会话成功"。
    
3.  会话列表自动定位到目标队列（待回复或已回复）。
    
4.  接管后，人工客服可正常使用消息输入框发送消息、转接、邀请协作者等全部功能。
    

#### 5.5.5 接管条件

*   AI Agent 队列中的会话均可被接管，不区分会话状态。
    
*   同一会话同一时间只能被一个人工客服接管。
    
*   接管操作不可逆——一旦接管，会话不能再退回 AI Agent 队列。
    

#### 5.5.6 分配会话流程

在 AI Agent 会话详情底部点击“分配会话”按钮，打开分配弹窗。

1.  弹窗展示可分配的客服列表（支持关键词搜索），按在线状态排序。
    
2.  点击某客服的“分配”按钮后，显示二次确认气泡（“确定分配给该客服吗？”）。
    
3.  点击“确定”后，会话分配给该客服，从 AI Agent 列表中移除，并自动选择列表中第一个会话。Toast 提示“会话分配成功”。
    
4.  分配会话不校验接收方当前的并发接待上限，直接分配成功。
    

#### 5.5.7 分配会话 Push 通知

会话分配成功后，系统需向被分配人发送 Push 通知（网页端和  APP 都需要）

1.  title：新会话消息
    
2.  内容描述：{分配人昵称}已将会话转移给你

## 九、访客端 AI Agent 交互需求

### 9.1 总述

访客端（Web 聊天窗口）需要对 AI Agent 的交互进行专门适配，确保访客在与 AI Agent 对话时拥有清晰、自然的体验。

AI Agent 不采用流式输出，内容生成完毕后一次性返回。

### 9.2 AI Agent 回复中状态

| 功能 | 说明 |
| --- | --- |
| 回复中消息气泡 | 当 AI Agent 正在生成回复时，在消息区域展示一个"回复中"消息气泡（三点跳动），告知访客 AI Agent 正在处理。回复生成完成后，"回复中"气泡替换为实际回复内容 |
| AI Agent 标识 | 若"AI Agent 标签"配置为"显示"，则 AI Agent 发送的每条消息气泡下显示"AI Agent"标识标签，帮助访客区分 AI 回复和人工回复 |

### 9.3 转接人工过渡

| 场景 | 访客端表现 |
| --- | --- |
| AI Agent 发起转接 | 展示转接提示语消息气泡（如"正在为你转接人工客服，请稍候"），随后由人工客服接入继续对话 |
| 所有人工客服不在线 | 展示客服离线提示消息气泡（如"当前客服不在线。你可以先留下问题或联系方式…"） |
| 人工客服接管成功 | 对话无缝继续，访客无需感知切换过程。若 AI Agent 标签为"显示"，后续人工回复的消息气泡不再展示"AI Agent"标识 |

### 9.4 跟进与自动关闭

| 场景 | 访客端表现 |
| --- | --- |
| 跟进消息 | AI Agent 发送的跟进消息作为普通消息气泡展示在对话中 |
| 自动关闭 | 会话自动关闭后，展示系统消息提示"会话已结束"。访客再次发送消息时开启新会话 |

### 9.5 无法处理内容

当访客发送图片、文件、语音或敏感等内容时，AI Agent 回复预设的不支持消息（如："抱歉，我暂时无法处理这类内容，请用文字描述或换个问题"）。

## 十、访客连续消息处理机制

### 10.1 场景说明

访客可能在短时间内连续发送多条消息，AI Agent 需要有合理的应对机制，避免对每条消息逐一回复导致回复碎片化或消息交叉。

### 10.2 处理规则

| 场景 | AI Agent 行为 | 说明 |
| --- | --- | --- |
| 访客在 AI Agent 尚未回复时连续发送消息 | 等待 + 合并处理 | AI Agent 在收到第一条消息后启动一个短暂的等待窗口（建议 3 秒）。若等待期间收到新消息，重置计时器。等待结束后，将窗口期内的所有消息作为一个整体进行理解和回复 |
| 访客在 AI Agent 正在生成回复时发送新消息 | 中断当前回复 + 合并 | AI Agent 放弃正在生成的回复，将新消息纳入上下文，重新生成更完整的回复。<br>交互：<br>\* 气泡保持在原位，打字动画继续<br>\* 访客的新消息插入到气泡上方（正常的消息顺序）<br>\* 气泡里的内容无缝切换为新生成的完整回复 |
| 访客在 AI Agent 已回复后又追加消息 | 正常响应 | AI Agent 将追加消息视为新一轮对话，正常生成回复 |

*   AI Agent 接入的会话计入"AI Agent"相关统计（首页概览、AI Agent 报表）。
    
*   从 AI Agent 接管到人工客服的会话：在 AI Agent 统计中记为"转人工"，同时**也计入人工客服的数据统计**（从接管时刻起纳入人工客服会话总数、响应时间等指标）。
    
*   即：同一会话如果经历了 AI Agent → 人工接管的流程，该会话在 AI Agent 统计和人工客服统计中**均会被计入**，分别反映各阶段的服务贡献。

## 十三、注意事项

### 13.1 性能与并发

*   AI Agent 可能同时处理大量会话，系统需确保消息队列的处理性能和回复的及时性。
    
*   连续消息合并窗口的计时器需在服务端实现，避免客户端计时不准确。
    

### 13.2 异常情况

| 异常场景 | 处理方式 |
| --- | --- |
| AI Agent 服务异常/超时 | 向访客发送兜底回复消息 |
| 知识库为空 | AI Agent 基于通用能力和提示词进行回复，严格模式下发送提示词中定义的通用回复 |
| 访客发送非文本内容（图片、文件、语音等） | AI Agent 发送预设的不支持消息（如："抱歉，我暂时无法处理这类内容，请用文字描述或换个问题"） |

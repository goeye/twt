# PRD：会话标记为待处理交互调整

> 版本：1.0
> 最后更新：2026-03-13
> 状态：已实现

---

## 1. 概述

### 1.1 功能定位

「标记为待处理」是会话工作台中的会话状态流转操作，客服人员可以将当前正在处理的会话标记为「待处理」状态，表示该会话需要后续跟进但无需立即回复。会话进入「待处理」队列后，消息编辑器中的勾选项自动切换为「标记为已解决」，方便客服在后续跟进完成后将会话流转回「已回复」状态。

### 1.2 功能目标

- 提供两种触发路径（发送消息时勾选 + 会话头部下拉菜单），覆盖不同操作场景
- 会话标记为待处理后，编辑器底部从「标记为待处理」切换为「标记为已解决」，实现状态闭环
- 会话状态流转清晰：待回复/排队中 → 待处理 → 已回复

### 1.3 功能入口

| 入口 | 位置 | 触发方式 |
|-----|------|---------|
| 消息编辑器勾选框 | 聊天面板底部 → 发送按钮左侧 | 勾选「标记为待处理」后点击发送 |
| 会话头部下拉菜单 | 会话标题右侧 → 「···」更多操作按钮 → 下拉菜单 | 点击「标记为待处理」菜单项 |

### 1.4 影响范围

本功能涉及以下页面/模块：

| 页面/模块 | 影响说明 |
|----------|---------|
| 会话工作台（App.vue） | 核心交互逻辑：状态流转、队列切换、编辑器条件显示 |
| MessageComposer 组件 | 底部勾选框条件展示（标记为待处理 / 标记为已解决） |
| ConversationHeader 组件 | 下拉菜单中的「标记为待处理」操作项 |
| SessionQueueNav 组件 | 左侧导航队列中「待处理」队列计数 |
| 首页（HomeRoutePage） | 实时会话统计中的「待处理会话」数值展示 |
| 归档会话列表（ArchiveConversationTab） | 会话状态筛选项、状态标签、汇总统计 |
| 设置 → Webhook 参数 | Webhook 推送参数中 status 字段包含「3: 待处理」 |

## 2. 核心概念 / 实体定义

### 2.1 会话队列状态

会话在「在线会话」分组下存在以下 4 种队列状态：

| 队列 Key | 显示名称 | Emoji | 说明 |
|----------|---------|-------|------|
| pending-reply | 待回复 | 👋 | 访客发起会话后等待客服首次回复 |
| queueing | 排队中 | ⌛ | 访客排队等待分配客服 |
| processing | 待处理 | 📝 | 客服主动标记，表示需后续跟进 |
| resolved | 已回复 | ↩️ | 客服已回复，等待访客反馈或自动关闭 |

### 2.2 会话状态生命周期

```
待回复 (pending-reply)  ──┐
                          ├──→  待处理 (processing)  ──→  已回复 (resolved)
排队中 (queueing)  ───────┘
```

- **待处理 → 已回复**：客服在「待处理」会话中发送消息并勾选「标记为已解决」
- **待回复/排队中 → 待处理**：客服通过勾选框或下拉菜单触发
- 已处于「待处理」状态的会话**不可重复标记**（`markSessionAsPending` 会跳过）

## 3. 数据 / 配置详情

### 3.1 会话数据结构（相关字段）

| 字段 | 类型 | 说明 |
|-----|------|------|
| id | string | 会话唯一标识，如 `s-6001` |
| queueKey | string | 当前队列状态：`pending-reply` / `queueing` / `processing` / `resolved` |
| customerName | string | 会话标题（访客话题摘要） |
| assignee | string | 当前负责客服名称 |
| assistants | string[] | 协助客服列表 |

### 3.2 编辑器状态字段

| 字段 | 类型 | 默认值 | 说明 |
|-----|------|-------|------|
| markPendingChecked | boolean | false | 「标记为待处理」勾选状态 |
| markResolvedChecked | boolean | false | 「标记为已解决」勾选状态 |
| showMarkPending | boolean | 计算值 | 当会话**不处于**待处理状态时显示 |
| showMarkResolved | boolean | 计算值 | 当会话**处于**待处理状态时显示 |

### 3.3 Webhook 状态码映射

| 状态码 | 含义 |
|-------|------|
| 1 | 待回复 |
| 2 | 排队中 |
| 3 | 待处理 |
| 4 | 已回复 |

## 4. 操作流程

### 4.1 通过发送消息勾选「标记为待处理」

**前置条件**：当前会话不处于「待处理」状态（即 `queueKey !== "processing"`）

| 步骤 | 操作 | 系统行为 |
|-----|------|---------|
| 1 | 在消息编辑器底部勾选「标记为待处理」 | 勾选框选中，发送按钮无额外变化 |
| 2 | 输入消息文本后点击「发送」 | ① 消息正常发送到聊天流 ② 会话 `queueKey` 变更为 `processing` ③ 勾选状态重置为 `false` ④ 左侧队列自动切换到「待处理」队列 |

**校验规则**：
- 消息文本为空时发送按钮禁用（`disabled`），无法触发发送
- 勾选「标记为待处理」不影响发送按钮的启用/禁用状态

### 4.2 通过会话头部下拉菜单标记

**前置条件**：当前已选中一个活跃会话

| 步骤 | 操作 | 系统行为 |
|-----|------|---------|
| 1 | 点击会话标题右侧的「···」更多操作按钮 | 弹出下拉菜单 |
| 2 | 点击「标记为待处理」 | ① 下拉菜单关闭 ② 会话 `queueKey` 变更为 `processing` ③ 左侧队列自动切换到「待处理」队列 |

**注意**：通过下拉菜单标记**不需要发送消息**，直接变更状态。

### 4.3 从「待处理」标记为已解决

**前置条件**：当前会话处于「待处理」状态（`queueKey === "processing"`）

| 步骤 | 操作 | 系统行为 |
|-----|------|---------|
| 1 | 编辑器底部显示「标记为已解决」勾选框（替代「标记为待处理」） | 自动条件切换 |
| 2 | 勾选「标记为已解决」后输入消息并发送 | ① 消息正常发送 ② 会话 `queueKey` 变更为 `resolved` ③ 勾选状态重置为 `false` ④ 左侧队列自动切换到「已回复」队列 |

## 5. 交互规则

### 5.1 编辑器底部勾选框互斥显示

| 当前会话状态 | 编辑器底部显示 |
|------------|--------------|
| 待回复 (pending-reply) | ☑ 标记为待处理 |
| 排队中 (queueing) | ☑ 标记为待处理 |
| 已回复 (resolved) | ☑ 标记为待处理 |
| 待处理 (processing) | ☑ 标记为已解决 |
| AI Agent 会话 | 无编辑器（显示 AI 接管提示） |

两个勾选框**不会同时出现**，由 `isProcessingSession` 计算属性决定显示哪个。

### 5.2 下拉菜单「标记为待处理」始终可见

- 下拉菜单中的「标记为待处理」菜单项**始终显示**，不根据当前会话状态隐藏
- 但如果会话已处于 `processing` 状态，调用 `markSessionAsPending()` 时会直接 return（防重复操作）

### 5.3 队列自动切换

- 标记为待处理后，左侧队列导航自动选中「待处理」队列（`activeQueueKey = "processing"`）
- 标记为已解决后，左侧队列导航自动选中「已回复」队列（`activeQueueKey = "resolved"`）

### 5.4 勾选状态重置

- 发送消息后无论是否触发状态变更，`markPendingChecked` 和 `markResolvedChecked` 均重置为 `false`
- 切换会话时勾选状态**不保留**（切换回来时为默认未勾选状态）

### 5.5 下拉菜单中的协作操作与待处理的关系

- 「添加客服」和「转移会话」菜单项在排队中（`queueing`）状态下为禁用态（灰色不可点击）
- 「标记为待处理」不受协作状态限制，始终可点击

## 6. 关联与依赖

### 6.1 左侧队列导航（SessionQueueNav）

- 「待处理」队列的计数（红色徽章数字）根据 `allSessions` 中 `queueKey === "processing"` 的会话数实时更新
- 「已回复」队列**不显示**计数徽章（`shouldShowBadge` 中硬编码返回 false）

### 6.2 首页统计（HomeRoutePage）

- 「实时会话统计」卡片中包含「待处理会话」指标项，展示当前处于待处理状态的会话数
- 数据结构：`{ key: "pending-process", label: "待处理会话", value: 1 }`

### 6.3 归档会话列表（ArchiveConversationTab）

- 会话状态枚举包含 `processing`（待处理）
- 状态筛选下拉菜单中包含「待处理」选项
- 状态标签样式：`processing` → 灰色（`#8f9aae`）
- 汇总统计中包含「待处理」数量
- 待处理状态的会话：负责客服显示为「–」，客服数为 0，受理时间和服务时长显示为「–」

### 6.4 Webhook 推送

- 设置页面 Webhook 参数表中，`status` 字段值 `3` 对应「待处理」
- 会话状态变更时可通过 Webhook 推送通知外部系统

## 7. 业务规则汇总表

| # | 规则 | 触发条件 | 预期行为 |
|---|------|---------|---------|
| 1 | 防止重复标记 | 会话已处于 `processing` 状态时调用 `markSessionAsPending()` | 直接 return，不做任何操作 |
| 2 | 非待处理会话不可标记已解决 | 会话不处于 `processing` 状态时调用 `markSessionAsResolved()` | 直接 return，不做任何操作 |
| 3 | 发送消息时互斥触发 | 同时勾选不可能（UI 互斥），但代码使用 `else if` | 优先处理「标记为待处理」，其次处理「标记为已解决」 |
| 4 | 空消息不可发送 | 消息文本 trim 后为空 | 发送按钮禁用，无法触发任何状态变更 |
| 5 | 编辑器勾选框条件显示 | 会话 `queueKey` 变更 | 非 processing → 显示「标记为待处理」；processing → 显示「标记为已解决」 |
| 6 | 队列自动跳转 | 标记为待处理或已解决后 | 自动切换 `activeQueueKey` 到对应队列 |
| 7 | 勾选状态发送后重置 | 消息发送完成后 | `markPendingChecked` 和 `markResolvedChecked` 均重置为 `false` |

### 8.1 核心标记流程

|-----|---------|---------|

| 用例 | 操作步骤 | 预期结果 |
|-----|---------|---------|
| 已处于待处理状态时再次标记 | 在「待处理」会话中点击下拉菜单「标记为待处理」 | 无任何变化（静默忽略） |
| 空消息带勾选发送 | 勾选「标记为待处理」但不输入消息 → 点击发送 | 发送按钮禁用，无法点击，会话状态不变 |
| 排队中会话标记 | 在「排队中」会话中通过下拉菜单标记为待处理 | 会话成功移动到「待处理」队列 |

### 8.3 编辑器勾选框切换

| 用例 | 操作步骤 | 预期结果 |
|-----|---------|---------|
| 非待处理会话编辑器 | 选中一个「待回复」或「已回复」会话 | 编辑器底部显示「标记为待处理」勾选框 |
| 待处理会话编辑器 | 选中一个「待处理」会话 | 编辑器底部显示「标记为已解决」勾选框 |
| 状态变更后勾选框切换 | 在「待回复」会话中通过下拉菜单标记为待处理 | 编辑器底部勾选框从「标记为待处理」变为「标记为已解决」 |

| 用例 | 操作步骤 | 预期结果 |
|-----|---------|---------|

## 10. 名词解释

| 名词 | 说明 |
|-----|------|
| 待处理 (processing) | 客服主动标记的会话状态，表示需要后续跟进但当前无需立即回复 |
| 标记为已解决 | 仅在「待处理」会话中可用的操作，将会话从待处理流转到已回复状态 |
| 队列 (queueKey) | 会话所属的分类队列，决定会话在左侧导航的哪个列表中显示 |
| 编辑器 (MessageComposer) | 聊天面板底部的消息输入区域，包含工具栏、文本框、勾选框和发送按钮 |
| 更多操作菜单 | 会话标题右侧「···」按钮展开的下拉菜单，包含添加客服、转移会话、标记为待处理、结束会话 |

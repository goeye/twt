# PRD - APP 在线会话支持备注消息类型

## 需求概述

在 mobile-app 的会话聊天详情中显示备注消息类型，备注消息的发送人昵称和时间使用专属视觉样式与普通消息区分。不增加备注回复模式（即不支持在 APP 中发送备注消息）。

## 功能范围

### 包含功能

- 会话聊天详情中显示备注消息
- 备注消息的发送人昵称和时间使用专属视觉标识

### 排除功能

- 不支持在 APP 中发送备注消息
- 不增加备注回复模式切换

## 数据结构

### 消息对象扩展

在现有 `Message` 接口中，`type` 字段已支持 `'note'` 类型：

```typescript
interface Message {
  id: number;
  type?: 'normal' | 'system' | 'note' | 'rating';
  role?: 'customer' | 'agent';
  sender: string;
  avatar: string;
  time: string;
  content: string;
}
```

## 交互规则

### 消息显示规则

**备注消息识别**
- 当消息对象的 `type` 字段为 `'note'` 时，识别为备注消息
- 备注消息仅客服端可见，访客端不显示

**消息布局**
- 备注消息使用与普通客服消息相同的布局：右对齐，显示发送者、时间、头像、消息气泡
- 消息气泡的圆角、内边距、最大宽度与普通消息保持一致

**视觉区分**
- 备注消息的发送人昵称和时间使用专属视觉标识（与普通消息的灰色文字区分）
- 消息气泡背景使用专属背景色（与普通消息的白色背景区分）

**消息内容**
- 备注消息的文本内容正常显示，支持换行
- 不显示任何"仅内部可见"的提示文案

### 消息排序

- 备注消息按时间顺序插入消息列表中，与普通消息、系统消息混合显示
- 不单独分组或置顶

## 页面位置

**会话聊天详情页面** (`/session/:id`)
- 在消息列表中渲染备注消息
- 备注消息与普通消息、系统消息、评价消息混合显示

## 边界情况

**无备注消息**
- 如果会话中没有备注消息，不显示任何占位符或提示

**备注消息发送者**
- 备注消息的 `sender` 字段显示发送该备注的客服昵称
- 如果发送者信息缺失，显示"未知客服"

**备注消息时间**
- 备注消息的 `time` 字段显示发送时间（格式：HH:mm）
- 如果时间信息缺失，显示"--:--"

## 参考实现

**web-agent 中的备注消息实现**
- 文件：`packages/ui-agent/src/components/conversation/MessageBubble.vue`
- 备注消息通过 `isNote` 属性标记
- 备注消息隐藏"回复"、"翻译"、"更多"按钮，仅保留"复制"按钮

**mobile-app 中的现有消息类型**
- 文件：`apps/mobile-app/src/views/SessionDetailView.vue`
- 已支持 `type: 'note'` 的消息类型定义
- 当前实现中备注消息的发送者和时间已使用专属视觉标识

# PRD：在线会话与档案增加搜索沟通记录

> **版本**：v1.1 · 2026-04-04
> **状态**：已交付

---

## 1. 概述

### 1.1 背景与动机

| 痛点 | 影响 |
|------|------|
| 会话和历史档案中，无法精准定位包含特定内容的聊天消息 | 客服查找历史沟通记录耗时过长，服务效率低下 |

在线客服系统需要支持在当前会话和历史归档中高效地搜索并定位到具体的沟通记录。本功能旨在提供一套统一的搜索沟通记录交互：支持在列表页展示匹配数量，提供聚合的匹配消息列表供用户选择，并在选中后精准定位至原上下文并高亮展示。

### 1.2 目标

| Key Result | 量化标准 |
|-----------|---------|
| KR1：统一各入口的搜索交互体验 | 在线会话、在线聊天、Autopilott、会话记录、聊天记录五处搜索沟通记录的交互模式一致 |
| KR2：提升多条匹配时的定位效率 | 多条匹配时提供独立的匹配消息列表面板供用户选择，点击即达 |

## 3. 功能设计

### 3.1 核心流程

```mermaid
graph TD
    A[用户输入关键词搜索沟通记录] --> B{是否匹配到消息?}
    B -- 否 --> C[列表过滤掉该会话]
    B -- 匹配 1 条 (仅在线会话) --> D[列表显示单条匹配预览]
    B -- 匹配多条 (或在档案页) --> E[列表显示匹配数量]
    
    D --> F[用户点击会话项]
    F --> G[直接进入会话详情并高亮该消息]
    
    E --> H[用户点击"N条相关记录"或会话项]
    H --> I{入口类型}
    I -- 在线会话页面 --> J[弹出居中的消息列表弹窗]
    I -- 档案页面 --> K[右侧划出消息列表抽屉]
    
    J --> L[用户点击具体消息项]
    K --> L
    L --> G
```

### 3.2 子功能详述

#### 3.2.1 在线会话/在线聊天/Autopilot搜索沟通记录

**功能描述**：在进行中的客服工作台中搜索具体的聊天消息内容。

**前置条件**：
搜索字段选择为「全部」或「沟通记录」

**交互流程**：
1. 用户在搜索框输入关键词，列表实时过滤
2. 对于匹配 1 条消息的会话：列表项展示该条消息的截断预览
3. 对于匹配 >1 条消息的会话：列表项展示「N条相关记录」
4. 用户点击单条匹配的会话：直接在右侧打开会话详情，消息列表滚动至匹配消息，该消息整行高亮，3s后高亮消失
5. 用户点击多条匹配的会话：屏幕中央弹出「消息列表」模态框，列出该会话内所有匹配消息
6. 用户点击模态框内的某条消息：模态框关闭，右侧打开会话详情，消息列表滚动至该选中消息并高亮，3s后高亮消失

**需求描述（功能规则）**：
1. **搜索字段**：
   - 在线会话/Autopilot搜索条件中增加：增加「会话标题」、「沟通记录」搜索项
   - 在线会话/Autopilo搜索字段顺序调整：全部、访客姓名、访客备注名、会话标题、沟通记录、客户标识
   - 在线聊天搜索条件中增加：增加「客服姓名」、「聊天标题」、「沟通记录」搜索项
   - 在线聊天搜索字段顺序调整：全部、访客姓名、访客备注名、客户姓名、聊天标题、沟通记录、客户标识
   - 默认选中「全部」；选择「全部」时同时匹配所有字段（OR 逻辑）
   - 在线会话/Autopilot/在线聊天搜索项中的「客户标识」字段仅在项目已接入客户模块时展示，未接入时自动隐藏
2. **搜索规则**：
   - 不区分大小写
   - 仅搜索消息的文本原文内容（排除系统消息、图片、文件、语音、发送失败等非文本有效消息，包含富文本/HTML类型消息的文本，包含客服内部沟通的备注文本）
   - 不搜索被翻译的文本内容，仅搜索原始发出的文本
   - 搜索范围包含已撤回、已删除消息的原始内容
2. **匹配多条消息时消息列表弹窗呈现**：
   - 包含发信人头像、昵称/备注名、发送时间和气泡文本
   - 消息按时间正序排列
   - 所有角色的消息气泡全部左对齐显示（与正常聊天流中自己发的消息右对齐不同）
   - 文本气泡内容最多展示 1 行，超出部分以省略号截断
   - 气泡内容以 KWIC 格式展示：采用固定字符截取方案（具体截取字符数待确认），关键词前后各保留若干字符作为上下文，被截断部分以 `...` 表示，关键词以蓝色背景高亮显示
   - 弹窗支持内部垂直滚动，弹窗头部与底部固定
   - 支持点击遮罩层外部空白处关闭
3. **定位规则**：
   - 选中并进入会话详情后，系统需通过平滑滚动将选中的消息置于可视区域居中位置
   - 跳转后仅有当前选中的消息展示高亮状态
   - 高亮状态持续 3 秒后自动渐隐消失

#### 3.2.2 档案页面搜索沟通记录（会话记录/聊天记录）

**功能描述**：在历史归档数据中搜索聊天消息。

**前置条件**：
1. 用户位于「档案 - 会话记录」或「档案 - 聊天记录」标签页
2. 筛选项中的搜索字段选择为「沟通记录」并输入关键词

**交互流程**：
1. 用户点击「搜索」按钮提交过滤请求
2. 在结果列表的会话/聊天标题单元格下方，匹配的行会展示「N条相关记录」的文本按钮
3. 用户点击「N条相关记录」：右侧滑出标题为「消息列表」的抽屉，展示所有匹配消息
4. 用户直接点击会话/聊天标题：右侧滑出完整的「会话详情/聊天详情」抽屉
5. 用户点击「消息列表」抽屉内的某条匹配消息：该抽屉关闭，重新滑出包含所有上下文的完整详情抽屉，并自动滚动至选中的消息并高亮

**需求描述（功能规则）**：
1. **搜索字段**：
   - 会话记录搜索字段增加：增加「全部」、「沟通记录」搜索项
   - 会话记录搜索字段顺序：全部、访客姓名、访客备注名、会话标题、沟通记录、客户标识
   - 聊天记录搜索字段增加：增加「全部」、「聊天标题」、「沟通记录」、「客户标识」搜索项
   - 聊天记录搜索字段顺序：全部、访客姓名、访客备注名、客服姓名、聊天标题、沟通记录、客户标识
   - 默认选中「全部」；选择「全部」时同时匹配所有字段（OR 逻辑）
   - 会话记录/聊天记录中的「客户标识」字段仅在账号已接入客户模块时展示，未接入时自动隐藏
2. **搜索逻辑**：同 3.2.1 节搜索规则。
2. **列表抽屉呈现**：
   - UI 结构和数据展示规则与在线会话的「列表弹窗」完全一致（左对齐、1行截断、头像/昵称/时间/内容）
   - 当搜索字段为「沟通记录」时，气泡内容以 KWIC 格式展示（规则同 3.2.1）
   - 形态为从屏幕右侧滑出的抽屉，宽度与常规详情抽屉保持一致
   - 消息列表区域支持垂直内部滚动
   - 支持点击遮罩层外部空白处关闭、支持按 Esc 键关闭
3. **定位规则**：
   - 同 3.2.1 节定位规则
   - 仅在通过匹配消息列表点击进入时，才执行平滑滚动和消息高亮

## 4. 权限与角色

此功能作为基础操作体验的优化，不对权限角色作差异化区分，所有具备访问在线会话或档案模块权限的用户（如客服专员、客服主管、管理员）均享有相同的搜索与匹配列表交互体验。

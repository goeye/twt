# PRD：实时聊天内容风控

> 版本：1.0
> 最后更新：2026-03-15
> 状态：部分实现（第二级 LLM/API 精确判断尚未接入真实服务，当前预筛命中即触发提醒）

---

## 1. 概述

### 1.1 功能定位

实时聊天内容风控是客服系统的安全防护模块，在会话进行中对每条文本消息进行实时检测。当检测到涉及资金转账、合同签署、个人证件等高风险内容时，自动在聊天流中插入一条系统风控提醒消息，提示双方注意安全风险。该功能为**只提醒不阻断**模式，不影响正常消息的发送和接收。

### 1.2 功能目标

- 对会话中的高风险内容进行实时检测和预警
- 通过两级检测策略平衡检测精度与成本：关键词预筛降低无效调用，LLM/API 精确判断减少误报
- 在聊天流中以醒目的警告样式展示风控提醒，确保双方注意到风险
- 同一规则短时间内不重复提醒，避免信息干扰

### 1.3 功能入口

风控系统为后台自动运行机制，无需用户主动操作。每条文本消息发送或接收后自动触发检测。

### 1.4 影响范围

| 页面/模块 | 改动类型 |
|-----------|---------|
| 客户端聊天页 | 新增风控检测逻辑、新增风控提醒消息类型及样式 |
| 演示工具栏 | 新增「模拟风控触发」按钮 |

## 2. 核心概念

### 2.1 风控规则

风控规则是检测的基本单元，每条规则包含以下属性：

| 属性 | 说明 |
|------|------|
| 规则标识 | 唯一标识，用于去重判断 |
| 风险分类 | 规则所属的风险类别（详见 2.2） |
| 关键词模式 | 用于第一级预筛的关键词集合，支持中英文 |
| 提醒文案 | 命中规则后展示给用户的提醒文字 |
| 启用状态 | 规则是否生效 |

### 2.2 风险分类

| 分类 | 说明 |
|------|------|
| 资金类 | 涉及转账、汇款、定金、预付款、账户信息等与资金流动相关的内容 |
| 合同类 | 涉及合同签署、协议、签约等法律文书相关的内容 |
| 个人信息类 | 涉及身份证、护照、驾照等个人证件信息的内容 |

### 2.3 风控提醒消息

风控提醒是一种特殊的系统消息类型，在聊天流中以警告样式展示。与普通聊天气泡的区别：

- 居中对齐，不属于任何一方
- 黄色警告背景，带警告图标
- 不可回复、不可操作
- 仅作为提示信息存在，不阻断任何消息的发送和接收

## 3. 检测策略

### 3.1 两级检测流程

整体流程为**先筛后判**：

1. **第一级 - 本地关键词预筛**：对消息文本进行关键词匹配，未命中则直接放行，不产生任何额外成本
2. **去重检查**：命中关键词后，检查该规则是否在冷却期内（5 分钟），若在冷却期内则跳过
3. **第二级 - LLM/第三方 API 精确判断**：仅对通过预筛和去重检查的消息调用外部服务进行语义级判断，减少误报

### 3.2 检测范围

| 消息类型 | 是否检测 | 说明 |
|---------|---------|------|
| 文本消息 | 是 | 核心检测对象 |
| 文件消息 | 仅检测文件名 | 检测文件名中是否包含敏感词（如"转账凭证.pdf"） |
| 图片消息 | 否 | 需 OCR 识别，成本和复杂度高 |
| 语音消息 | 否 | 需语音转文字，链路长、延迟高 |
| 系统消息 | 否 | 系统生成的状态消息，无风险内容 |
| 风控提醒消息 | 否 | 风控系统自身生成的消息，不二次检测 |

### 3.3 检测方向

双向监控：对访客发送的消息和客服/AI 回复的消息均进行检测。

## 4. 风控规则明细

### 4.1 资金类规则

#### 4.1.1 资金转账

- **触发关键词**：转账、汇款、打款、付款、收款、付钱、打钱、银行卡（中文）；bank card、wire transfer、remittance、payment（英文）
- **提醒文案**：风控提醒：检测到涉及资金转账的内容，请注意核实对方身份，谨防诈骗。

#### 4.1.2 预付款/定金

- **触发关键词**：定金、预付款、押金、保证金（中文）；deposit、advance payment、down payment（英文）
- **提醒文案**：风控提醒：检测到涉及预付款/定金的内容，请通过官方渠道完成支付，切勿私下转账。

#### 4.1.3 账户信息

- **触发关键词**：账号、账户、卡号、收款码、支付宝、微信支付（中文）；account number、card number、IBAN、SWIFT、PayPal（英文）
- **提醒文案**：风控提醒：检测到涉及账户/支付信息的内容，请勿在聊天中直接发送敏感账户信息。

### 4.2 合同类规则

#### 4.2.1 合同签署

- **触发关键词**：合同、签约、协议、签署（中文）；contract、agreement、sign（英文）
- **提醒文案**：风控提醒：检测到涉及合同/协议的内容，请确认通过正规流程签署，注意条款细节。

### 4.3 个人信息类规则

#### 4.3.1 身份证件

- **触发关键词**：身份证、护照、驾照、社保号（中文）；ID card、passport、driver's license、SSN（英文）
- **提醒文案**：风控提醒：检测到涉及身份证件信息的内容，请勿在聊天中发送证件号码等敏感信息。

## 5. 交互规则

### 5.1 提醒消息展示

- 风控提醒消息在聊天流中**紧跟触发消息之后**展示
- 提醒消息水平居中对齐，不偏向访客或客服任何一侧
- 检测为异步执行，提醒消息在检测完成后立即插入

### 5.2 去重机制

- 同一条风控规则在同一会话中，5 分钟内最多触发一次提醒
- 不同规则之间互不影响（如同一条消息同时命中「资金转账」和「账户信息」两条规则，仅触发优先匹配到的第一条）
- 会话重置（如重新开始新会话）时清空所有去重记录

### 5.3 与现有功能的关系

- 风控检测与 AI Agent 回复、转人工流程并行执行，互不阻塞
- 风控提醒不影响已读回执、打字动画等现有功能
- 会话结束后不再触发风控检测

## 6. 业务规则汇总表

| # | 规则 | 触发条件 | 预期行为 |
|---|------|---------|---------|
| 1 | 文本消息双向检测 | 访客或客服/AI 发送文本消息 | 自动触发风控检测流程 |
| 2 | 非文本消息跳过 | 发送图片、语音、系统消息等非文本内容 | 不触发风控检测 |
| 3 | 风控提醒不自检 | 风控系统插入提醒消息 | 提醒消息本身不触发二次检测 |
| 4 | 预筛未命中放行 | 消息不包含任何风控关键词 | 不调用第二级检测，不产生提醒 |
| 5 | 冷却期内跳过 | 同一规则 5 分钟内已触发过 | 跳过本次检测，不产生提醒 |
| 6 | 命中后插入提醒 | 消息通过两级检测确认命中 | 在触发消息后插入居中的风控提醒消息 |
| 7 | 会话重置清空记录 | 用户重新开始会话 | 清空所有去重记录，规则可重新触发 |
| 8 | 只提醒不阻断 | 命中风控规则 | 消息正常发送和显示，仅额外插入提醒 |
| 9 | 待输入消息不检测 | 消息处于打字动画（待送达）状态 | 待消息内容实际填充后才触发检测 |
| 10 | 会话结束不检测 | 会话已结束 | 不接受新消息，不触发风控检测 |

## 7. 测试要点与边界场景

### 7.1 基础检测

| 用例 | 操作步骤 | 预期结果 |
|------|---------|---------|
| 普通消息不触发 | 发送「你好，我想咨询一下产品价格」 | 无风控提醒 |
| 资金转账关键词触发 | 发送「请把钱转账到我的银行卡」 | 消息后出现资金转账风控提醒 |
| 定金关键词触发 | 发送「需要先付定金 500 元」 | 消息后出现预付款风控提醒 |
| 账户信息关键词触发 | 发送「我的支付宝账号是 xxx」 | 消息后出现账户信息风控提醒 |
| 合同关键词触发 | 发送「我们可以签合同了」 | 消息后出现合同风控提醒 |
| 证件关键词触发 | 发送「请发一下你的身份证号」 | 消息后出现证件信息风控提醒 |
| 英文关键词触发 | 发送「Please send the wire transfer」 | 消息后出现资金转账风控提醒 |

### 7.2 去重机制

| 用例 | 操作步骤 | 预期结果 |
|------|---------|---------|
| 同一规则 5 分钟内不重复 | 连续发送两条包含「转账」的消息 | 仅第一条产生风控提醒 |
| 不同规则独立触发 | 先发送「转账」再发送「合同」 | 两条消息分别触发各自的风控提醒 |
| 会话重置后规则可重新触发 | 触发转账提醒 → 重置会话 → 再次发送含「转账」的消息 | 重置后的消息再次触发提醒 |

### 7.3 消息类型边界

| 用例 | 操作步骤 | 预期结果 |
|------|---------|---------|
| 空消息不检测 | 发送空白文本 | 无风控提醒 |
| AI 回复也检测 | AI 回复包含风控关键词 | 该回复后出现风控提醒 |
| 图片/文件消息不检测 | 发送图片消息 | 无风控提醒 |
| 打字动画状态不检测 | 消息处于待送达状态 | 打字动画期间不触发检测 |
| 会话结束后不检测 | 会话已结束时尝试发送消息 | 不触发风控检测 |

### 7.4 演示功能

| 用例 | 操作步骤 | 预期结果 |
|------|---------|---------|
| 模拟风控触发 | 点击浮动工具栏的「模拟风控触发」按钮 | 插入一条包含银行卡转账内容的访客消息，随后出现风控提醒 |

## 9. 名词解释

| 名词 | 说明 |
|------|------|
| 风控提醒 | 风控系统检测到风险内容后在聊天流中插入的警告消息 |
| 关键词预筛 | 第一级检测，使用关键词匹配快速过滤无关消息，零成本 |
| LLM 精确判断 | 第二级检测，将预筛命中的消息发送至大语言模型或第三方 API 进行语义级风险确认 |
| 冷却时间 | 同一规则连续触发之间的最短间隔（5 分钟），避免重复提醒干扰用户 |
| 只提醒不阻断 | 风控系统仅插入提醒消息，不阻止原始消息的发送和接收 |

# PRD：实时聊天内容风控

## 1. 概述

### 1.1 功能定位

实时聊天内容风控是客服系统的安全防护模块，在会话进行中对每条文本消息进行实时检测。当检测到涉及资金转账、合同签署、个人证件等高风险内容时，自动在聊天流中插入一条系统风控提醒消息，提示双方注意安全风险。该功能为**只提醒不阻断**模式，不影响正常消息的发送和接收。

### 1.2 功能目标

- 对会话中的高风险内容进行实时检测和预警
- 通过两级检测策略平衡检测精度与成本：关键词预筛降低无效调用，LLM/API 精确判断减少误报
- 在聊天流中以醒目的警告样式展示风控提醒，确保双方注意到风险
- 同一会话短时间内不重复提醒，避免信息干扰

### 1.3 功能入口

风控系统为后台自动运行机制，无需用户主动操作。每条文本消息发送或接收后自动触发检测。

### 1.4 影响范围

| 页面/模块 | 改动类型 |
|-----------|---------|
| 客户端会话/聊天 | 新增风控检测逻辑、新增风控提醒消息类型 |
| 访客端会话/聊天  | 新增风控检测逻辑、新增风控提醒消息类型 |
| 运营后台 → 词库管理 | 新增页面，管理风控关键词词库 |
| 运营后台 → 预警列表 | 新增页面，查看风控触发记录 |

## 2. 核心概念

### 2.1 风控提醒消息

风控提醒是一种特殊的系统消息类型，在聊天流中以灰色居中文本展示

## 3. 检测策略

### 3.1 两级检测流程

整体流程为**先筛后判**：

1. **第一级 - 本地关键词预筛**：对消息文本进行关键词匹配，未命中则直接放行，不产生任何额外成本
2. **去重检查**：命中关键词后，检查该会话是否在冷却期内（5 分钟），若在冷却期内则跳过
3. **第二级 - LLM/第三方 API 精确判断**：仅对通过预筛和去重检查的消息调用外部服务进行语义级判断，减少误报

### 3.2 检测范围

| 消息类型 | 是否检测 | 说明 |
|---------|---------|------|
| 文本消息 | 是 | 核心检测对象 |
| 文件消息 | 否 | 不检测 |
| 图片消息 | 否 | 需 OCR 识别，成本和复杂度高 |
| 语音消息 | 否 | 需语音转文字，链路长、延迟高 |
| 系统消息 | 否 | 系统生成的状态消息，无风险内容 |
| 风控提醒消息 | 否 | 风控系统自身生成的消息，不二次检测 |

### 3.3 检测方向

双向监控：对访客发送的消息和客服/AI 回复的消息均进行检测。

## 4. 风控规则明细

命中后统一使用相同的提醒文案：

> 在聊天过程中，涉及合同汇款、转账、资金、中奖、回款等信息，请仔细核实对方身份，以避免被骗导致钱款损失。

---

## 5. 交互规则

### 5.1 提醒消息展示

- 提醒消息在访客和客服端都展示
- 检测为异步执行，提醒消息在检测完成后立即插入，不用强调在触发消息下方

### 5.2 去重机制

- 同一会话中，5 分钟内最多触发一次提醒
- 会话重置（如重新开始新会话）时清空所有去重记录

### 5.3 与现有功能的关系

- 风控检测与 AI Agent 回复、转人工流程并行执行，互不阻塞
- 风控提醒不影响已读回执等现有功能
- 会话结束后不再触发风控检测

## 5.5 运营后台——词库管理

运营后台「合规管理 → 词库管理」提供风控关键词的后台管理能力。

### 5.5.1 词库列表

| 入口 | 路径 |
|------|------|
| 词库列表 | 运营后台 → 合规管理 → 词库管理 |

操作说明：

| 操作 | 说明 |
|------|------|
| 新增词库 | 弹窗中设置词库名称（必填）和批量输入词（一行一个） |
| 管理 | 进入词库详情页，查看和管理该词库下的所有词条 |
| 清空 | 清空该词库中的所有词条，需二次确认 |

表格列：词库名称、词数量、创建人、更新时间、操作（管理/清空）

### 5.5.2 词库详情

从词库列表点击「管理」进入词库详情页

操作说明：

| 操作 | 说明 |
|------|------|
| 添加 | 弹窗中批量输入词（一行一个），添加到当前词库 |
| 删除 | 删除单条词条，需二次确认 |

表格列：文本、创建时间（支持排序）、操作（删除）

## 5.6 运营后台——预警列表

运营后台「合规管理 → 预警列表」展示所有风控触发记录，便于运营人员审查和追溯。

| 入口 | 路径 |
|------|------|
| 预警列表 | 运营后台 → 合规管理 → 预警列表 |

筛选栏：日期范围（label + 日期选择器）+ 项目名称（label + 输入框）+ 项目ID（label + 输入框）+ 类型（label + 下拉选择：全部/会话/聊天）+ 搜索/重置按钮

表格列：

| 列名 | 说明 |
|------|------|
| 文本内容 | 触发风控的消息原文 |
| 类型 | 消息来源类型：会话或聊天 |
| 发送人 | 发送该消息的用户名称，前置角色标签（绿色「访客」或蓝色「客服」） |
| 会话/聊天标题 | 该消息所属会话或聊天的标题 |
| 项目名称 | 该会话所属项目的名称 |
| 项目ID | 该会话所属项目的 ID |
| 触发时间 | 风控规则被触发的时间 |

## 6. 业务规则汇总表

| # | 规则 | 触发条件 | 预期行为 |
|---|------|---------|---------|
| 1 | 文本消息双向检测 | 访客或客服/AI 发送文本消息 | 自动触发风控检测流程 |
| 2 | 非文本消息跳过 | 发送图片、语音、文件、系统消息等非文本内容 | 不触发风控检测 |
| 3 | 风控提醒不自检 | 风控系统插入提醒消息 | 提醒消息本身不触发二次检测 |
| 4 | 预筛未命中放行 | 消息不包含任何风控关键词 | 不调用第二级检测，不产生提醒 |
| 5 | 冷却期内跳过 | 同一会话 5 分钟内已触发过 | 跳过本次检测，不产生提醒 |
| 6 | 命中后插入提醒 | 消息通过两级检测确认命中 | 在聊天流中插入风控提醒系统消息 |
| 7 | 会话重置清空记录 | 用户重新开始会话 | 清空所有去重记录，重新触发 |
| 8 | 只提醒不阻断 | 命中风控规则 | 消息正常发送和显示，仅额外插入提醒 |
| 9 | 会话结束不检测 | 会话已结束 | 不接受新消息，不触发风控检测 |

## 7. 测试要点与边界场景

### 7.1 去重机制

| 用例 | 操作步骤 | 预期结果 |
|------|---------|---------|
| 同一会话 5 分钟内不重复 | 连续发送两条包含「风控词」的消息 | 仅第一条产生风控提醒 |
| 会话重置后规则可重新触发 | 触发风控提醒 → 重置会话 → 再次发送含「风控词」的消息 | 重置后的消息再次触发提醒 |

### 7.3 消息类型边界

| 用例 | 操作步骤 | 预期结果 |
|------|---------|---------|
| AI 回复也检测 | AI 回复包含风控关键词 | 该回复后出现风控提醒 |
| 图片/文件/视频/系统消息不检测 | 发送图片消息 | 无风控提醒 |
| 会话结束后不检测 | 会话已结束时尝试发送消息 | 不触发风控检测 |

## 8. 名词解释

| 名词 | 说明 |
|------|------|
| 风控提醒 | 风控系统检测到风险内容后在聊天流中插入的警告消息 |
| 关键词预筛 | 第一级检测，使用关键词匹配快速过滤无关消息，零成本 |
| LLM 精确判断 | 第二级检测，将预筛命中的消息发送至大语言模型或第三方 API 进行语义级风险确认 |
| 冷却时间 | 同一会话连续触发之间的最短间隔（5 分钟），避免重复提醒干扰用户 |
| 只提醒不阻断 | 风控系统仅插入提醒消息，不阻止原始消息的发送和接收 |

# PRD：实时聊天内容风控

## 1. 概述

### 1.1 功能定位

实时聊天内容风控是客服系统的安全防护模块，在会话进行中对每条文本消息进行实时关键词检测。当检测到涉及资金转账、合同签署、个人证件等高风险内容时，自动在运营后台的预警列表中生成一条预警记录，便于运营人员审查和追溯。该功能**不阻断消息、不在聊天端展示提醒**，不影响正常消息的发送和接收。

### 1.2 功能目标

- 对会话中的高风险内容进行实时检测和预警
- 通过本地关键词匹配检测风险内容
- 命中关键词后自动生成预警记录，统一在运营后台预警列表中展示

### 1.3 功能入口

风控系统为后台自动运行机制，无需用户主动操作。每条文本消息发送或接收后自动触发检测。

### 1.4 影响范围

| 页面/模块 | 改动类型 |
|-----------|---------|
| 后台风控检测服务 | 新增本地关键词检测逻辑 |
| 运营后台 → 词库管理 | 新增页面，管理风控关键词词库 |
| 运营后台 → 预警列表 | 新增页面，查看风控触发记录 |

---

## 2. 核心概念

### 2.1 预警记录

当消息命中风控关键词后，系统自动生成一条预警记录，记录触发消息原文、命中的关键词、发送人、所属项目、会话/聊天标题等信息。所有预警记录统一在运营后台的预警列表中展示，聊天端（客服端和访客端）不做任何提醒展示。

---

## 3. 检测策略

### 3.1 检测流程

1. **关键词匹配**：对消息文本进行本地关键词匹配，未命中则直接放行
2. **生成预警记录**：自动生成预警记录写入运营后台预警列表

### 3.2 检测范围

| 消息类型 | 是否检测 | 说明 |
|---------|---------|------|
| 文本消息 | 是 | 核心检测对象 |
| 文件消息 | 否 | 不检测 |
| 图片消息 | 否 | 不检测 |
| 语音消息 | 否 | 不检测 |
| 系统消息 | 否 | 系统生成的状态消息，无风险内容 |

### 3.3 检测方向

双向监控：对访客发送的消息和客服/AI 回复的消息均进行检测。

---

## 4. 命中行为

关键词命中后，系统自动生成一条预警记录写入运营后台预警列表。聊天端不做任何提醒或阻断。

---

## 5. 交互规则

### 5.1 去重机制

- 同一会话中，访客和客服的每条消息都检测
- 一条消息可能匹配到多个关键词，所有命中的关键词均记录在预警记录中

### 5.2 与现有功能的关系

- 风控检测与 AI Agent 回复、转人工流程并行执行，互不阻塞
- 风控检测不影响已读回执等现有功能
- 会话结束后不再触发风控检测

---

## 5.5 运营后台——词库管理

运营后台「合规管理 → 词库管理」提供风控关键词的后台管理能力。

### 5.5.1 词库列表

| 入口 | 路径 |
|------|------|
| 词库列表 | 运营后台 → 合规管理 → 词库管理 |

操作说明：

| 操作 | 说明 |
|------|------|
| 新增词库 | 弹窗中设置词库名称（必填）和批量输入词（一行一个） |
| 管理 | 进入词库详情页，查看和管理该词库下的所有词条 |
| 清空 | 清空该词库中的所有词条，需二次确认 |

输入约束：

| 字段 | 必填 | 最大长度 | 唯一性 | 字符范围 |
|------|------|---------|--------|---------|
| 词库名称 | 是 | 50 字符 | 全局唯一，不可重复 | 不限 |
| 批量输入词 | 否 | 单条 100 字符 | 同词库内自动去重，不同词库间可重复 | 不限，自动去除首尾空白和空行 |

单次批量添加上限：1000 条。

操作反馈：

| 操作 | 成功反馈 | 失败反馈 | 二次确认 |
|------|---------|---------|---------|
| 新增词库 | Toast "新增成功"，弹窗关闭 | 名称为空：Toast "请输入词库名称"；名称重复：Toast "词库名称已存在" | 无 |
| 清空词库 | Toast "清空成功" | — | 气泡确认框 "确定清空吗？"，按钮 "确定 / 取消" |

分页与排序：

| 规则 | 说明 |
|------|------|
| 默认排序 | 按创建时间倒序（最新创建在前） |
| 默认每页条数 | 10 条 |
| 可选每页条数 | 10 / 20 / 50 / 100 |
| 是否支持跳页 | 是 |

表格列：词库名称、词数量、创建人、更新时间、操作（管理/清空）

### 5.5.2 词库详情

从词库列表点击「管理」进入词库详情页

操作说明：

| 操作 | 说明 |
|------|------|
| 添加 | 弹窗中批量输入词（一行一个），添加到当前词库 |
| 删除 | 删除单条词条，需二次确认 |

输入约束：

| 字段 | 必填 | 最大长度 | 说明 |
|------|------|---------|------|
| 批量输入词 | 至少一条 | 单条 100 字符 | 一行一个，自动去除空白行，同词库内已有的词条自动跳过 |

单次批量添加上限：1000 条。

操作反馈：

| 操作 | 成功反馈 | 失败反馈 | 二次确认 |
|------|---------|---------|---------|
| 添加词条 | Toast "添加成功"，弹窗关闭 | 输入为空：Toast "请输入至少一个词" | 无 |
| 删除词条 | Toast "删除成功" | — | 气泡确认框 "确定删除吗？"，按钮 "确定 / 取消" |

分页与排序：

| 规则 | 说明 |
|------|------|
| 默认排序 | 按创建时间倒序（最新添加在前） |
| 可排序列 | 创建时间（支持升序/降序切换） |
| 默认每页条数 | 10 条 |
| 可选每页条数 | 10 / 20 / 50 / 100 |

表格列：文本、创建时间（支持排序）、操作（删除）

## 5.6 运营后台——预警列表

运营后台「合规管理 → 预警列表」展示所有风控触发记录，便于运营人员审查和追溯。

| 入口 | 路径 |
|------|------|
| 预警列表 | 运营后台 → 合规管理 → 预警列表 |

筛选栏：日期范围（label + 日期选择器）+ 项目名称（label + 输入框）+ 项目ID（label + 输入框）+ 类型（label + 下拉选择：全部/会话/聊天）+ 搜索/重置按钮

类型定义：

| 类型 | 说明 |
|------|------|
| 会话 | 访客和客服之间的会话 |
| 聊天 | 客服主动发起的单聊或群聊（客服与客服，或客服与访客） |

筛选项中的"全部"仅为不做类型过滤的选项，不是类型值。

筛选项默认值与规则：

| 筛选项 | 默认值 | 说明 |
|--------|-------|------|
| 日期范围 | 空 | 包含起止日整天（起始日 00:00:00 至结束日 23:59:59），不可选择未来日期 |
| 项目名称 | 空 | 模糊匹配 |
| 项目 ID | 空 | 模糊匹配 |
| 类型 | 全部 | 可选：全部 / 会话 / 聊天 |

表格列：

| 列名 | 说明 |
|------|------|
| 文本内容 | 触发风控的消息原文。若该消息匹配到多个关键词，在原文下方以标签形式展示所有命中的关键词 |
| 类型 | 消息来源类型，取值为"会话"或"聊天" |
| 发送人 | 发送该消息的用户名称。访客有备注名取备注名，没有取姓名；客服取客服昵称；如果访客/客服修改了名称列表字段需要同步更新 |
| 会话/聊天标题 | 该消息所属会话或聊天的标题；若标题有调整列表需要同步更新 |
| 项目名称 | 该会话所属项目的名称；若项目名称有调整列表需要同步更新 |
| 项目ID | 该会话所属项目的 ID |
| 触发时间 | 风控规则被触发的时间 |

分页与排序：

| 规则 | 说明 |
|------|------|
| 默认排序 | 按触发时间倒序（最新触发在前） |
| 默认每页条数 | 10 条 |
| 可选每页条数 | 10 / 20 / 50 / 100 |
| 是否支持跳页 | 是 |

---

## 5.7 权限说明

运营后台有自己的权限系统，词库管理和预警列表功能默认对外公开。所有已登录的运营后台用户均可访问「合规管理」下的词库管理和预警列表页面，并可执行所有操作（新增词库、清空、添加词条、删除词条、查看预警记录）。

数据范围：预警列表展示所有项目的预警记录，不做项目级数据隔离。如需按项目筛选，通过筛选栏的项目名称/项目 ID 进行。

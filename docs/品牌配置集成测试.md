## 如何测试

### 1. 启动 web-agent

```bash
pnpm --filter web-agent dev
```

访问：`http://localhost:5173`

### 2. 验证默认租户（TWT）

打开浏览器，你应该看到：
- 左侧导航栏顶部显示 "TWT Chat"
- 品牌标记显示 "T"
- 主题色为 `#2F6BFF`（蓝色）

### 3. 切换到 Demo 租户

修改 `packages/branding/src/loader.ts`：

```typescript
export async function loadTenantConfig(): Promise<TenantConfig> {
  const hostname = window.location.hostname

  // 开发环境：加载 demo 租户（测试用）
  if (import.meta.env.DEV) {
    const demoConfig = await import('./tenants/reseller-demo.json')
    return demoConfig.default as TenantConfig
  }

  // ... 其他代码保持不变
}
```

刷新页面，你应该看到：
- 左侧导航栏顶部显示 "Demo 客服系统"
- 品牌标记显示 "D"
- 主题色变为 `#FF6B35`（橙色）

### 4. 创建自己的租户配置

复制 `packages/branding/src/tenants/reseller-demo.json`，创建新的租户配置：

```bash
cp packages/branding/src/tenants/reseller-demo.json packages/branding/src/tenants/my-brand.json
```

编辑 `my-brand.json`，修改：
- `tenantId`: "my-brand"
- `name`: "我的品牌"
- `displayName`: "我的品牌客服系统"
- `primaryColor`: "#00C853"（绿色）

然后在 `loader.ts` 中加载你的配置：

```typescript
const myConfig = await import('./tenants/my-brand.json')
return myConfig.default as TenantConfig
```

## 注意事项

### 开发环境 vs 生产环境

**开发环境：**
- 直接在 `loader.ts` 中返回指定的租户配置
- 适合快速测试不同租户的效果

**生产环境：**
- 根据域名动态加载租户配置
- 从 `/tenants/${hostname}.json` 加载配置文件
- 需要配置 Nginx 反向代理

### CSS 变量覆盖

品牌配置的主题色会覆盖默认的 CSS 变量：

```typescript
document.documentElement.style.setProperty(
  '--agent-color-brand-primary',
  tenantConfig.primaryColor
)
```

这意味着所有使用 `--agent-color-brand-primary` 的地方都会自动更新为租户的主题色。

### 性能优化

租户配置在应用启动时加载一次，不会影响运行时性能。如果需要动态切换租户，可以：

1. 重新加载页面
2. 或者实现一个 `reloadTenant()` 函数，重新加载配置并更新 CSS 变量

## 故障排查

### 问题：品牌名称没有更新

**原因：** `useTenant()` 在组件中调用失败

**解决：** 确保在 `main.ts` 中调用了 `provideTenant(tenantConfig)`

### 问题：主题色没有更新

**原因：** CSS 变量没有正确注入

**解决：** 检查 `main.ts` 中是否有这行代码：

```typescript
document.documentElement.style.setProperty(
  '--agent-color-brand-primary',
  tenantConfig.primaryColor
)
```

### 问题：TypeScript 类型错误

**原因：** `@twt/branding` 包的类型定义没有正确导出

**解决：** 运行 `pnpm install` 重新安装依赖

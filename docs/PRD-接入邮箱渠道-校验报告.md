## 问题清单

### 🔴 严重问题（阻塞开发）

#### A.1 访客/客户发送邮件缺少 Subject（主题）字段

**位置**：3.3.9 节、3.3.10 节

**问题描述**：
- 3.3.9「访客发送邮件」和 3.3.10「客户发送邮件」描述了从列表主动发送邮件的流程，但弹窗中只有富文本编辑区域，**没有邮件主题（Subject）字段**
- 每封邮件必须有 Subject，否则收件人看到的是空主题邮件
- 3.3.2 节提到"Email 会话标题取 subject 字段"，若无 subject 则回退到访客姓名，但主动发起的邮件应当允许填写主题

**建议**：
在发送邮件弹窗中增加「主题」输入字段：
- 必填，最多 200 字符
- 超出时自动截断并显示省略号
- 为空时"确认发送"按钮置灰

#### B.1 OAuth 授权失败/取消的处理未定义

**位置**：3.3.1 节 - 添加邮箱

**问题描述**：
- 只描述了"授权成功后回跳到邮件页面"的正常路径
- 未定义以下异常场景：
  - 用户在 Google OAuth 页面点击「拒绝」或关闭窗口
  - OAuth 授权过程中网络中断
  - Google 返回授权错误（如 scope 不足）

**建议**：
补充异常处理：
- 用户拒绝/关闭授权页面：回跳到 Email 页面，Toast 提示"授权已取消"
- 授权失败：回跳到 Email 页面，Toast 提示"授权失败，请重试"

---

#### B.2 重复添加同一 Gmail 邮箱的处理未定义

**位置**：3.3.1 节 - 添加邮箱

**问题描述**：
- 未说明同一个 Gmail 账户能否被重复添加
- 如果允许重复，会导致一封邮件产生多条会话
- 如果禁止重复，需要定义检测和提示规则

**建议**：
补充唯一性约束：
- 同一个 Gmail 邮箱地址不可重复添加
- 若检测到已存在，Toast 提示"该邮箱已添加，请勿重复操作"

---

#### B.3 邮件发送失败的反馈未定义

**位置**：3.3.4 节 - 邮件编辑器

**问题描述**：
- 编辑器描述了发送成功后的行为（"邮件以客服消息的形式追加到消息列表"、"编辑器内容和附件清空"），但未定义发送失败的处理
- 未定义发送按钮的 loading 状态
- 未定义发送成功的 Toast 提示

**建议**：
补充发送反馈规则：
- 点击发送后，按钮显示 loading 状态，禁止重复点击
- 发送成功：Toast 提示"发送成功"，编辑器清空
- 发送失败：Toast 提示"发送失败，请重试"，编辑器内容保留

#### C.1 「不支持聊天翻译」与邮件消息翻译功能矛盾

**位置**：1.3 节 vs 3.3.3 节 / US-07

**问题描述**：
- 1.3 节明确列出"不支持边写边译、**聊天翻译**、文本润色和远程协助功能"
- 但 3.3.3 节规定"Email 会话中的消息**仅显示翻译操作**"
- US-07 用户故事："hover 邮件消息时能**快速翻译**"
- "聊天翻译"与"消息翻译"概念边界不清，开发者无法判断邮件会话到底支不支持翻译

**建议**：
明确区分两个概念：
- 将 1.3 节修改为"不支持**边写边译**、**文本润色**和远程协助功能"（移除「聊天翻译」）
- 或在 1.3 节增加注释："聊天翻译指编辑器内的实时翻译面板，不影响消息级别的 hover 翻译功能"

#### E.1 删除已连接邮箱时活跃会话的处理未定义

**位置**：3.3.1 节 - 删除邮箱

**问题描述**：
- 只说"删除邮箱后，该邮箱不再接收新邮件"
- 未说明该邮箱关联的进行中会话如何处理
- 未说明其他会话的 From 下拉框中该邮箱是否移除
- 若会话最初接收邮件的邮箱被删，3.3.4 节的默认选中逻辑失效

**建议**：
补充删除影响规则：
- 删除邮箱后，该邮箱关联的进行中会话继续保留，但不再接收新邮件
- From 下拉框中移除已删除邮箱；若会话的默认发件邮箱已被删除，自动切换到列表中第一个可用邮箱
- 若所有邮箱均已删除，编辑器切换为禁用状态

#### E.2 邮件线程匹配逻辑缺失

**位置**：3.3.11 节 / 全文

**问题描述**：
- PRD 提到"后续邮件回复仍可重新激活会话"，但未说明系统如何将新收到的邮件匹���到已有会话
- 邮件线程匹配通常依赖 Message-ID / In-Reply-To / References 等邮件头
- 若匹配逻辑不明确，可能出现同一封回复邮件创建新会话而非追加到已有会话的问题

**建议**：
补充邮件线程匹配规则：
- 新邮件通过 In-Reply-To / References 邮件头匹配已有会话
- 匹配成功：追加到已有会话（若已结束则重新激活）
- 匹配失败：创建新会话
- 同一发件人的无关联新邮件视为独立会话

---

#### E.3 Gmail OAuth Token 过期/外部撤销的处理缺失

**位置**：3.3.1 节 / 全文

**问题描述**：
- Gmail OAuth token 有效期有限，refresh_token 也可能因用户在 Google 账户设置中撤销权限而失效
- PRD 未定义 token 失效后的系统行为和用户通知方式

**建议**：
补充 token 失效规则：
- 系统检测到 token 失效时，邮箱列表中该邮箱状态标记为"连接失效"
- 发送邮件时若发件邮箱 token 失效，Toast 提示"邮箱连接已失效，请重新授权"
- 管理员可在 Email 页面点击"重新连接"触发 OAuth 重新授权

---

#### E.4 新邮件到达的通知方式未定义

**位置**：全文

**问题描述**：
- PRD 描述了邮件"以会话形式进入队列并自动分配"，但未定义新邮件到达时的通知行为
- 3.3.7 节只提到档案分配时有浏览器 Push 通知
- 常规新邮件到达时客服是否收到通知？声音提醒？桌面推送？

**建议**：
补充通知规则：
- 新邮件会话分配给客服后，触发浏览器 Push 通知（与 Web 会话一致）
- 或明确说明"新邮件会话的通知行为与 Web 新会话一致"

#### G.1 访客去重逻辑不明确

**位置**：3.3.11 节

**问题描述**：
- "收到新邮件时系统自动生成一个访客"
- 未说明同一邮箱地址多次发邮件时是否去重
- 如果每封邮件都生成新访客，会导致大量重复访客

**建议**：
补充去重规则：
- 以发件人邮箱地址作为唯一标识进行去重
- 已存在相同邮箱的访客时，新邮件关联到已有访客，不重复创建
- 访客信息（姓名等）以首次收到的邮件为准，不覆盖更新

**位置**：3.3.3 节

## 改进建议

1. **补充邮件主题字段**：3.3.9/3.3.10 主动发送邮件时必须包含 Subject 输入
2. **完善异常处理**：OAuth 失败、邮件发送失败、token 过期等场景需要明确的错误反馈和恢复路径
3. **消除术语矛盾**：1.3 节「不支持聊天翻译」与 3.3.3/US-07 的消息翻译功能需要明确区分
4. **补充邮件线程匹配规则**：新邮件如何关联到已有会话是核心逻辑，必须明确
5. **定义删除邮箱的影响面**：活跃会话、From 下拉框、默认选中等联动处理
6. **补充状态机定义**：以状态图形式定义邮件会话完整生命周期
7. **修正 3.3.10 文案**：「访客姓名」→「客户名称」
8. **统一字符超限处理**：编辑器与弹窗的超限行为保持一致

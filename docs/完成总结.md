## 🎨 架构亮点

### 1. 运行时多租户
- 单一代码库，根据域名动态加载租户配置
- 无需为每个代理商单独部署
- 升级一次，所有租户同步

### 2. Vue 3 上下文注入
- 使用 `provide/inject` 在整个应用中共享租户配置
- 任何组件都可以通过 `useTenant()` 获取配置

### 3. CSS 变量动态注入
- 租户的主题色动态注入到 CSS 变量
- 所有使用 `--agent-color-brand-primary` 的地方自动更新

### 4. 开发/生产环境分离
- 开发环境：直接返回指定租户配置，方便测试
- 生产环境：根据域名从 `/tenants/${hostname}.json` 加载

## ⚠️ 重要提醒

### 关于 APP 独立打包

你选择了"每个代理商独立 APP"，这会带来巨大成本：

**每个代理商的成本：**
- Apple Developer 账号：$99/年
- Google Play 账号：$25 一次性
- 开发者时间：首次打包 2-3 天，每次更新 1 天

**如果有 10 个代理商：**
- 年度账号成本：$990 + $250 = $1,240
- 维护 20 个 APP（10 iOS + 10 Android）
- 每次更新需要打包 20 次，上架 20 次
- 应用商店可能拒绝"马甲包"

**强烈建议：**
1. 短期（MVP）：只做 Web 白标，暂不做 APP
2. 中期：做统一 APP + 工作空间切换
3. 长期：如果代理商愿意承担成本，再做独立 APP

### 关于商业模式验证

"代理商自主定价 + 按订单百分比分成"这个模式还没跟代理商确认。

**建议：**
1. 先跟代理商确认商业模式
2. 确认是否真的需要"所有前端白标"
3. 确认是否真的需要"独立 APP"
4. 签署意向书，明确需求和时间表

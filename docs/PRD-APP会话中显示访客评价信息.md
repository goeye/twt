# PRD - APP 会话中显示访客评价信息

## 需求概述

在 mobile-app 的会话聊天详情、档案、会话信息中显示访客评价信息，与 web-agent 保持一致。

## 功能范围

### 包含功能

- 会话聊天详情中显示评价消息
- 档案页面的会话记录列表中显示评价字段
- 会话信息页面中显示评价字段

### 排除功能

- 不支持在 APP 中提交或修改评价（评价由访客端提交）

## 数据结构

### 评价值定义

评价字段支持三档评价和未评价状态：

- `'satisfied'` - 满意
- `'neutral'` - 一般
- `'unsatisfied'` - 不满意
- `null` 或 `'none'` - 未评价

### 消息对象扩展

在会话聊天详情中，评价以消息形式显示：

```typescript
interface Message {
  id: number;
  type?: 'normal' | 'system' | 'note' | 'rating';
  role?: 'customer' | 'agent';
  sender: string;
  avatar: string;
  time: string;
  content: string;
  ratingValue?: 'satisfied' | 'neutral' | 'unsatisfied';
}
```

### 会话记录对象扩展

在档案页面的会话记录列表中，评价作为会话属性显示：

```typescript
interface ConversationRecord {
  id: string;
  // ... 其他字段
  rating: 'satisfied' | 'neutral' | 'unsatisfied' | 'none';
}
```

## 交互规则

### 会话聊天详情中的评价显示

**评价消息识别**
- 当消息对象的 `type` 字段为 `'rating'` 时，识别为评价消息
- 评价消息在会话结束后由访客提交，客服端仅展示

**评价消息布局**
- 评价消息使用系统消息样式：居中显示，无头像，无气泡
- 评价消息文案格式：`"访客提交了评价：{评价文本}"`
  - `'satisfied'` → "访客提交了评价：满意"
  - `'neutral'` → "访客提交了评价：一般"
  - `'unsatisfied'` → "访客提交了评价：不满意"

**评价消息位置**
- 评价消息按时间顺序插入消息列表中，通常在会话结束系统消息之后
- 如果访客未评价，不显示评价消息

### 档案页面中的评价显示

**会话记录列表**
- 在会话记录的"会话记录"标签页中，每条会话记录显示评价字段
- 评价字段位置：在会话状态、客服、访客、标签等字段之后

**评价文本映射**
- `'satisfied'` → 显示"满意"
- `'neutral'` → 显示"一般"
- `'unsatisfied'` → 显示"不满意"
- `'none'` 或 `null` → 显示"–"

**筛选功能**
- 支持按评价筛选会话记录：全部、满意、一般、不满意、未评价
- 筛选器位置：在档案页面的筛选区域中

### 会话信息页面中的评价显示

**评价字段位置**
- 在"会话信息"标签页的"基础信息"区域中
- 字段标签："会话评价"
- 字段位置：在"接待时间"字段下方

**评价文本映射**
- `'satisfied'` → 显示"满意"
- `'neutral'` → 显示"一般"
- `'unsatisfied'` → 显示"不满意"
- `null` → 显示"–"

**当前实现状态**
- 会话信息页面中已实现评价字段显示（`SessionInfoView.vue` 第 142-144 行）
- 当前使用 mock 数据，评价值为 `'satisfied'`

## 页面位置

**会话聊天详情页面** (`/session/:id`)
- 在消息列表中渲染评价消息
- 评价消息与普通消息、系统消息、备注消息混合显示

**档案页面** (`/archive`)
- 在"会话记录"标签页的会话列表中显示评价字段
- 支持按评价筛选会话记录

**会话信息页面** (`/session/:id/info`)
- 在"会话信息"标签页的"基础信息"区域显示评价字段

## 边界情况

**未评价状态**
- 如果访客未提交评价，会话聊天详情中不显示评价消息
- 档案页面和会话信息页面中显示"–"

**评价值缺失**
- 如果评价字段为 `undefined`，按未评价处理，显示"–"

**评价值异常**
- 如果评价值不在预定义范围内（如 `'unknown'`），显示"–"并记录错误日志

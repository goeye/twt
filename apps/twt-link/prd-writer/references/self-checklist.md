# Self Checklist

输出正式 PRD 前，必须逐项自检：

1. 是否先完成信息架构并获得确认。
2. PRD 是否先完整输出`1.1 业务背景`、`1.2 目标`、`1.3 验收指标`。
3. 若存在页面层，页面整体概览是否按页面仅输出一次，未按功能重复。
4. 编号是否统一为阿拉伯数字分级，且连续无跳级。
5. 每个功能是否完整包含 `f.4 ~ f.8` 五类规则。
6. 每条关键规则是否都保留了清晰字段名，字段标题是否已加粗，必要时是否补充了`**规则依据：** ...`。
7. 功能类型不适用项是否按需说明原因（如有）。
8. 是否覆盖主流程、分支流程、异常流程、权限差异、状态变化。
9. 不确定信息是否全部标注`（待确认）`。
10. 是否无技术实现细节泄漏。
11. 默认归档场景下，是否已在对应项目下创建独立归档文件并给出路径。
12. 若存在冲突指令，是否已按优先级执行并附“偏离说明”。
13. 超长文档场景下，是否已输出分批计划、批次进度与最终合并结论（无该场景可省略）。
14. 归档目录不存在时，是否已先创建目录并成功写入文件（无该场景可省略）。
15. 可选示例项无内容是否已省略，且未出现“空/无/暂无/N/A”等占位词。
16. 是否已按需输出字段标签式内容，且字段标题已加粗（如`**功能描述：**``**用户场景：**``**登录状态：**``**物品名称：**`）。
17. 输入规则中若缺少明确证据，是否给出了具体规则值并标注`（待确认）`，或仅保留已确认信息；且未出现“输入阈值与格式规则（待确认）”这类泛化占位语。
18. 若执行了归档，文件是否保存在对应项目的`prd/`目录，而非 skill 目录、临时目录或通用资料目录。
19. 若未执行归档，是否已明确说明未归档原因与原计划保存路径。

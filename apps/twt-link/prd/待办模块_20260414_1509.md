# 1 背景与目标

## 1.1 业务背景

**业务现状：** 当前项目已提供待办应用的桌面端、移动端、管理端入口，并与日历视图形成联动，但待办能力仍以原型形式呈现，规则口径、状态边界、筛选定义与多端一致性尚未形成可评审的正式需求说明。  
**核心痛点：** 用户缺少统一的待办处理入口与规则说明，难以快速完成创建、筛选、处理、复盘等连续动作；管理员缺少待办应用启停与成员范围的明确配置规则；桌面端、移动端与日历联动的状态同步口径尚未文档化，存在评审和落地偏差风险。  
**触发原因：** 需要将现有待办原型沉淀为正式需求文档，明确管理端配置、桌面端工作台、移动端待办页以及日历联动的业务范围、交互流程、状态规则与异常边界。  
**影响范围：** 企业管理员、组织成员（桌面端）、组织成员（移动端）；影响应用管理、待办工作台、待办详情、评论动态、日历待办展示等核心流程。

## 1.2 目标

**目标一：** 建立待办模块的统一需求基线，覆盖管理端、桌面端、移动端与日历联动。  
**衡量口径：** 正式 PRD 中完整覆盖已确认的信息架构范围，并可直接用于评审、设计与后续实现。  
**目标值或期望区间：** 全量覆盖本次确认范围内的 17 个功能节点。

**目标二：** 缩短用户创建、筛选、处理待办的操作路径，提升任务处理效率。  
**衡量口径：** 用户可在 3 步内完成常用动作，包括创建待办、完成待办、按条件定位待办。  
**目标值或期望区间：** 关键高频动作的主路径不超过 3 步。

**目标三：** 统一多端待办状态定义与展示反馈，减少状态理解偏差。  
**衡量口径：** 待办在未完成、已完成、已删除等关键状态下，桌面端、移动端和日历视图的展示结果一致。  
**目标值或期望区间：** 关键状态在多端展示一致率 100%。

## 1.3 验收指标

**指标名称：** 需求覆盖完整率  
**计算口径：** 已确认信息架构中的功能节点，全部在正式 PRD 中落入对应业务章节并输出完整规则。  
**统计周期：** 本期需求评审阶段。  
**验收阈值：** 100%。  
**数据来源：** 需求评审清单。

**指标名称：** 高优先级待办识别效率  
**计算口径：** 用户进入待办首页后，能否直接在默认页签或快捷标签中定位逾期/今日截止/紧急待办。  
**统计周期：** 上线后首个版本观察周期。  
**验收阈值：** 用户无需额外搜索即可识别高优先级待办（定性验收标准）。  
**数据来源：** 可用性走查记录、用户反馈。

**指标名称：** 多端状态同步准确性  
**计算口径：** 对同一条待办执行创建、完成、恢复、删除操作后，各关联视图的展示状态是否一致。  
**统计周期：** 联调与验收测试阶段。  
**验收阈值：** 关键操作场景通过率 100%。  
**数据来源：** 测试用例执行结果、联调记录。

# 2 待办模块

## 2.1 管理端应用管理页

### 2.1.1 页面整体概览

**页面定位：** 企业管理员在应用管理页查看待办应用，并对应用状态与成员可见范围进行配置。  
**页面结构：** 页面由应用列表区、应用基础信息区、成员范围展示区、操作区组成。  
**信息布局：** 每个应用卡片展示应用名称、归属分类、成员范围标签和操作入口；待办应用支持“设置”和“停用/启用”操作。  

```mermaid
graph TD
A[管理端应用管理页] --> B[应用列表区]
B --> C[待办应用项]
C --> D[应用基础信息]
C --> E[成员范围标签]
C --> F[操作区]
F --> G[设置]
F --> H[停用或启用]
```

### 2.1.2 待办应用启停与可见范围配置

#### 2.1.2.1 功能定义

**功能描述：** 用于管理待办应用在企业内的启用状态及成员可见范围。  
**用户场景：** 企业管理员需要控制哪些成员可使用待办应用，或临时停用待办应用时使用。  
**功能入口与触发方式：** 管理端应用列表中的“待办”应用行，点击“设置”或“停用/启用”。

#### 2.1.2.2 交互流程

**流程说明：** 管理员进入应用管理页后，定位待办应用，选择设置成员范围或切换应用状态；系统保存后在列表内即时回显最新结果。  

```mermaid
flowchart TD
A[进入应用管理页] --> B[定位待办应用]
B --> C{操作类型}
C -->|设置| D[打开待办应用设置]
D --> E[选择成员可见范围]
E --> F[确认保存]
F --> G[回显最新范围]
C -->|停用或启用| H[确认状态切换]
H --> I[更新应用状态]
I --> J[列表回显最新按钮文案]
```

#### 2.1.2.3 前置条件

**登录状态：** 仅已登录管理员可执行。  
**角色与权限：** 仅具备应用管理权限的企业管理员可执行。  
**前置业务条件：** 待办应用已在应用列表中配置展示。  
**依赖配置或前序步骤：** 企业组织架构与成员数据可正常读取。

#### 2.1.2.4 输入规则

**成员可见范围：** 单选，默认展示当前生效范围，初始展示“全部成员”；已确认可展示“全部成员”，其他可选值为“指定成员/指定部门。  
**应用状态：** 二值状态，取值为“已启用/已停用”。  
**确认操作：** 切换应用状态时需二次确认。

#### 2.1.2.5 校验规则

**成员范围：** 未选择成员范围时，禁用「确定」按钮  。
**状态切换：** 管理员确认前不执行状态变更；取消后保持原状态。  
**无权限校验：** 非管理员点击操作入口时，不允许提交操作并提示“无操作权限”。

#### 2.1.2.6 业务规则

**应用启用规则：** 应用启用后，处于可见范围内的成员可正常进入待办应用。
**应用停用规则：** 应用停用后，成员不可从工作台继续进入待办应用（历史待办数据仍然保留 ）；
**成员范围规则：** 成员范围变更后，仅在新范围内的成员继续可见待办应用。  
**权限差异规则：** 普通成员无权调整待办应用状态与范围。

#### 2.1.2.7 展示与交互状态规则

**范围展示：** 页面以标签形式展示当前生效成员范围。  
**状态按钮展示：** 已启用时展示“已停用”，已停用时展示“已启用”。  
**保存反馈：** 配置成功后提示保存成功，并刷新列表展示。  
**禁用态：** 无权限用户不展示入口。
**用户侧从不可见变为可见：**触发刷新后/重新登录后显示应用入口。
**用户侧从可见变为不可见：1、用户未在待办应用中：1）移动端：点击「待办」应用时，显示提示弹窗“无权限访问，请联系管理员”，用户点击「确认」按钮后退出待办应应用进入至「消息」页且待办应用入口隐藏。2）客户端：点击「待办」应用时，显示提示弹窗“无权限访问，请联系管理员”，用户点击「确认」按钮后退出待办应用，默认选中「消息」显示消息列表及右侧初始内容，同时待办应用入口隐藏。2、用户在待办应用中：1）移动端：任何操作都会显示提示弹窗“无权限访问，请联系管理员”，用户点击「确认」按钮后退出待办应应用进入至「消息」页且待办应用入口隐藏。2）客户端：任何操作都会显示提示弹窗“无权限访问，请联系管理员”，用户点击「确认」按钮后退出待办应用，默认选中「消息」显示消息列表及右侧初始内容；同时待办应用入口隐藏。

#### 2.1.2.8 异常处理

**请求失败：** 提示“设置失败，请稍后重试”，用户可再次提交。  
**无权限：** 提示“无操作权限”，用户仅可返回列表。  
**重复提交：** 连续点击保存时，仅受理一次有效请求，其余操作拦截并提示“处理中，请稍候”。  
**超时或中断：** 提示“网络异常，请重新尝试”，用户可重新发起操作。

#### 2.1.2.9 后置条件

**配置结果：** 系统保存最新应用状态与成员可见范围。  
**页面反馈：** 应用列表即时刷新对应标签和按钮文案。  
**影响范围：** 后续成员访问待办应用时按新配置生效。  
**操作记录：** 需要记录管理员操作日志。

#### 2.1.2.10 补充条件

**历史数据兼容策略：** 停用或范围缩小时，既有待办数据保留在个人历史中。  
**例外情况：** 超级管理员始终保留待办应用访问权限。

## 2.2 桌面端待办工作台

### 2.2.1 页面整体概览

**页面定位：** 组织成员在桌面端统一查看、筛选、处理和维护待办事项的主工作台。  
**页面结构：** 页面由左侧导航区、顶部工具区、快捷标签区、待办列表区、新建待办弹窗、待办详情抽屉组成。  
**信息布局：** 左侧导航承载分栏切换；顶部工具区承载搜索、排序、筛选；列表区展示待办卡片；详情抽屉承载待办编辑、动态与评论。  

```mermaid
graph TD
A[桌面端待办工作台] --> B[左侧导航区]
B --> B1[待我处理]
B --> B2[优先处理]
B --> B3[我已处理]
B --> B4[我创建的]
A --> C[顶部工具区]
C --> C1[搜索]
C --> C2[排序]
C --> C3[筛选]
A --> D[快捷标签区]
D --> D1[今日截止]
D --> D2[已逾期]
A --> E[待办列表区]
A --> F[新建待办弹窗]
A --> G[待办详情抽屉]
```

### 2.2.2 待办列表浏览与分栏切换

#### 2.2.2.1 功能定义

**功能描述：** 用于按业务视角切换待办列表并查看不同处理范围内的任务。  
**用户场景：** 用户希望快速区分待处理、高优先级、已完成以及本人创建的待办。  
**功能入口与触发方式：** 桌面端待办工作台左侧导航项。

#### 2.2.2.2 交互流程

**流程说明：** 用户进入待办工作台后默认进入“优先处理”；点击其他分栏时，系统重新计算标题、角标与列表结果。  

```mermaid
flowchart TD
A[进入桌面端待办] --> B[默认展示优先处理]
B --> C[点击任一分栏]
C --> D[加载对应待办结果]
D --> E[更新页面标题和数量]
E --> F{是否有数据}
F -->|有| G[展示待办卡片]
F -->|无| H[展示空态]
```

#### 2.2.2.3 前置条件

**登录状态：** 仅已登录用户可执行。  
**角色与权限：** 在待办应用可见范围内的成员可访问。  
**前置业务条件：** 待办应用处于启用状态。  
**依赖配置或前序步骤：** 用户已有待办数据时展示角标与列表，无数据时展示空态。

#### 2.2.2.4 输入规则

**分栏标签：** 单选切换，可选值为“待我处理 / 优先处理 / 我已处理 / 我创建的”。  
**默认分栏：** 默认进入“优先处理”。  
**数量角标：** “待我处理”和“优先处理”显示角标数量；其他分栏不显示角标

#### 2.2.2.5 校验规则

**分栏切换：** 同一时刻仅允许一个分栏处于激活状态。  
**结果加载：** 分栏切换后，旧分栏结果需被新分栏结果覆盖。  
**空态校验：** 分栏结果为空时必须展示空态，不可保留旧分栏数据。

#### 2.2.2.6 业务规则

**待我处理：** 展示当前用户未完成的待办；
**优先处理：** 展示当前用户未完成且今日截止、已逾期的待办 ；数据排序：默认按照创建时间倒序排列  
**我已处理：** 展示当前用户已完成的待办；数据排序：默认按照标记完成时间倒序排列
**我创建的：** 展示当前用户创建的待办，完成与未完成状态均可展示；数据排序：默认按照创建时间倒序排列  
**标题联动：** 分栏切换后，主标题同步切换为当前分栏名称。

#### 2.2.2.7 展示与交互状态规则

**激活态：** 当前分栏高亮显示。  
**列表展示：** 每张待办卡片展示复选框（选中代表已完成，取消选中待办取消完成）、标题、截止信息、创建时间、创建人、来源和操作按钮（未完成待办列表展示更多、完成待办按钮；已完成待办列表展示更多、已完成按钮；点击更多按钮显示删除按钮弹窗；已完成按钮/完成待办按钮点击可互相切换）。  
**优先级视觉：** 逾期待办突出显示风险色，即将截止待办突出显示提示色；  
**空态：** 无数据时展示“暂无待办内容”。

#### 2.2.2.8 异常处理

**无数据：** 展示空态，用户可切换其他分栏或创建待办。  
**请求失败：** 提示“加载失败，请刷新重试”，用户可点击刷新。  
**无权限：** 提示“当前无访问权限”。

#### 2.2.2.9 后置条件

**页面状态：** 当前分栏状态被更新。  
**数量结果：** 角标与全局待办提醒数按最新数据重算。  
**用户结果：** 用户可继续执行搜索、筛选、查看详情等后续操作。

#### 2.2.2.10 补充条件

**高优先级口径：** “优先处理”仅包含已逾期、今日截止待办；
**列表分页：**以10 条数据为 1 页，采用滚动加载方式分页展示。

### 2.2.3 待办搜索、排序与筛选

#### 2.2.3.1 功能定义

**功能描述：** 用于帮助用户快速定位目标待办，并按业务条件重新组织列表结果。  
**用户场景：** 用户需要按标题检索待办，或按截止时间、完成状态、优先级快速过滤结果。  
**功能入口与触发方式：** 顶部搜索框、排序入口、筛选弹窗、快捷标签“今日截止 / 已逾期”。

#### 2.2.3.2 交互流程

**流程说明：** 用户输入关键词后系统展示下拉结果；用户选择排序或筛选条件后，系统刷新当前列表；快捷标签与筛选弹窗中的截止时间条件联动。  

```mermaid
flowchart TD
A[进入待办列表] --> B{操作类型}
B -->|搜索| C[输入关键词]
C --> D[展示搜索结果下拉]
D --> E[点击结果打开详情]
B -->|排序| F[选择排序字段和方向]
F --> G[刷新列表顺序]
B -->|筛选| H[选择状态优先级截止时间]
H --> I[确认筛选]
I --> J[刷新列表结果并回显筛选态]
B -->|快捷标签| K[点击今日截止或已逾期]
K --> L[联动筛选结果]
```

#### 2.2.3.3 前置条件

**登录状态：** 仅已登录用户可执行。  
**角色与权限：** 用户可正常访问待办列表。  
**前置业务条件：** 当前分栏已加载待办数据。  
**依赖配置或前序步骤：** 搜索、排序、筛选基于当前分栏数据生效。

#### 2.2.3.4 输入规则

**搜索关键词：** 文本输入，选填，按待办标题进行模糊匹配。  
**搜索清空：** 支持一键清空关键词。  
**排序字段：** 单选，可选值为“截止时间 / 创建时间 / 更新时间 / 优先级”。  
**排序方向：** 单选；时间类字段支持“从早到晚 / 从晚到早”，优先级支持“从低到高 / 从高到低”。  
**完成状态：** 单选，候选值为“未完成 / 已完成”；不同分栏存在默认锁定规则。
**优先级：** 单选，候选值为：“较低 / 普通 / 较高 / 紧急”。  
**截止时间：**单选，候选值为：已逾期、今天、明天、未来 7 天（未来 7 个自然日） 
**快捷标签：** 单选切换，可选值为“今日截止 / 已逾期”（仅待我处理、优先处理中显示此快捷标签）。

#### 2.2.3.5 校验规则

**搜索校验：** 关键词为空时不展示搜索结果下拉。  
**排序校验：** 同一时刻仅允许一个排序字段和一个排序方向生效。  
**筛选校验：** “待我处理”和“优先处理”分栏默认锁定“未完成”；“我已处理”默认锁定“已完成”；用户不可取消该默认状态。  
**筛选结果校验：** 无匹配结果时必须展示空态。  
**条件联动校验：** 快捷标签与筛选弹窗中的截止时间条件需互相同步。

#### 2.2.3.6 业务规则

**搜索范围：** 仅按待办标题检索。  
**排序范围：** 在当前分栏结果内排序，不跨分栏合并。  
**筛选范围：** 在当前分栏结果内按完成状态、优先级、截止时间组合筛选。  
**默认排序：** 默认按“创建时间 从晚到早”。  
**快捷标签规则：** 激活“今日截止”时，仅展示今日到期待办；激活“已逾期”时，仅展示已逾期待办；再次点击可取消激活。  


#### 2.2.3.7 展示与交互状态规则

**搜索结果展示：** 以浮层列表展示匹配项，并展示标题与截止信息。  
**筛选激活态：** 存在有效筛选条件时，筛选入口高亮显示。  
**快捷标签激活态：** 当前标签选中后改变底色与字体颜色。  
**空态：** 搜索无结果时展示“无搜索结果”；筛选无结果时展示“暂无待办内容”。

#### 2.2.3.8 异常处理

**无搜索结果：** 提示“无搜索结果”，用户可修改或清空关键词。  
**请求失败：** 提示“操作失败，请重试”，用户可再次选择排序或筛选。  
**重复操作：** 同一筛选条件重复点击不应造成多次重复计算。  
**超时或中断：** 提示“网络异常，请稍后重试”。

#### 2.2.3.9 后置条件

**列表结果：** 当前列表仅保留符合条件的待办。  
**页面反馈：** 排序文案、筛选入口状态、快捷标签状态与结果保持一致。  
**后续动作：** 用户可直接在结果列表中查看详情或处理待办。

#### 2.2.3.10 补充条件

**优先级口径冲突：** 筛选条件与编辑条件中的优先级档位命名不一致，需统一后再评审。

### 2.2.4 新建待办

flowchart TD

**状态结果：** 新待办状态为未完成。

### 2.2.6 完成待办与恢复待办

#### 2.2.6.1 功能定义

**功能描述：** 用于在桌面端将待办从未完成切换为已完成，或从已完成恢复为未完成。  
**用户场景：** 用户完成任务后需要归档处理结果，或误操作后恢复待办继续处理。  
**功能入口与触发方式：** 列表卡片复选框、卡片按钮、详情抽屉顶部状态按钮。

#### 2.2.6.2 交互流程

**流程说明：** 用户点击完成待办按钮后，系统更新状态并调整列表位置；若再次点击已完成按钮，系统恢复为未完成。  

```mermaid
flowchart TD
A[点击完成待办] --> B[校验当前状态]
B --> C{当前是否未完成}
C -->|是| D[更新为已完成]
D --> E[移动至已完成结果区]
E --> F[刷新按钮文案和角标]
C -->|否| G[恢复为未完成]
G --> H[移回未完成结果区]
H --> I[刷新按钮文案和角标]
```

#### 2.2.6.3 前置条件

**登录状态：** 仅已登录用户可执行。  
**角色与权限：** 对当前待办具有处理权限的成员可执行。  
**前置业务条件：** 目标待办存在。  
**依赖配置或前序步骤：** 用户已进入列表页或详情页。

#### 2.2.6.4 输入规则

**状态操作：** 单次点击触发一次状态切换。  
**目标状态：** 可选结果为“已完成 / 未完成”。  
**操作入口：** 列表与详情入口触发结果一致。

#### 2.2.6.5 校验规则

**重复点击校验：** 状态变更处理中不可重复提交。  
**权限校验：** 无处理权限时不允许切换状态。  
**数据存在校验：** 待办已删除时不允许切换状态，并提示待办不存在。

#### 2.2.6.6 业务规则

**完成规则：** 待办完成后进入已完成结果集，并在“我已处理”中展示。  
**恢复规则：** 已完成待办恢复后重新进入未完成结果集。  
**优先处理重算：** 状态变化后，待我处理数量、优先处理数量和全局提醒数需重新计算。  
**展示文案规则：** 完成后按钮文案变为“已完成”，恢复后变为“完成待办”。  
**详情联动规则：** 若详情抽屉已打开，顶部状态按钮与当前状态同步。

#### 2.2.6.7 展示与交互状态规则

**完成态展示：** 卡片使用完成样式，标题可展示删除线或弱化色。  
**恢复态展示：** 卡片恢复未完成样式。  
**反馈提示：** 完成后提示“待办已完成”，恢复后提示“已恢复待办”。  
**位置变化：** 状态变化后列表内的位置和所属结果集同步更新。

#### 2.2.6.8 异常处理

**请求失败：** 提示“操作失败，请稍后重试”，用户可再次点击。  
**网络异常：** 提示“网络异常，请稍后重试”。  
**无权限：** 提示“无处理权限”，用户可仅查看详情。  
**重复提交：** 忽略短时间内的连续重复点击。

#### 2.2.6.9 后置条件

**状态结果：** 待办状态更新为已完成或未完成。  
**列表结果：** 卡片移动至对应结果集。  
**数量结果：** 角标与提醒数被重新计算。  
**联动结果：** 日历视图中的待办状态同步更新。

#### 2.2.6.10 补充条件

**完成定义：** 已完成记录完成人和完成时间。  
**恢复权限：** 已完成待办仅允许原处理人恢复为未完成待办。

### 2.2.7 删除待办

#### 2.2.7.1 功能定义

**功能描述：** 用于删除不再需要保留的待办事项。  
**用户场景：** 用户误创建待办，或待办已失效需要彻底移除时使用。  
**功能入口与触发方式：** 列表卡片更多菜单、详情抽屉更多菜单点击“删除”。

#### 2.2.7.2 交互流程

**流程说明：** 用户点击删除后，系统展示二次确认弹窗；用户确认后删除待办并关闭相关详情或浮层。  

```mermaid
flowchart TD
A[点击删除] --> B[弹出二次确认]
B --> C{是否确认}
C -->|取消| D[关闭弹窗]
C -->|确认删除| E[删除待办]
E --> F[刷新列表与相关视图]
F --> G[提示删除成功]
```

#### 2.2.7.3 前置条件

**登录状态：** 仅已登录用户可执行。  
**角色与权限：** 对当前待办具有删除权限的成员可执行； 
**前置业务条件：** 目标待办存在。  
**依赖配置或前序步骤：** 用户已打开卡片更多菜单或详情更多菜单。

#### 2.2.7.4 输入规则

**删除确认：** 二次确认弹窗必经。  
**确认文案：** 提示“删除的待办不可恢复”。  
**操作结果：** 用户可选“取消 / 删除”。

#### 2.2.7.5 校验规则

**确认校验：** 未确认前不执行删除。  
**权限校验：** 无删除权限时，不展示删除入口或禁止删除。  
**存在校验：** 待办已被删除时不可再次删除。

#### 2.2.7.6 业务规则

**删除规则：** 确认删除后，目标待办从当前结果集移除。  
**不可恢复规则：** 删除后不提供恢复入口。  
**联动规则：** 删除后需同步移除日历中的对应待办展示。  
**详情处理：** 若删除操作发生在详情页，删除成功后关闭详情抽屉。

#### 2.2.7.7 展示与交互状态规则

**确认弹窗：** 居中展示删除确认信息。  
**成功反馈：** 删除成功后提示“删除成功”。  
**刷新方式：** 列表和相关浮层即时刷新。  
**空态反馈：** 删除后若当前结果集为空，展示空态。

#### 2.2.7.8 异常处理

**删除失败：** 提示“删除失败，请稍后重试”。  
**无权限：** 提示“无删除权限”。  
**网络异常：** 提示“网络异常，请稍后重试”。  
**重复提交：** 连续点击删除时仅受理一次。

#### 2.2.7.9 后置条件

**数据结果：** 目标待办被删除。  
**页面结果：** 当前列表、详情页、日历浮层同步移除目标待办。  
**数量结果：** 角标与提醒数被重新计算。  
**日志结果：** 删除行为保留在审计记录。

#### 2.2.7.10 补充条件
  
**附件处理：** 待办删除后附件同步删除

#### 2.3.2.2 交互流程

**流程说明：** 用户进入日历后查看指定日期或时段的待办；点击待办项后，系统展示待办浮层卡片弹窗（显示标题、来源、截止日期字段及查看详情按钮、删除待办按钮、完成待办/已完成按钮、卡片弹窗隐藏按钮）；点击卡片中的查看详情打开待办详情弹窗。

**依赖配置或前序步骤：** 待办与日历已建立关联展示关系。

#### 2.3.2.4 输入规则

**视图类型：** 可选值为“月视图 / 周视图 / 日视图 / 列表视图”。  
**日期选择：** 用户可切换查看不同日期。  
**待办项点击：** 单次点击打开目标待办浮层卡片弹窗。

#### 2.3.2.5 校验规则

**数据匹配校验：** 仅展示与当前日期或时段匹配的待办。  
**状态展示校验：** 已完成待办与未完成待办需有可区分的展示样式。  
**空态校验：** 当月或当前视图无待办时必须展示空态提示。

#### 2.3.2.6 业务规则

**日期归属规则：** 待办按截止日期归入对应日期展示。  
**时间归属规则：** 在周视图和日视图中，待办按截止时间落入对应时间槽。  
**详情联动规则：** 从日历点击待办后，打开的是同一条待办的统一详情。  
**状态联动规则：** 已完成待办在日历中保留展示，但样式弱化。

#### 2.3.2.7 展示与交互状态规则

**月视图展示：** 日期格内展示阳历日期、农历信息和待办项；单条待办展示圆点、完成状态图标和待办标题，标题超长时单行省略。  
**月视图交互：** 单日待办超过 5 条时，日期格内显示前 4 条，并展示“还有X项...”入口；点击单条待办打开待办浮层卡片，点击“还有X项...”打开该日剩余待办列表浮层。  
**周/日视图展示：** 待办按截止时间落入对应小时槽位；同一时段待办不超过 2 条时全部直出，超过 2 条时直出 1 条并展示“还有X项...”入口。  
**周/日视图状态：** 今日对应日期或列高亮展示；若当前查看日期或本周包含今天，则展示当前时间红线并自动滚动到当前时间附近。  
**列表视图展示：** 按日期分组展示待办；左侧展示日期、周几和农历，右侧展示待办列表项；单条待办展示时间、完成状态图标和待办标题。  
**待办浮层展示：** 浮层卡片展示待办标题、来源“来自待办”、截止时间、时区信息“GMT+8”，并提供“查看详情”“完成待办/标记为未完成”“删除”“关闭”操作。  
**完成态展示：** 已完成待办保留展示，圆点和勾选图标弱化，标题显示浅灰色并加中划线。  
**空态：** 月、日、周视图在无待办时不展示待办标记；


#### 2.3.2.8 异常处理

**无数据：** 展示空态，用户可切换日期继续查看。  
**加载失败：** 提示“加载失败，请重试”。  
**无权限：** 提示“无查看权限”，用户不可进入详情。  
**网络异常：** 提示“网络异常，请稍后重试”。

#### 2.3.2.9 后置条件

**查看结果：** 用户可在日历视图中完成待办浏览和详情查看。  
**页面结果：** 当前日历视图保留定位状态。  
**后续动作：** 用户可在详情中继续完成、恢复或删除待办。

#### 2.3.2.10 补充条件

**日/周视图展示上限：** 同一时段最多直接完整展示 2 条待办，超过部分通过“还有X项...”承接。  

### 2.3.3 待办与日历数据同步

#### 2.3.3.1 功能定义

**功能描述：** 用于确保待办列表和日历视图在创建、完成、恢复、删除后的展示结果保持一致。  
**用户场景：** 用户在任一入口处理待办后，希望另一个入口立即呈现同样的状态。  
**功能入口与触发方式：** 在待办列表、待办详情、日历视图中执行创建、完成、恢复、删除等动作时自动触发。

#### 2.3.3.2 交互流程

**流程说明：** 用户在待办相关入口完成操作后，系统刷新日历和待办视图中的对应数据，保持当前页面结果一致。  

```mermaid
flowchart TD
A[用户执行待办操作] --> B{操作类型}
B -->|创建| C[新增待办数据]
B -->|完成或恢复| D[更新待办状态]
B -->|删除| E[移除待办数据]
C --> F[刷新待办列表]
D --> F
E --> F
F --> G[刷新日历视图]
G --> H[保持多入口展示一致]
```

#### 2.3.3.3 前置条件

**登录状态：** 仅已登录用户可执行。  
**角色与权限：** 对目标待办拥有相应操作权限的成员可触发同步。  
**前置业务条件：** 目标待办已与日历形成日期关联。  
**依赖配置或前序步骤：** 至少一个待办入口和一个日历入口处于可用状态。

#### 2.3.3.4 输入规则

**触发动作：** 包含创建、编辑截止时间、完成、恢复、删除。  
**同步范围：** 当前会话内的待办列表与日历各视图。  
**刷新策略：** 自动触发，无需用户手动确认。

#### 2.3.3.5 校验规则

**一致性校验：** 同一待办在多个入口的标题、时间、状态必须一致。  
**日期校验：** 截止时间变化后，需从原日期位置移除并更新到新日期位置。  
**删除校验：** 删除后不得在任一日历视图中继续出现。

#### 2.3.3.6 业务规则

**创建同步：** 新建待办后，日历中对应日期立即展示新待办。  
**状态同步：** 完成或恢复后，日历中的视觉状态与列表状态同步变化。  
**删除同步：** 删除待办后，日历与列表同时移除。  
**时间同步：** 修改截止时间后，日历中的归属日期和时间槽同步更新。

#### 2.3.3.7 展示与交互状态规则

**刷新反馈：** 同步完成后无需额外弹窗提示，以结果刷新为主。  
**当前视图保持：** 在用户当前所在视图内完成局部或整体刷新，不强制跳转视图。  
**详情联动：** 若详情抽屉已打开，展示内容保持最新状态。  
**浮层联动：** 若日历单条待办浮层或“还有X项...”浮层已打开，完成、恢复或删除后浮层内容即时重绘；无剩余待办时关闭对应浮层。

#### 2.3.3.8 异常处理

**同步失败：** 提示“同步失败，请刷新后重试”
**部分成功：** 若列表更新成功但日历未刷新，提示“数据已更新，日历展示稍后刷新”。  
**网络异常：** 提示“网络异常，请稍后重试”。  
**重复操作：** 重复触发同一操作时，不应生成重复待办项。

#### 2.3.3.9 后置条件

**数据结果：** 待办与日历的展示数据保持一致。  
**页面结果：** 用户在不同视图中看到同一条待办的统一状态。  
**统计结果：** 相关数量与提醒状态按最新结果更新。

#### 2.3.3.10 补充条件

**跨端同步时效：** 桌面端与移动端之间需要实时同步。

**确认校验：** 未确认前不得删除。  
**权限校验：** 无删除权限时不允许执行删除。  
**存在校验：** 已删除待办不可重复删除。

#### 2.4.7.6 业务规则

**删除规则：** 删除成功后从移动端列表中移除。  
**详情规则：** 删除后关闭详情页并返回列表。  
**联动规则：** 删除后同步移除相关日历卡片。  
**不可恢复规则：** 删除后不提供恢复入口。

#### 2.4.7.7 展示与交互状态规则

**确认弹窗：** 以全局弹窗形式展示。  
**成功反馈：** 删除后列表即时刷新；单独提示成功文案为“删除成功”。  
**日历反馈：** 若删除的是当前日期对应待办，移动端日历当日列表即时移除该条待办；若当前日期移除后无待办，则切换为当日无待办展示。  
**空态：** 若删除后当前标签无数据，则展示空态。

#### 2.4.7.8 异常处理

**删除失败：** 提示“删除失败，请稍后重试”。  
**无权限：** 提示“无删除权限”。  
**网络异常：** 提示“网络异常，请稍后重试”。  
**重复提交：** 连续确认时仅受理一次。

#### 2.4.7.9 后置条件

**数据结果：** 待办被删除。  
**页面结果：** 详情页关闭，列表与日历结果同步刷新。  
**数量结果：** 标签角标被重新计算。

#### 2.4.7.10 补充条件
 
**附件处理：** 删除待办后附件也一并删除。

### 2.4.8 移动端动态与评论记录

#### 2.4.8.1 功能定义

**功能描述：** 用于在移动端详情页查看待办动态和评论。  
**用户场景：** 用户需要在手机端追踪待办变更历史并补充协同评论。  
**功能入口与触发方式：** 移动端详情页中的“动态 / 评论”标签及评论输入区。

#### 2.4.8.2 交互流程

**流程说明：** 用户在详情页切换动态与评论标签；在评论页输入内容后发送评论，也可删除已有评论。  

```mermaid
flowchart TD
A[进入移动端详情页] --> B[查看动态]
B --> C[切换到评论]
C --> D[输入评论]
D --> E[发送评论]
E --> F[刷新评论列表]
C --> G[删除评论]
G --> H[刷新评论或展示空态]
```

#### 2.4.8.3 前置条件

**登录状态：** 仅已登录用户可执行。  
**角色与权限：** 具有详情查看权限的成员可查看；评论删除权限为评论发布者。  
**前置业务条件：** 目标待办存在。  
**依赖配置或前序步骤：** 用户已进入移动端详情页。

#### 2.4.8.4 输入规则

**标签切换：** 单选，可选值为“动态 / 评论”。  
**评论内容：** 多行文本输入，发送前必须存在有效内容；最大长度为500字符。  
**评论删除：** 单条评论支持删除。  
**评论附件：** 支持新增附件，支持图片和文件（图片与文件的限制同新建待办附件上传的图片和文件限制），每次限制上传一个附件。

#### 2.4.8.5 校验规则

**发送校验：** 评论内容为空时不允许发送。  
**删除校验：** 删除前校验权限。  
**待办存在校验：** 待办已删除时不可继续评论。

#### 2.4.8.6 业务规则

**动态记录：** 至少记录创建、完成、恢复等关键动作。  
**评论规则：** 评论发送成功后加入评论列表。  
**空态规则：** 无评论时展示空态。  
**删除规则：** 删除评论后若列表为空，自动切换为空态展示。

#### 2.4.8.7 展示与交互状态规则

**默认标签：** 详情页默认展示“动态”标签，“评论”标签按切换后展示。  
**动态展示：** 展示操作者、动作内容和时间；当前标签激活时以下划线或高亮样式标识。  
**评论展示：** 展示评论人、评论时间和评论正文；附件型评论展示附件卡片。  
**发送按钮：** 输入有效内容后显示，发送完成后隐藏并清空输入框高度。  
**空态：** 无评论内容时展示“暂无评论内容”或同义空态。

#### 2.4.8.8 异常处理

**发送失败：** 提示“评论发送失败，请稍后重试”。
**删除失败：** 提示“删除失败，请稍后重试”。  
**无权限：** 提示“无评论操作权限”。  
**网络异常：** 提示“网络异常，请稍后重试”。

#### 2.4.8.9 后置条件

**数据结果：** 动态或评论列表按最新结果刷新。  
**页面结果：** 当前标签内容即时更新。

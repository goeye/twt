# 1 背景与目标

## 1.1 业务背景

**业务背景：** Link 系统当前已具备消息应用、应用管理、客户端应用导航、移动端会话页以及访客网页端组件等基础能力，客服模块需要作为 Link 内嵌应用完成统一接入，并承接企业内外部服务场景中的客服接待、访客沟通、群聊协同、安装配置与运营管理。  
**当前现状：** 仓库中已存在 Link 后台管理端的客服应用入口、Link 客户端的客服应用入口、客服管理后台原型、移动端客服会话与群设置页面、访客端网站组件与聊天页面原型。消息收发与消息摘要逻辑已存在，不属于本次新增范围。  
**核心痛点：** 当前客服能力以原型与分散页面形式存在，尚缺少统一的正式 PRD 来明确多端范围、角色权限、启停规则、会话处理规则、访客端自定义映射关系、群聊展示边界以及异常处理口径，容易在评审、设计和实现阶段产生偏差。  
**触发原因：** 需要基于已确认的四端范围，输出一份可直接评审和归档的客服模块正式需求文档，统一 Link 后台管理端、客户端、移动端、访客端的业务边界与规则定义。  
**影响范围：** 企业管理员、客服管理员、客服成员、访客、客户、实施人员；涉及应用管理、客服接待、会话浏览、会话转接、标签管理、快捷回复、安装接入、自定义配置、访客端聊天展示等核心流程。  
**范围边界：** 本期不新增底层消息收发能力，不新增消息摘要能力，不展开灰置中的 AI 知识库、安全控制等扩展模块；仅在被现有原型直接覆盖的范围内定义客服业务规则。  

## 1.2 目标

**目标一：** 建立客服模块的统一需求基线，覆盖 Link 后台管理端、客户端、移动端与访客端。  
**衡量口径：** 正式 PRD 完整覆盖本次确认的信息架构范围，并可直接用于需求评审、交互设计与后续实现。  
**目标值或期望区间：** 全量覆盖本次确认范围内的 25 个功能节点。  

**目标二：** 统一客服应用在 Link 体系内的入口、权限与启停口径。  
**衡量口径：** 后台配置变化后，客户端与移动端客服应用入口展示、禁用反馈和访问结果保持一致。  
**目标值或期望区间：** 核心入口与权限联动场景一致率 100%。  

**目标三：** 统一客服管理后台对访客端展示的配置映射关系。  
**衡量口径：** 安装-自定义页面中可配置项可在访客端首页、消息页、表单页、最小化页形成明确展示结果。  
**目标值或期望区间：** 已确认配置项与访客端展示映射覆盖率 100%。  

**目标四：** 明确群聊场景下多端成员查看与详情访问边界。  
**衡量口径：** 客服端与访客端在群聊成员查看、客服详情、访客详情访问范围上按角色执行一致规则。  
**目标值或期望区间：** 关键群聊边界场景通过率 100%。  

## 1.3 验收指标

**指标名称：** 需求覆盖完整率  
**计算口径：** 已确认信息架构中的功能节点，全部在正式 PRD 中落入对应页面与功能章节，且输出完整规则。  
**统计周期：** 本期需求评审阶段。  
**验收阈值：** 100%。  
**数据来源：** 需求评审清单、信息架构确认稿。

**统计周期：** 联调与验收测试阶段。  
**验收阈值：** 核心联动场景通过率 100%。  
**数据来源：** 联调记录、测试用例执行结果。  

**指标名称：** 访客端配置映射完成率  
**计算口径：** 安装-自定义中已确认的配置项，在访客端对应页面是否存在明确展示或行为变化。  
**统计周期：** 本期设计评审与验收阶段。  
**验收阈值：** 100%。  
**数据来源：** 配置项映射清单、走查记录。  

**指标名称：** 群聊边界规则正确率  
**计算口径：** 在群聊场景下验证“客服端可查看成员及对应详情”“访客端仅可查看群成员列表，不可查看客服详情与访客详情页”等规则是否正确生效。  
**统计周期：** 联调与验收测试阶段。  
**验收阈值：** 100%。  
**数据来源：** 测试用例执行结果、产品走查记录。  

# 2 客服模块

## 2.1 Link 后台管理端应用管理页

### 2.1.1 页面整体概览

**页面定位：** 企业管理员在 Link 后台统一管理客服应用的启停状态、可见范围和超级管理员范围。  
**页面结构：** 页面由应用列表区、应用基础信息区、启停开关区、可见范围区、管理员配置区组成。  
**信息布局：** 应用列表展示客服应用条目与状态；进入应用设置后，顶部展示应用信息与启停开关，中部配置可见范围，底部配置客服管理后台超级管理员。  

```mermaid
graph TD
A[Link 后台应用管理页] --> B[应用列表区]
B --> C[客服应用条目]
C --> D[设置]
C --> E[停用或启用]
A --> F[客服应用设置页]
F --> G[应用基础信息]
F --> H[启停开关]
F --> I[可见范围]
F --> J[管理员配置]
```

### 2.1.2 客服应用启停与可见范围配置

#### 2.1.2.1 功能定义

**功能描述：** 用于管理客服应用在 Link 体系内的启用状态、成员可见范围及客服管理后台超级管理员范围。  
**用户场景：** 企业管理员需要控制哪些成员可以看到并使用客服应用，以及哪些成员可以进入客服管理后台执行配置。  
**功能入口与触发方式：** Link 后台管理端应用管理页点击客服应用“设置”或“停用/启用”。  

#### 2.1.2.2 交互流程

**流程说明：** 管理员进入客服应用设置页后，可选择修改启停状态、可见范围、管理员范围；保存后，系统在列表与各端入口同步生效。  

```mermaid
flowchart TD
A[进入应用管理页] --> B[点击客服应用设置]
B --> C{操作类型}
C -->|调整启停| D[切换应用状态]
D --> E[确认停用或直接启用]
E --> F[刷新列表状态]
C -->|调整可见范围| G[选择全部成员或部分成员]
G --> H[设置部门或成员]
H --> I[保存配置]
I --> J[刷新可见范围标签]
C -->|调整管理员| K[设置超级管理员]
K --> L[保存配置]
L --> M[客服管理后台按新管理员范围生效]
```

#### 2.1.2.3 前置条件

**登录状态：** 仅已登录管理员可执行。  
**角色与权限：** 仅具备 Link 后台应用管理权限的企业管理员可执行。  
**前置业务条件：** 客服应用已在应用管理列表中展示。  
**依赖配置或前序步骤：** 组织架构、成员数据、部门数据可正常读取。  

#### 2.1.2.4 输入规则

**应用状态：** 二值开关，取值为“已启用/已停用”。  
**可见范围：** 单选，取值为“全部成员/部分成员”。  
**部分成员：** 通过设置入口选择部门或成员；未选择任何部门或成员时，展示“暂无所选部门/人员”。  
**管理员：** 通过设置入口选择用户；选中的用户成为客服管理后台超级管理员。  
**停用确认：** 停用时弹出二次确认弹窗，启用时直接生效。  

#### 2.1.2.5 校验规则

**可见范围：** 当选择“部分成员”但未选择任何部门或成员时，不允许保存，提示“请选择部门或成员”（待确认）。  
**管理员：** 超级管理员人数上限未在原型中明确，建议支持多选，数量上限 50 人（待确认）。  
**权限校验：** 非应用管理员不可提交任何配置，提示“无操作权限”。  
**停用确认：** 点击“取消”后保持当前状态不变。  

#### 2.1.2.6 业务规则

**启用规则：** 客服应用启用后，处于可见范围内的成员可在客户端与移动端看到客服应用入口。  
**停用规则：** 客服应用停用后，所有成员不可继续进入客服应用；应用列表中范围标签展示为“已停用”。  
**范围规则：** 可见范围由“全部成员”切换为“部分成员”后，仅新范围内成员保留入口。  
**管理员规则：** 仅被配置为超级管理员的成员可进入客服管理后台执行系统级配置。  
**跨端联动规则：** 后台配置变更后，客户端与移动端在刷新后按新范围生效。  
**既有数据规则：** 停用或缩小范围时，不删除历史会话、标签、快捷回复等既有数据。  

#### 2.1.2.7 展示与交互状态规则

**列表状态展示：** 已启用时操作文案显示“停用”，已停用时显示“启用”。  
**范围标签展示：** 已启用时展示当前可见范围；已停用时展示灰态“已停用”。  
**详情页状态展示：** 设置页顶部展示应用名称、应用描述与启停开关状态。  
**保存反馈：** 配置成功后以页面局部刷新方式更新状态与标签。  
**入口联动展示：** 被移出可见范围的成员刷新后不再显示客服应用入口。  
**无权限展示：** 无权限用户不展示后台配置入口。  

#### 2.1.2.8 异常处理

**请求失败：** 提示“设置失败，请稍后重试”，用户可重新提交。  
**无权限：** 提示“无操作权限”，用户可返回应用列表。  
**重复提交：** 同一时刻仅受理一次保存请求，其余点击拦截并提示“处理中，请稍候”（待确认）。  
**超时或中断：** 提示“网络异常，请重新尝试”，页面保持原状态。  

#### 2.1.2.9 后置条件

**配置结果：** 客服应用启停状态、可见范围、管理员范围被保存。  
**页面结果：** 应用列表即时刷新显示结果。  
**联动结果：** 客户端、移动端、客服管理后台入口权限按新配置生效。  
**日志结果：** 记录管理员操作日志。  

#### 2.1.2.10 补充条件

**例外情况：** 超级管理员始终可见客服应用入口。  
**历史兼容策略：** 配置变更不影响既有历史会话数据的保存。  

## 2.2 客户端客服工作台

### 2.2.1 页面整体概览

**页面定位：** 客服成员在 Link 客户端内使用客服应用完成会话查看、接待处理、群聊协同和访客信息维护。  
**页面结构：** 页面由左侧应用入口区、会话列表区、聊天内容区、聊天工具区、右侧信息区组成。  
**信息布局：** 左侧展示客服应用入口与角标；中部左侧展示会话/群聊列表及筛选；中部右侧展示聊天内容与输入区；最右侧展示访客信息、会话信息或群设置。

```mermaid
A[客户端客服工作台] --> B[应用入口区]
A --> C[会话列表区]
C --> C1[筛选和分类]
C --> C2[会话列表]
A --> D[聊天内容区]
D --> D1[消息区]
D --> D2[输入工具区]
A --> E[右侧信息区]
E --> E1[访客信息]
E --> E2[会话信息]
E --> E3[群设置]
```

### 2.2.2 客服应用入口展示与未读提醒

#### 2.2.2.1 功能定义

**功能描述：** 用于在 Link 客户端展示客服应用入口，并对待处理量进行角标提醒。  
**用户场景：** 客服成员需要从 Link 客户端直接进入客服工作台，并快速感知是否存在待处理会话。  
**功能入口与触发方式：** Link 客户端左侧应用栏点击“客服”。  

#### 2.2.2.2 交互流程

**流程说明：** 客服成员登录 Link 客户端后，系统根据后台可见范围决定是否展示客服入口；有待处理会话时展示角标，点击后进入客服工作台。  

```mermaid
flowchart TD
A[成员登录客户端] --> B{是否在客服应用可见范围内}
B -->|是| C[展示客服应用入口]
B -->|否| D[隐藏客服应用入口]
C --> E{是否有待处理数量}
E -->|有| F[展示角标]
E -->|无| G[不展示角标]
F --> H[点击客服入口]
G --> H
H --> I[进入客服工作台]
```

#### 2.2.2.3 前置条件

**登录状态：** 仅已登录成员可执行。  
**角色与权限：** 仅在后台可见范围内的成员可见入口。  
**前置业务条件：** 客服应用处于启用状态。  
**依赖配置或前序步骤：** 后台已完成客服应用可见范围配置。  

#### 2.2.2.4 输入规则

**应用入口：** 固定应用入口，名称为“客服”。  
**角标数量：** 整数展示，超过 99 时显示“99+”。  
**进入方式：** 单击应用入口进入客服工作台。  

#### 2.2.2.5 校验规则

**权限校验：** 不在可见范围内的成员不可通过常规导航进入客服应用。  
**状态校验：** 客服应用停用时入口不允许进入。  
**角标校验：** 角标数量需与当前待处理量保持同步。  

#### 2.2.2.6 业务规则

**入口展示规则：** 客服入口与日历、待办同级，作为 Link 内嵌应用存在。  
**角标规则：** 角标聚合展示客服相关未处理数量。  
**停用联动规则：** 后台停用后，客户端刷新后不再展示客服入口。  
**权限变更规则：** 成员被移出可见范围后，刷新或重新登录后入口消失。  

#### 2.2.2.7 展示与交互状态规则

**选中态：** 当前进入客服应用时，导航高亮客服入口。  
**角标展示：** 有未处理量时展示红色角标，无数量时隐藏。  
**禁入反馈：** 已在客服应用中的成员若权限被移除，后续任一操作触发“无权限访问，请联系管理员”提示并退出至消息页

#### 2.2.2.8 异常处理

**无权限：** 提示“无权限访问，请联系管理员”，并返回默认消息页。  
**请求失败：** 提示“加载失败，请稍后重试”。  
**超时或中断：** 提示“网络异常，请重新尝试”。  

#### 2.2.2.9 后置条件

**页面结果：** 用户进入客服工作台或停留在原页面。  
**展示结果：** 导航高亮与角标状态完成刷新。  

### 2.2.3 会话/群聊列表筛选与分类浏览

#### 2.2.3.1 功能定义

**功能描述：** 用于在客户端客服工作台按会话类型、未读状态、对象类型等维度浏览会话列表。  
**用户场景：** 客服成员需要快速定位单聊、群聊、访客、客户或未读会话。  
**功能入口与触发方式：** 会话列表头部筛选按钮、下拉项、标签项。  

#### 2.2.3.2 交互流程

**流程说明：** 用户进入客服工作台后，系统默认展示消息列表；用户可通过快捷筛选或分类下拉切换结果集，列表随之刷新。  

```mermaid
flowchart TD
A[进入客服工作台] --> B[默认加载消息列表]
B --> C{选择筛选方式}
C -->|顶部标签| D[切换标签]
C -->|下拉筛选| E[选择维度]
D --> F[刷新列表结果]
E --> F
F --> G{是否有结果}
G -->|有| H[展示会话列表]
G -->|无| I[展示空态]
```

#### 2.2.3.3 前置条件

**登录状态：** 仅已登录客服成员可执行。  
**角色与权限：** 具备客服应用访问权限。  
**前置业务条件：** 当前账号存在客服会话数据或具备接待权限。  
**依赖配置或前序步骤：** 会话列表数据源可正常返回。  

#### 2.2.3.4 输入规则

**筛选标签：** 可选值为“消息/未读/单聊/群聊/客服/客户/访客”。  
**分类下拉：** 与筛选标签口径保持一致。  
**搜索词：** 客户端原型未明确搜索框，当前页面以筛选为主，不新增搜索项。  
**列表项：** 每项至少展示名称、最后消息、时间、未读数、会话类型。  

#### 2.2.3.5 校验规则

**单选校验：** 同一时刻仅允许一个主筛选条件处于激活状态。  
**列表同步校验：** 切换筛选后，必须清空旧结果并刷新为新结果。  
**未读校验：** 未读筛选下仅展示存在未读消息的会话。  

#### 2.2.3.6 业务规则

**消息规则：** 默认展示全部客服相关消息列表。  
**未读规则：** 展示存在未读内容的会话。  
**单聊规则：** 展示一对一会话。  
**群聊规则：** 展示多人群聊会话。  
**客服规则：** 展示以客服成员为主要对象的会话集合。  
**客户规则：** 展示客户对象会话集合。  
**访客规则：** 展示访客对象会话集合。

**离开规则：** 标记离开后，该客服不再接受新的客户分配。

**用户场景：** 企业希望访客端页面视觉与品牌、入口和样式保持一致。  
**功能入口与触发方式：** 访客端首页、消息页、表单页、最小化页加载时自动触发。  

#### 2.7.4.2 交互流程

**流程说明：** 访客端页面加载时读取后台配置，动态渲染品牌信息、页面样式和交互入口。  

```mermaid
flowchart TD
A[访客端页面加载] --> B[读取后台自定义配置]
B --> C[渲染品牌和样式]
C --> D[渲染入口和文案]
D --> E[展示对应页面]
```

#### 2.7.4.3 前置条件

**登录状态：** 访客无需登录。  
**角色与权限：** 面向访客与客户开放。  
**前置业务条件：** 后台自定义配置已保存。  
**依赖配置或前序步骤：** 对应渠道页面可读取配置。  

#### 2.7.4.4 输入规则

**品牌名称：** 文本展示。  
**品牌图标：** 图片或默认图标。  
**组件位置：** 位置配置项。  
**快捷入口：** 列表配置项。  
**Powered by Link：** 开关配置项。  

#### 2.7.4.5 校验规则

**品牌资源：** 图标资源为空时使用默认图标。  
**快捷入口：** 无有效入口时不展示快捷入口区。  

#### 2.7.4.6 业务规则

**来源规则：** 访客端相关逻辑及样式显示均需参考客服管理后台“安装-自定义”配置。  
**位置规则：** 小部件位置由后台配置控制。  
**品牌规则：** 页面头部品牌名称与图标由后台配置控制。  
**标识规则：** Powered by Link 是否显示由后台配置控制。  

#### 2.7.4.7 展示与交互状态规则

**首页展示：** 展示品牌信息和预设欢迎内容。  
**消息页展示：** 按配置展示头部名称、快捷入口和底部品牌标识。  
**最小化展示：** 展示最小化悬浮入口。  

#### 2.7.4.8 异常处理

**配置读取失败：** 退回默认样式并提示“配置加载失败，已使用默认设置”（待确认）。  

#### 2.7.4.9 后置条件

**页面结果：** 访客端按最新配置渲染。  

### 2.7.5 会话状态反馈与辅助信息展示

#### 2.7.5.1 功能定义

**功能描述：** 用于在访客会话过程中展示欢迎语、结束语、离线回复、排队提醒、访客评价、消息已读状态和客服在线状态。  
**用户场景：** 访客希望清晰感知当前接待状态与会话结果。  
**功能入口与触发方式：** 访客在消息页发生对应会话状态时自动触发。  

#### 2.7.5.2 交互流程

**流程说明：** 访客消息页根据后台配置和当前会话状态自动展示不同提示内容。  

```mermaid
flowchart TD
A[进入消息页] --> B{会话状态}
B -->|新会话| C[展示欢迎语]
B -->|离线| D[展示离线回复]
B -->|排队中| E[展示排队提醒]
B -->|已结束| F[展示结束语]
F --> G{是否开启评价}
G -->|是| H[展示访客评价]
B -->|单聊在线| I[展示客服在线状态]
B -->|已读开启| J[展示已读状态]
```

#### 2.7.5.3 前置条件

**登录状态：** 访客无需登录。  
**角色与权限：** 面向访客与客户开放。  
**前置业务条件：** 对应配置项已开启。  
**依赖配置或前序步骤：** 当前会话状态可被识别。  

#### 2.7.5.4 输入规则

**欢迎语：** 文本内容。  
**结束语：** 文本内容。  
**离线回复：** 文本内容。  
**排队提醒：** 文本内容，包含前方人数。  
**访客评价：** 满意/一般/不满意。  
**已读状态：** 开关型展示。  
**客服在线状态：** 单聊场景下展示。  

#### 2.7.5.5 校验规则

**评价触发：** 仅在主动或自动关闭会话后触发。  
**在线状态：** 仅在单聊并开启配置时展示。  
**排队提醒：** 仅排队状态下展示。  

#### 2.7.5.6 业务规则

**欢迎语规则：** 访客发起会话时自动发送。  
**结束语规则：** 会话被客服主动结束或自动结束时自动发送。  
**离线回复规则：** 全部客服不在线或单聊客服不在线时自动发送。  
**排队提醒规则：** 访客进入排队时显示当前排队信息。  
**评价规则：** 会话关闭后可邀请访客评价。  
**已读规则：** 开启后，访客可在会话中看到客服已读状态。  
**在线状态规则：** 开启后，访客可在单聊中看到客服在线状态。  

#### 2.7.5.7 展示与交互状态规则

**消息气泡展示：** 欢迎语、结束语、离线回复以系统消息气泡展示。  
**排队提醒展示：** 以悬浮提示或顶部提示展示。  
**评价展示：** 以评价卡片展示。  
**结束态展示：** 会话已结束时隐藏输入区并展示“会话已结束，请重新咨询”。  

#### 2.7.5.8 异常处理

**状态读取失败：** 按普通消息页展示，不触发对应状态组件（待确认）。  
**提交评价失败：** 提示“评价失败，请稍后重试”（待确认）。  

#### 2.7.5.9 后置条件

**页面结果：** 访客获得当前会话状态反馈，并可继续或结束交互。  

### 2.7.6 群聊成员查看

#### 2.7.6.1 功能定义

**功能描述：** 用于在访客或客户参与的群聊中查看当前群成员列表。  
**用户场景：** 访客希望明确当前群聊中有哪些参与成员。  
**功能入口与触发方式：** 访客/客户群聊页面点击群成员入口。  

#### 2.7.6.2 交互流程

**流程说明：** 访客进入群聊后点击群成员入口，系统展示成员列表，仅供浏览，不进入详情页。  

```mermaid
flowchart TD
A[进入访客群聊] --> B[点击群成员入口]
B --> C[展示群成员列表]
C --> D{点击成员}
D -->|任意成员| E[仅展示基础列表态]
```

#### 2.7.6.3 前置条件

**登录状态：** 访客无需登录。  
**角色与权限：** 仅限当前群聊成员查看。  
**前置业务条件：** 当前会话为群聊。  
**依赖配置或前序步骤：** 群成员数据可正常读取。  

#### 2.7.6.4 输入规则

**成员列表：** 展示成员头像、简称、名称、人数统计。  
**成员点击：** 仅支持点击查看基础列表态，不跳转详情。  

#### 2.7.6.5 校验规则

**详情限制校验：** 访客端点击任一群成员时，不得进入客服详情页，也不得进入访客详情页。  
**权限校验：** 非当前群聊成员不可查看成员列表（待确认）。  

#### 2.7.6.6 业务规则

**可查看范围规则：** 访客/客户消息群聊中支持查看群成员。  
**详情边界规则：** 不支持查看客服详情。  
**详情边界规则二：** 不支持查看访客详情页面。  
**样式来源规则：** 群成员区域的逻辑与样式需参考客服管理后台“安装-自定义”的整体访客端配置风格。  

#### 2.7.6.7 展示与交互状态规则

**成员展示：** 以成员列表或宫格方式展示成员头像和名称。  
**只读态：** 成员项仅用于查看，不展示编辑、添加、移除按钮。  
**人数展示：** 头部展示群成员总数。  

#### 2.7.6.8 异常处理

**加载失败：** 提示“成员加载失败，请稍后重试”。  
**无数据：** 提示“暂无群成员”（待确认）。  

#### 2.7.6.9 后置条件

**页面结果：** 访客完成群成员查看，停留在当前群聊上下文。

**消息能力边界：** 消息收发、消息摘要逻辑已存在，本期仅复用，不在本 PRD 内定义底层实现。

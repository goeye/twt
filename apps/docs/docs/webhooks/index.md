# 功能说明

将 TWT Chat 平台的事件实时推送至你的外部服务，实现业务流程自动化。

## 能做什么

- 实时接收聊天事件（如访客消息超时未回复、新会话创建等）
- 验证请求签名，确保推送来源可信
- 订阅指定事件类型，过滤无关通知

## 怎么做

1. [配置指南](./setup) — 在后台设置 Webhook URL 和事件订阅
2. [签名校验](./signature) — 验证每条推送请求的来源
3. [事件类型](./events) — 查看支持的事件及触发条件
4. [请求体结构](./payload) — 解析推送请求的字段格式

## 能获得什么

- 聊天事件驱动 CRM 更新、工单创建、消息通知等业务流程
- 无需轮询，事件发生时立即触发你的服务
- 与现有业务系统低耦合集成

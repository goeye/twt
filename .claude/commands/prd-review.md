# PRD 文档校验

深度校验 PRD 文档的完整性、一致性和清晰度，发现歧义、矛盾、遗漏和边界问题。

调用方式：`/prd-review docs/PRD-功能名称.md`

参数：$ARGUMENTS（PRD 文档路径）

## 步骤

### 第一步：加载上下文

1. 从参数获取 PRD 路径。若为空，列出 `docs/` 下所有 PRD 文档供选择
2. 完整读取 PRD 文档
3. **读取记忆文件**（不可跳过）：
   - `memory/feedback_prd_writing.md`：PRD 编写规范（禁止项 + 必须项）
   - `memory/feedback_ui_global.md`：全局 UI 约定（Toast 文案、弹窗规范等）
   - `memory/decisions-*.md`：与该功能相关的产品决策记录

### 第二步：执行 8 维校验

逐项检查以下维度，发现问题立即记录，**不可跳过任何检查项**：

#### A. 输入组件完整性

针对文档中**每个用户输入字段**，检查是否定义：
- 必填/选填 → 字符长度上限 → 格式校验规则 → 唯一性约束
- 允许字符范围 → 长文本处理方式（换行/截断/省略号） → 空值显示规则 → 默认值

> ❌ "用户填写备注名称" — 缺少长度限制、必填状态
> ✅ "备注名称选填，最多 50 字符，超出截断省略号，空值显示'--'"

#### B. 交互行为明确性

针对文档中**每个用户操作**，检查是否定义：
- 按钮禁用/启用条件 → 加载状态 → 成功 Toast（须符合全局规范） → 失败提示
- 二次确认弹窗（不可逆操作的标题、描述、按钮文案） → 取消/放弃行为 → 页面跳转时机

> ❌ "点击删除按钮删除角色" — 缺少确认弹窗、成功/失败反馈
> ✅ "点击删除 → 确认弹窗(标题/描述/按钮) → 成功Toast'删除成功' → 失败Toast'{错误原因}'"

#### C. 业务逻辑一致性

检查文档中的规则前后是否矛盾：
- 状态定义一致性 → 枚举值一致性（筛选项 vs 表格列 vs 详情页）
- 数量/配额规则一致性 → 权限规则一致性 → 数据流转逻辑合理性

> ❌ 3.3.1 节说"最多 10 个角色"，6 节说"免费版最多 5 个" — 矛盾

#### D. 状态机完整性

如果涉及状态流转：
- 所有状态是否有定义（含义、触发条件） → 转换路径是否完整 → 转换条件是否明确
- 终态是否明确 → 初始状态是否明确 → 每个状态下允许/禁止的操作

> ❌ "待激活 → 已激活 → 已禁用"，未说明已禁用能否恢复

#### E. 边界情况覆盖度

检查常见边界场景：
- 空数据状态 → 数量上限/下限 → 重复提交幂等性 → 并发冲突
- 依赖缺失（关联数据被删除） → 权限变更降级 → 网络异常

> ❌ 只描述了"删除角色"，未说明角色下有客服时能否删除

#### F. 跨模块影响面

检查功能改动对其他模块的影响：
- 其他列表/详情页展示更新 → 关联数据处理 → 权限联动 → 配额联动

> ❌ 只描述了"角色管理"页面，未提及"客服列表"中角色列的规则

#### G. 歧义与模糊表述

检查是否存在：
- "可以/应该" → 改为"必须/禁止/允许"
- "合适的/适当的" → 改为具体数值或规则
- "等/相关" → 补充完整列表
- "一般/通常" → 明确具体场景
- 多重否定 → 改为肯定表述
- 代词指代不清 → 明确指代对象

> ❌ "用户可以填写相关信息" — "相关"指什么？必填还是选填？

#### H. UI 样式违规检查

**对照 `memory/feedback_prd_writing.md` 的禁止清单**，检查是否出现：
- 颜色描述（"蓝色按钮"、"#333"） → 字体描述（"16px"、"加粗"）
- 尺寸描述（"高度 32px"、"圆角 4px"） → 布局描述（"flex"、"间距 16px"）
- 样式描述（"阴影"、"边框样式"、"渐变"）

#### I. 决策记录一致性（新增）

**对照 `memory/decisions-*.md`**，检查 PRD 内容是否与已确认的决策记录一致：
- 决策文件中明确的规则是否在 PRD 中正确体现
- 是否遗漏了决策文件中已确认但 PRD 未覆盖的功能点
- PRD 中的描述是否与决策记录矛盾

### 第三步：输出校验报告

按以下格式输出：

```markdown
# PRD 校验报告：{功能名称}

> **文档路径**：{路径}
> **校验时间**：{YYYY-MM-DD HH:mm}
> **校验结果**：通过 / 存在问题

---
```

## 注意事项

- 必须**逐字逐句**阅读 PRD，不遗漏任何章节
- 每个问题必须指出**具体位置**（章节号）和**具体内容**
- 每个问题必须给出**明确的修改方案**，不能只说"需要补充"
- 严重程度分级：🔴 缺少关键规则/逻辑矛盾/边界遗漏，🟡 表述不清/一致性问题，🟢 措辞优化/非关键问题
- 如果 PRD 质量很高，如实说明，不为找问题而找问题

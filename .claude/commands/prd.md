# 生成 PRD 文档

根据源码自动生成产品需求文档（PRD），保存至 `docs/` 目录。

调用方式：`/prd 功能名称`（如 `/prd 角色权限管理`、`/prd 访客标签`）

参数：$ARGUMENTS

## 步骤

### 第一阶段：定位源码

1. 从参数中获取功能名称（即 `$ARGUMENTS`）。若参数为空，询问用户要为哪个功能生成 PRD
2. 运行 `git log --oneline -30` 和 `git diff --stat HEAD~10` 浏览近期改动，定位与该功能相关的源码文件
3. 使用 Glob / Grep 搜索 `apps/web-agent/src/` 和 `packages/ui-agent/src/` 下与功能名称相关的 Vue/TS 文件

### 第二阶段：深度阅读（核心页面 + 关联页面）

4. 逐一阅读**核心功能页面**的文件，提取业务逻辑：
   - 数据结构（接口定义、mock 数据、枚举值）
   - 操作流程（CRUD 入口、路由跳转、弹窗触发）
   - 校验规则（表单验证、maxlength、必填项、按钮禁用条件）
   - 文案字符串（弹窗标题、描述文本、Toast 提示语）
   - 保存方式（按钮提交 vs 失焦保存 vs 实时保存）

5. **全局影响面扫描**（关键步骤，不可跳过）：
   用 Grep 搜索核心实体名称（如角色ID、类型枚举、API 路径等）在**整个项目**中的引用，找出所有关联页面。必须逐一检查：
   - 该功能涉及的**其他页面/模块**的改动（如：角色功能 → 客服列表、客服详情、邀请弹窗、个人资料页面等）
   - 不同**用户视角**下的差异行为（如：管理员看到 vs 普通客服看到；查看自己 vs 查看他人）
   - **列表页面**中该实体的所有状态标签、tag 徽章、操作菜单差异
   - 路由守卫、权限校验、403 页面等**权限拦截点**

### 第三阶段：补充排查清单

6. 在写 PRD 之前，对照以下清单逐项排查，确认是否遗漏：

   **A. 实体生命周期**：
   - 该功能涉及的核心实体有哪些**状态**？（如：待激活、已激活、已禁用、已删除）
   - 每个状态下可执行什么操作？不可执行什么操作？
   - 状态之间如何流转？有无不可逆操作？

   **B. 资源约束与限制**：
   - 是否涉及**配额/名额**？（如：坐席数、角色上限）
   - 不同状态的实体是否占用配额？（如：未激活客服是否占坐席）
   - 是否有**频率限制/冷却时间**？（如：重新邀请间隔）
   - 是否有**数量上限**？（如：最多创建 N 个自定义角色）

   **C. 关联页面改动**：
   - 该功能改动是否导致**其他页面**的 UI/交互发生变化？
   - 列表中的标签、徽章、操作菜单是否需要更新？
   - 详情页/编辑页的字段、按钮、保存方式是否有变更？
   - 不同角色/身份的用户看到的页面是否不同？

   **D. 权限正反面行为**：
   - 如果功能涉及权限，每个权限项**具备时能做什么**？
   - **不具备时表现是什么**？（隐藏入口？按钮置灰？403？）

   **E. 外部触点**：
   - 该功能是否需要在**官网/营销页面**中更新权益说明？
   - 是否需要在**定价/套餐对比**中体现差异？
   - 是否需要更新**帮助文档/用户引导**？

### 第四阶段：撰写文档

7. 按下方「PRD 文档大纲」撰写完整文档
8. 保存至 `docs/PRD-{功能名称}.md`
9. 告知用户文档已生成，给出文件路径

## PRD 文档大纲

生成的文档须包含以下章节（可根据功能复杂度合理增减小节）：

```
# PRD：{功能名称}

> 版本：1.0
> 最后更新：{YYYY-MM-DD}
> 状态：已实现

---

## 1. 概述
### 1.1 功能定位
### 1.2 功能目标
### 1.3 功能入口
### 1.4 影响范围
（列出本功能涉及改动的所有页面/模块清单，避免遗漏关联改动）

## 2. 核心概念 / 实体定义
（属性表格、分类、系统预设 vs 自定义等）
### 2.x 实体状态与生命周期
（列出实体的所有状态、每个状态下可执行的操作、状态流转图）

## 3. 数据 / 配置详情
（完整字段清单、层级结构、枚举值）
### 3.x 权限/能力明细表（如涉及权限功能）
（每个权限项：具备时能做什么、不具备时表现是什么）

## 4. 操作流程
### 4.x 列表 / 新增 / 查看 / 编辑 / 删除
（每个操作：入口、流程步骤、页面标题、校验规则、成功提示、保存方式）

## 5. 交互规则
（联动逻辑、状态变化、半选/禁用等细节）

## 6. 关联与依赖
### 6.x 关联页面改动
（逐一说明每个受影响页面的具体变更：列表标签、详情字段、操作菜单、保存方式等）
### 6.x 不同用户视角差异
（管理员视角 vs 普通用户视角、查看自己 vs 查看他人）
### 6.x 资源约束
（配额、名额、频率限制、数量上限等）

## 7. 业务规则汇总表
| # | 规则 | 触发条件 | 预期行为 |

## 8. 测试要点与边界场景
### 8.x 按类别分组的测试用例表格
（须覆盖：核心 CRUD、关联页面改动、状态流转、资源约束边界、不同用户视角）
| 用例 | 操作步骤 | 预期结果 |

## 9. 外部触点
（官网权益说明、定价页面、帮助文档等需同步更新的外部内容。若无则省略本章节）

## 10. 名词解释
| 名词 | 说明 |
```

## 注意事项

- 全文使用**中文**撰写
- 只描述业务逻辑和测试边界，**不涉及**代码实现、技术架构、CSS/UI 结构、组件命名
- 所有数据从源码中提取（接口数据结构、mock 数据、校验规则、弹窗文案等），不凭空编造
- 行文风格和表格格式参考 `docs/PRD-角色权限管理.md`
- 如果功能尚未完整实现，在概述中标注「状态：部分实现」并说明缺失部分
- 章节若无相关内容可省略，但不要填充无意义的占位文字
- **写完后必须回顾第三阶段的排查清单（A-E），逐项确认无遗漏后再输出**
